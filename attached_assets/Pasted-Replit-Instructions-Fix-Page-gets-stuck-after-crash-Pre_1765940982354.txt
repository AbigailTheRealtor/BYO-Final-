Replit Instructions — Fix “Page gets stuck after crash” + Prevent future crashes (Commercial Tenant Broker Compensation)
Issue

In Commercial → Hire a Tenant’s Agent → Broker Compensation, one selection triggers an ArgumentCountError and after that, any click in the tab keeps throwing the error until page refresh.

Root Cause

A dynamic Livewire mount is being called with an empty component name (null/blank). After it happens once, Livewire hydration becomes unstable and every interaction triggers the same failure.

1) Fix the Dynamic Livewire Mount (Must Never Be Blank)

Search in the Commercial Broker Compensation blade/partials for any of these patterns:

@livewire($something)

<livewire:{{ $something }} />

Livewire::mount($something, ...)

If the variable can ever be blank, wrap it:

@if(!blank($componentName))
  @livewire($componentName)
@else
  {{-- fallback view or show the inputs directly --}}
  @include('livewire.tenant-agent-auction-tabs.commission-based.commercial.lease-fee-fallback')
@endif


✅ Do not call Livewire at all if the name is blank.

2) Fix the Mapping for Commercial Lease Fee Types

Your dropdown values for Commercial lease fee types must map to something real for:

Percentage of the Net Aggregate Rent

Flat Fee + Percentage of the Net Aggregate Rent

Make the internal stored values stable keys (recommended):

percent_net_aggregate_rent

flat_plus_percent_net_aggregate_rent

Then map those keys.

If the code is currently mapping from the full label string, it will break easily.

3) Add Transition Guards in the Livewire Component (Prevents “Stuck” State)

In the relevant Livewire component(s) handling Tenant auction broker comp (create, edit, bid, counter if applicable), add “updated” hooks so the fee type can’t become blank during switching:

Example:

public function updatedTenantBrokerLeaseFeeType($value)
{
    if (blank($value)) {
        $this->tenant_broker_lease_fee_type = 'percent_net_aggregate_rent';
    }

    $this->resetValidation();
    $this->resetErrorBag();

    // clear dependent inputs so stale values don’t cause render issues
    $this->tenant_broker_lease_flat_fee = null;
    $this->tenant_broker_lease_percent = null;
}


Do the same for purchase fee type if it’s also dynamic.

✅ This prevents the “back out and click anything → crash again” behavior because the type never sits in a blank/invalid state.

4) Ensure Broker Compensation Tab Does Not Build IDs / Component Names From Raw Labels

If any component name or tab ID is derived from label text, sanitize it or use fixed keys.

5) QA

Commercial → Broker Compensation

Select each lease fee type 10 times rapidly

After switching, click other fields in the tab — no crash

Confirm user does NOT need to restart the listing after switching

What to tell them (copy/paste)

This is not just “one dropdown crash.” After it happens once, Livewire hydration becomes unstable and every click crashes. Fix the root issue: a dynamic Livewire component mount is being called with a blank component name in Commercial broker compensation. Add a fallback so @livewire is never called with an empty value, and add updated hooks so the fee type cannot become blank during transitions.
