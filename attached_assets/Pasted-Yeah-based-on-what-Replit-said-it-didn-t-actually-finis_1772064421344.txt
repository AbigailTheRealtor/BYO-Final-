## ✅ Platform-Wide Fix (Seller/Buyer/Landlord/Tenant)

### Goal

Selecting multiple options should **never flash**, never lose focus, never “lock.”

---

# 1) Delete ALL “message.processed Select2 re-init” code everywhere

In the entire project, search for these patterns and remove/disable any blocks that call `.select2()` inside them:

* `Livewire.hook('message.processed'`
* `message.processed`
* `message.received`
* `message.sent`

Also remove any:

* `select2('destroy')`
* `$(...).empty().select2(...)` (rebuild pattern)
* “reinitialize all select2” loops

✅ **Rule:** Select2 must **NOT** be initialized inside Livewire lifecycle hooks.

**Why:** Livewire updates happen constantly. Re-init during interaction causes the flash and prevents multi-select.

---

# 2) Put `wire:ignore` on every Select2 `<select>` wrapper (everywhere)

For every Select2 field (especially multi-select chips/tags), wrap like:

```html
<div wire:ignore class="select2-wrap">
  <select class="js-select2" data-model="FIELD_NAME" multiple>
    ...
  </select>
</div>
```

* Do this in **Create, Edit, Draft** blades for all listing types.
* If you have conditional rendering (property type changes), wrap the Select2 in a stable container and update options via JS, not Livewire patching.

**Why:** Livewire patching replaces the `<select>` node → Select2 gets destroyed mid-click.

---

# 3) Add ONE global Select2 manager (in app layout, loaded once)

Create a single JS file included site-wide (layout/app.blade.php), e.g. `select2-manager.js`.

### Required behavior

* Initialize Select2 **only if not already initialized**
* Attach exactly one change handler
* Push values to Livewire with `@this.set(...)` (or Livewire.find(componentId))
* **Never** destroy/rebuild
* **Never** run on every message processed

### Canonical pattern (what your dev should implement)

**Initialization function** (pseudo-logic):

* Find all `.js-select2` elements
* If element already has Select2 (`.select2-hidden-accessible`) → skip
* Init Select2
* On change → update Livewire model

And **only re-run init** on:

* initial page load
* Livewire “component loaded”
* Livewire “new DOM inserted” events (not every processed message)

---

# 4) Sync values WITHOUT re-initializing (and don’t sync while dropdown is open)

The “flash then can’t select” often happens because some script tries to **sync values** while the dropdown is open.

So implement a safe sync rule:

✅ Only sync `.val([...]).trigger('change.select2')` if:

* select2 dropdown is **NOT open**
* and the element is **NOT focused**
* and the value is actually different

**Do NOT sync while open**.
