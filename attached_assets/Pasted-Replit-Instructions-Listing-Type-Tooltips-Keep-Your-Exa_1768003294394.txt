B) Dropdown Option Hover Tooltips (Detailed)

These tooltips must appear ONLY when hovering the options inside the open dropdown list.

Required Tooltip Copy (use exactly)
Bidding Period — Hover Tooltip

Agents may submit bids until the bidding timer ends.

During the bidding period, bids cannot be accepted or finalized. Once the timer ends, all Agent bids are revealed for review and comparison, including Services Offered, Broker Compensation & Agency Agreement Terms, and Match Scores.

After the bidding period closes, the listing continues in Traditional mode for reviewing, countering, and accepting bids.

Traditional — Hover Tooltip

Agents may submit bids at any time.

Agent bids are visible as they are received and may be reviewed, accepted, countered, or rejected immediately. There is no bidding timer, and bids are handled on a rolling basis.

Implementation Steps (choose the correct method based on dropdown type)

C) Fix the “Long Popup Still Appearing” Issue (Most Important)

Your screenshot shows a long tooltip/popup still firing in the wrong place. That usually happens because:

Cause 1: A leftover title="..." attribute

Browsers show native tooltips for title even if you add your own.

✅ Fix:

Remove any title attribute from:

Listing Type <select>

option wrappers

Select2/React-Select rendered labels

Only for Listing Type (don’t touch other fields).

Cause 2: The long tooltip is still bound to the dropdown container

Sometimes the tooltip is attached to the field wrapper div instead of the info icon.

✅ Fix:

Ensure the long tooltip is NOT attached to:

.form-group wrapper

dropdown container div

label container

Only attach the short tooltip to the info icon.

Cause 3: Two tooltip systems are active

Example: Bootstrap tooltip + browser title tooltip or Tippy + Bootstrap.

✅ Fix:

For Listing Type, use ONE tooltip system only.

Remove the other trigger bindings for that field.
