# 1) ROOT CAUSE TO FIX: DATE FIELD TYPE/FORMAT BREAKS SUBMIT AFTER EDIT

## What’s happening

When user edits the Date on a Draft, the form state stores a different type/format (Date object / locale string / timestamp), and the submit endpoint rejects it or validation fails — often silently.

## Expected behavior

Editing Date on Draft should NOT affect submission in any role.

## Required implementation (ALL roles: Tenant, Buyer, Seller)

### 1.1 Create ONE canonical date normalization function

Add a shared helper used by **both Save Draft and Submit**:

**Example behavior (do not change UI):**

* If date is a JS `Date` → convert to `YYYY-MM-DD` (or whatever DB expects)
* If date is `MM/DD/YYYY` → convert to canonical
* If date is ISO → keep it
* If date is empty and required → show visible validation error

### 1.2 Call normalization immediately before validation + before API submit

In every submit handler:

* `payload = buildPayloadFromForm()`
* `payload = normalizeDates(payload)` ✅
* `validate(payload)` ✅
* `await submit(payload)` ✅

**Critical:** normalization must run **on submit**, not only on draft save.

### 1.3 Server-side: accept only canonical date format

If backend expects `YYYY-MM-DD`, enforce it consistently and return clear validation errors.
