### 1) Scope & Rules (Must Follow)

1. **Listing Types / Property Types**

   * **Seller & Buyer** support: Residential, Income, Commercial, Business, Vacant Land
   * **Tenant & Landlord** support: Residential, Commercial
2. **Strict Mapping Rule:** The “View Listing” must display the **exact same fields/labels/verbiage** used in the **listing creation form** for that listing type + property type.

   * Do not paraphrase labels.
   * Do not swap in fields from another listing type.
3. **Conditional Display Rule:**

   * If a field was not filled out, it **must not display** on the listing (no empty rows, no placeholders, no “Example”, no blank chips).
4. **“Other” Rule:**

   * If user selected **Other** with a custom input, the listing must display **only the custom input text**, NOT the word “Other”.
5. **Formatting Rules:**

   * Currency: **$ must be before the number** (e.g., **$2,000**, not **2000$**).
   * Percent: **% must be after the number** (e.g., **3%**).
   * Thousands separators: use commas (**1,200** not **1200**).
   * Dates must be spelled out: **January 12, 2026** (not 01-12-2026 or 2026-01-12).
6. **Secondary/Follow-up Questions:**

   * Any follow-up fields should display **only when triggered and filled** (ex: “Garage Needed = Yes” → show “Garage Spaces” only if provided).
7. **Non-Destructive Safeguard:**

   * Do not remove database columns or change existing meaning. If anything is missing, add new fields carefully and map them.
