# FIX 1 — HIRE A BUYER’S AGENT — EDIT REQUIRED FIELD SHOULD HIGHLIGHT IN RED, NOT JUMP SCREEN

## ISSUE

When editing a Buyer listing:

* user removes a required field
* clicks save
* the page jumps/flickers/goes up and down on the field

That makes it hard to type.

## REQUIRED RESULT

For **Hire a Buyer’s Agent** edit flow:

* if user clears a required field and clicks save
* save should be blocked
* the missing field should be clearly marked/highlighted in **red**
* the user should see it is **required**
* the page should **not** keep jumping up and down

### Important

Do **not** remove validation.

Just stop the repeated scroll/focus behavior and keep the required field visibly marked.

## PREFERRED UX

Use the same required-field validation, but make it stable:

* highlight missing field in red
* show required warning/message
* no repeated auto-scroll/focus loop
* user can immediately type into the field without the page bouncing

## FIRST STEP

Inspect the exact Buyer edit validation path for:

* **Save Draft** on edit
* **Submit / Save Edit** on edit

Identify:

1. where missing required fields are detected
2. what script or method scrolls to the missing field
3. what script or method focuses the field
4. whether that logic is firing repeatedly on rerender

## MOST LIKELY ROOT CAUSES TO CHECK

* duplicate listeners attached after rerender
* repeated scroll/focus on every Livewire update
* validation loop continuously trying to take user back to the same field
* auto-focus and auto-scroll fighting each other

## REQUIRED IMPLEMENTATION RULE

For Buyer edit:

* run required validation once per save attempt
* visually mark the missing field
* do **not** repeatedly jump the screen
* let the user type normally after validation fails

### If the Buyer create flow already has field highlighting:

reuse that same visual pattern for edit

### If scroll-to-field exists:

it must happen **once at most**, not continuously
