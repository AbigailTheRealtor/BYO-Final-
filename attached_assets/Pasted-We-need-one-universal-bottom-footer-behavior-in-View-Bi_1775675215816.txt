We need one universal bottom-footer behavior in View Bid across all 4 listing types:

- Tenant
- Buyer
- Seller
- Landlord

Right now the footer behavior is inconsistent.
We need to standardize the footer state machine.

==================================================
GOAL
==================================================

Make the bottom of View Bid behave the SAME across all 4 roles.

Use one universal rule set.

==================================================
RECOMMENDED UNIVERSAL FOOTER RULES
==================================================

STATE 1 — NO COUNTER EXISTS YET
Show:
- Accept Bid
- Counter Bid
- Reject Bid

This is the initial response state.

--------------------------------------------------

STATE 2 — LISTING OWNER HAS ALREADY SUBMITTED A COUNTER AND IS WAITING
Show:
- “You have submitted a counter offer for this bid.”
- View Counter Terms
- Edit Counter Terms

Do NOT keep:
- Accept Bid
- Counter Bid
- Reject Bid

visible in this state.

This should apply consistently across:
- Tenant
- Buyer
- Seller
- Landlord

--------------------------------------------------

STATE 3 — OTHER SIDE HAS COUNTERED BACK AND THIS USER NOW NEEDS TO RESPOND
Show:
- a counter-submitted notice
- View Counter Terms

Recommendation:
Do not duplicate full response actions in the View Bid footer.
Keep the actual response actions on the dedicated View Counter Terms page:
- Accept Counter
- Counter Back
- Reject Counter

Use View Bid footer as a status + navigation area once counters exist.

--------------------------------------------------

STATE 4 — ACCEPTED
Show:
- accepted message/banner
- View Accepted Bid Summary
- E-Sign Summary
- Download PDF if applicable

--------------------------------------------------

STATE 5 — REJECTED
Show:
- rejected message/banner
- Close only

--------------------------------------------------

STATE 6 — LOCKED / EXPIRED / TIMER ACTIVE
Show only the correct notice.

==================================================

Tenant should switch to the same counter-submitted footer pattern as Seller/Landlord:
