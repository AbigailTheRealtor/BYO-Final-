Copy/Paste Replit Instructions (Final Fix: Safe Slugs + One Source of Truth)
Objective

For Tenant, Buyer, Seller, and Landlord wizards:

Clicking Next from Additional Details must open the same Broker Compensation pane as clicking the tab manually.

Broker Compensation must never appear blank.

Next must work on Broker Compensation.

IDs must be stable and safe (no &, no spaces, no punctuation).

1) Stop Using the Tab Label to Generate IDs (Must Use Slugs)

Currently the nav is generating IDs from the label (“Broker Compensation & Agency Agreement Terms”), which creates invalid/unstable IDs.

Required Fix

Create a dedicated slug function and use it for both:

data-bs-target="#..."

pane <div id="...">

Slug rules:

lowercase

replace spaces and punctuation with -

remove anything not [a-z0-9-]

collapse multiple dashes

Example slug:
Broker Compensation & Agency Agreement Terms → broker-compensation-agency-agreement-terms

✅ No ampersand in IDs.

2) Standardize These IDs Across ALL 4 Listing Types

These should be the same everywhere:

listing-details

property-details (or property-preferences, but pick one and stick to it per wizard)

leasing-terms / sale-terms

pre-screening (if applicable)

services

additional-details

broker-compensation-agency-agreement-terms

{role}-information (tenant-information, buyer-information, seller-information, landlord-information)

Do not use tenant-info anymore. Use the full role-based ID.

3) Fix Next/Back to Navigate Only Using data-bs-target IDs

Do NOT compute “tab IDs from labels” or from other strings.

Required navigation approach:

get current active tab’s data-bs-target

get the next tab element by looking up the next ID in an explicit order array OR a DOM-derived list (but DOM-derived must be filtered)

If using DOM-derived order, filter out:

hidden tabs

duplicate tabs

any href-based tabs not using data-bs-target

4) Eliminate Duplicate IDs / Duplicate Panes

A very common reason for “blank when Next, but works when clicked” is that there are two panes with the same or similar IDs and one is empty.

Required

Search in each wizard blade for:

duplicate id="broker-compensation..." pane

duplicate role info panes

duplicate tab markup (Full Service vs Limited Service) that both exist simultaneously in DOM

If there are two sets, only one can be present at a time (or IDs must be namespaced).

5) Make “Next” on Broker Compensation a Real Button and Ensure It Calls the Same Handler

On Broker Compensation pane:

ensure Next is present

ensure type="button"

ensure it calls the same goToNextTab() function

Also ensure no redirect logic exists (no link wrappers).

6) QA Proof Required

For Tenant, Buyer, Seller, Landlord:

Additional Details → Next → Broker Compensation shows fields

Broker Compensation → Next → Role Information

Manual click Broker Compensation shows the same content as Next

Repeat in both Residential + Commercial where applicable

“Do not make pane IDs match ampersand labels. Instead, sanitize tab IDs to safe slugs (no &) and use the same slug for both nav data-bs-target and pane id across all flows.”
