## B) Fix by status code (choose the one that matches)

### ✅ If status is **413 (Payload Too Large)**

PHP limits are not the only limit — the reverse proxy/web server is blocking it.

**Fix:** Raise server/body limits at the web server layer:

* If using **Nginx**: set `client_max_body_size 50M;`
* If using **Caddy**/proxy: set request body max higher (project-specific)
* If Replit proxy has limits, reduce max file size and show a friendly error (fallback).

Also ensure:

* `post_max_size` > `upload_max_filesize`

---

### ✅ If status is **419 (CSRF / session expired)**

Livewire upload fails when:

* session cookie settings / domain mismatch / https issues

**Fix checklist:**

1. Ensure layout includes:

   * `@livewireStyles`
   * `@livewireScripts`
   * `<meta name="csrf-token" content="{{ csrf_token() }}">`
2. Ensure `APP_URL` in `.env` matches the domain you are using.
3. Ensure session cookie config isn’t blocking:

   * `SESSION_SECURE_COOKIE=true` if using https
   * `SESSION_SAME_SITE=lax` (or none if embedded/iframe scenario)

---

### ✅ If status is **422 (validation rejecting file)**

That means file rules or mime rules are wrong.

**Fix:**

* Ensure rules include:

  * `promoMaterials.*.files` is `nullable|array`
  * `promoMaterials.*.files.*` is `file|mimes:...|max:...`

Also note: iPhone “jpg” screenshots sometimes upload as HEIC-ish. Allow:

* `heic` if needed OR show a clear error that HEIC isn’t allowed.

---

### ✅ If status is **500 (server error)**

Almost always temp directory / permissions / disk misconfig.

**Fix checklist:**

1. Ensure folder exists and writable:

   * `storage/app/livewire-tmp`
2. Ensure your `config/livewire.php` temporary upload is correct:

```php
'temporary_file_upload' => [
  'disk' => 'local',
  'directory' => 'livewire-tmp',
],
```

3. Ensure `storage` is writable in the Replit environment.
4. Confirm no code is trying to json_encode the TemporaryUploadedFile before storing.
