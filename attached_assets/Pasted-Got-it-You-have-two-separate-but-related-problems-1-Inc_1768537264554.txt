# PART 1 — Fix the Edit Listing crash (in_array error)

## ✅ What’s happening

Your Blade view is calling something like:

```php
in_array('Other', $sale_provision)
```

…but `$sale_provision` is sometimes a **string** (likely stored as JSON string or comma-separated), not an array. That’s why Edit crashes.

### 📍 WHERE THIS MUST UPDATE

File from your error:
`resources/views/livewire/hire-buyer-agent/buyer-agent-auction-tabs/commission-based/purchasing-terms.blade.php`

---

## ✅ Required Fix (Safe + Minimal)

### A) Make ALL `in_array()` checks array-safe (guard at the view level)

In that Blade file, locate any usage like:

```php
@if(in_array('Other', $sale_provision))
```

Replace with **one safe pattern** (choose one and use it consistently):

### Option 1 (Recommended): Cast the haystack to array inline

```php
@if(in_array('Other', (array) $sale_provision, true))
```

✅ This prevents crashes even if `$sale_provision` is null or string.
**Do this for every `in_array()` call in that file** where the second argument might not be an array.

---

### B) Also fix it at the SOURCE (Livewire hydration) so UI behaves correctly

Even if the view stops crashing, Edit will still act weird unless the property is normalized.

In the **Buyer Edit Livewire component** (the component used by `/buyer/agent/auction/edit/{id}`):

#### Add a normalization step after loading the listing:

**Rule:** Any multiselect field must be an array in Livewire state.

Example for `$this->sale_provision` (adjust property names to match yours):

```php
$this->sale_provision = $this->normalizeToArray($auction->sale_provision);
```

Add helper:

```php
private function normalizeToArray($value): array
{
    if (is_array($value)) return $value;

    if (is_null($value) || $value === '') return [];

    // If JSON string
    if (is_string($value)) {
        $decoded = json_decode($value, true);
        if (json_last_error() === JSON_ERROR_NONE && is_array($decoded)) {
            return $decoded;
        }

        // If comma-separated fallback
        if (str_contains($value, ',')) {
            return array_values(array_filter(array_map('trim', explode(',', $value))));
        }

        // Single value fallback
        return [$value];
    }

    return [];
}
```

### Safeguards

* ✅ Do not change DB storage format in this task
* ✅ Only normalize at runtime in component state
* ✅ Ensure `saveDraft()` / `save()` still writes in the same format currently used
