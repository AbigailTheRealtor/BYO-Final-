## Replit Instructions (WITH SAFEGUARDS) — Hire a Landlord’s Agent (Commercial) Edit Listings

### Fix “Other” inputs not showing for Tenant Pays / Owner Pays / Desired Lease Term / Terms of Lease (Edit + Draft Reopen)

### Applies To

* Login example: `tenant@exp.com`
* Tenant dashboard → **Hire a Landlord’s Agent**
* **Commercial listings only**
* **Edit listings + reopen drafts**
* Issue: When “Other” is selected, the text input does **not** appear (and/or saved “Other” values don’t repopulate).

---

# NON-NEGOTIABLE SAFEGUARDS

### SAFEGUARD A — Edit must hydrate all “Other” toggles on mount

When opening an edit screen (or reopening a draft), the component must:

1. load saved selections
2. set the `showOther...` booleans based on those selections
3. load the saved “Other” text values into the input properties

🚫 Do not rely on `updated()` hooks alone — those only run when the user changes a value, not on initial page load.

---

### SAFEGUARD B — Store BOTH the selection and the “Other” text

For each of these fields, persist two values:

* the selected option (array or string)
* the “Other” free-text input value

**Never** store only “Other” and expect the UI to infer it later.

---

### SAFEGUARD C — Do not wipe “Other” text on unrelated saves

Use merge-save logic:

* Saving other tabs must not overwrite these “Other” values with `null` / empty.

Only clear an “Other” text value if the user explicitly removes “Other” from the selection.

---

# REQUIRED FIXES (Commercial Edit)

## 1) Tenant Pays — “Other” selected but input not shown (Edit)

### Required behavior

* If “Other” is selected in **Tenant Pays**, show the “Other” textbox.
* On edit/draft reopen, if saved selection includes “Other”, textbox must appear and repopulate with saved value.

### Implementation requirements

* Add a boolean state variable (or computed) such as:

  * `showTenantPaysOther`
* On `mount()` / `loadDraft()`:

  * set `showTenantPaysOther = in_array('Other', tenant_pays)` (or equivalent)
  * set `tenant_pays_other = saved value`

✅ Blade must conditionally render input based on `showTenantPaysOther`.

---

## 2) Owner Pays — “Other” selected but input not shown (Edit)

Same fix pattern as Tenant Pays.

### Implementation requirements

* `showOwnerPaysOther`
* `owner_pays_other`
* In `mount()` / `loadDraft()`:

  * set toggle true if “Other” selected
  * set text value from saved data

---

## 3) Desired Lease Term — shows “Other” but no input (Edit)

### Required behavior

* If Desired Lease Term == “Other”, show the text input.
* On edit/draft reopen, auto-show input and repopulate.

### Implementation requirements

* `showDesiredLeaseTermOther`
* `desired_lease_term_other`
* In `mount()` / `loadDraft()`:

  * if selected == “Other” → toggle true and load saved “other” value

✅ If desired lease term is a dropdown, the condition is equality.
✅ If it’s multi-select, use contains/in_array.

---

## 4) Terms of Lease — input not showing (Edit)

This sounds like a conditional text/textarea that should appear when a certain option is chosen (often “Other” or “Custom”).

### Required behavior

* When the trigger option is selected, the Terms of Lease input appears.
* On edit/draft reopen, it must appear automatically if previously triggered, and the saved text must repopulate.

### Implementation requirements

* Identify the trigger selection for Terms of Lease (e.g., “Other”, “Custom Terms”, “Specify”).
* Store:

  * `terms_of_lease_type` (or selection)
  * `terms_of_lease_text` (the input value)
* Add:

  * `showTermsOfLeaseInput` boolean
* In `mount()` / `loadDraft()`:

  * set `showTermsOfLeaseInput` based on saved trigger selection
  * load `terms_of_lease_text`

✅ Blade must render textarea/input when `showTermsOfLeaseInput` is true.
