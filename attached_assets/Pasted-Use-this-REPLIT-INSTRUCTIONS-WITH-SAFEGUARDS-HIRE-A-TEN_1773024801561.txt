## REQUIRED FIXES

### 1) Offered Lease Term looks selected but is not actually showing/saving correctly

Fix **Offered Lease Term** so selected options truly bind, persist, and display correctly.

Requirements:

* When a user selects one or more **Offered Lease Term** options, the values must:

  * remain visibly selected
  * sync correctly to the real Livewire property
  * save in draft
  * save on submit
  * auto-populate on edit
  * display correctly on the actual listing
* Verify this is not a UI-only selection problem where Select2 shows a value but the backend property is not updated.
* Verify there is no:

  * wrong property binding
  * JSON bridge mismatch
  * array vs string mismatch
  * render cycle wiping values
  * hidden tab re-init issue
* If this field uses Select2, make sure the change handler updates the actual Livewire property, not just the visible widget state.

---

### 2) Next button flashes into Submit while filling in fields

Fix the button state logic so navigation actions are stable.

Requirements:

* While filling in fields, the button should **not flicker between Next and Submit**.
* **Next** should appear only on non-final tabs.
* **Submit** should appear only on the final tab.
* The button state must not change temporarily during field entry, blur events, validation refreshes, or Livewire re-renders.
* Verify the issue is not caused by:

  * incorrect current-tab tracking
  * unstable tab index state
  * Livewire re-render resetting button conditions
  * JS tab logic fighting Livewire state
  * final-step detection running too early or too often
* On the final tab, **Submit** must stay visible and clickable.
* On earlier tabs, **Next** must stay visible and clickable.
* This must work in create, draft, and edit.

---

### 3) Acceptable Property Styles says “No results found”

Fix **Acceptable Property Styles** so valid options load and can be selected.

Requirements:

* The field must show the correct options instead of **“No results found”** when valid selections should exist.
* If the field depends on another field such as **Property Type**, confirm:

  * the dependency value is being set correctly
  * the option source is being filtered correctly
  * the options are refreshed correctly after the parent field changes
* Verify the problem is not caused by:

  * empty or incorrectly filtered option array
  * mapping mismatch between saved values and available options
  * Select2 initialization before options are loaded
  * Livewire re-render clearing the available dataset
  * wrong label/value pair format
* The field must work correctly in:

  * create
  * save draft
  * edit
* Saved values must auto-populate correctly on edit and draft restore.
