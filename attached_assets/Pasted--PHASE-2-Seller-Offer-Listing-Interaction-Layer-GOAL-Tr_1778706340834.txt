# PHASE 2 — Seller Offer Listing Interaction Layer

GOAL:
Transform the Seller Offer Listing from a polished static listing page into an interactive listing hub connected to future offer, showing, AI, messaging, and negotiation workflows.

TARGET:
resources/views/offer-listing/seller/view.blade.php

IMPORTANT:
This phase is still frontend/UI-first.
Do NOT build full backend systems yet unless a safe existing route already exists.

CRITICAL SAFEGUARDS:
- ONLY edit resources/views/offer-listing/seller/view.blade.php unless absolutely necessary for a tiny page-scoped asset include.
- DO NOT modify database schema.
- DO NOT add migrations.
- DO NOT change controllers unless absolutely required to expose already-existing data.
- DO NOT add or modify routes unless linking to an already-existing safe route.
- DO NOT modify create/edit/save/draft flows.
- DO NOT touch Hire Agent views.

- DO NOT replace the existing photo/lightbox/gallery system.
- DO NOT remove the existing hero carousel.
- DO NOT introduce React/Vue/Alpine.

- No bidding engine logic yet.

- Schedule Showing

- Ask a Question
- Share Listing
- Listing Activity

“Choose how you want to interact with this property.”
