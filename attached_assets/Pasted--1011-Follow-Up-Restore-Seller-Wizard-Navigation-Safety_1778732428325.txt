## Problem Summary

Task #1011 migrated the wizard navigation to delegated listeners and removed the old button-cloning system. That direction is correct, BUT two regressions were introduced:

1. The Seller Create page lost the inline Next button fallback:

```html
onclick="if(typeof window._wizardNextHandler==='function')window._wizardNextHandler();"
```

Without this safety net, the Next button can intermittently stop responding during:

* Livewire morphs
* draft hydration
* initial load timing
* tab changes

2. The old function:

```js
removeWizardEventListeners()
```

was deleted, but stale calls to it still remain in the JS. Those calls now throw:

```txt
ReferenceError: removeWizardEventListeners is not defined
```

especially during draft load.
