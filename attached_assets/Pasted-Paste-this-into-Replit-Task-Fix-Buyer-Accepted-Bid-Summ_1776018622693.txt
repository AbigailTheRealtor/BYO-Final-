## Normalization rules

Apply these display rules consistently:

### 1. Boolean display

Convert:

* `yes` -> `Yes`
* `no` -> `No`
* `1` -> `Yes`
* `0` -> `No`

Only where the field is truly boolean-like.

### 2. Currency display

Format numeric money values as:

* `$1,000`
* `$5,000`
* `$100,000`

Do not show trailing `.00` unless the system already consistently does so everywhere for Buyer terms.

### 3. Percent display

Format as:

* `5%`
* not `5`
* not `0.05`

### 4. Timeframe display

Format units in human-readable form:

* `90 days`
* `9 Months`

If custom timeframe exists, show the actual custom value, not the word `custom`.

### 5. Title-case human labels

Display:

* `Yes`
* `No`

Do not output lowercase raw enum text unless the source text itself is user-entered freeform.
