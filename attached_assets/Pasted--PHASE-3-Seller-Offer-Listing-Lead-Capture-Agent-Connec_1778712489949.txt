TARGET:

/offer-listing/seller/view/{id}

PRIMARY FILE:
resources/views/offer-listing/seller/view.blade.php

IMPORTANT:
This phase should make Schedule Showing and Ask a Question real lead-capture workflows.

CRITICAL SAFEGUARDS:
- Do NOT modify create/edit/draft flows.
- Do NOT touch Buyer, Landlord, or Tenant views.
- Do NOT touch Hire Agent or Hire Agent Direct flows.
- Do NOT change existing listing data rendering.
- Do NOT remove existing modals, hero, carousel, interaction hub, or lightbox.
- Do NOT build full AI, offer engine, calendar sync, SMS, payments, or negotiation room.

--------------------------------------------------
1. CREATE SIMPLE LEAD CAPTURE STORAGE
--------------------------------------------------

Create one new table if no suitable existing table exists:

seller_listing_inquiries

Suggested fields:
- id
- auction_id nullable/indexed
- type string/indexed
  values:
  - question
  - showing
- name string nullable
- email string nullable
- phone string nullable
- preferred_date date nullable
- preferred_time string nullable
- message text nullable
- question text nullable
- status string default 'new'
- source string default 'seller_offer_listing'
- ip_address nullable
- user_agent nullable
- created_at
- updated_at

Keep migration simple.

Create model:
App\Models\SellerListingInquiry

Fillable fields only.

--------------------------------------------------
2. ADD SAFE CONTROLLER ENDPOINT
--------------------------------------------------

Create a minimal controller or add a small method to an appropriate existing public listing controller.

Preferred route examples:
POST /offer-listing/seller/{auction}/question
POST /offer-listing/seller/{auction}/showing

Requirements:
- Validate inputs.
- Confirm the Seller listing exists.
- Store inquiry linked to auction/listing ID.
- Return back with success flash message.
- Do not expose sensitive errors.
- Do not modify listing data.

Validation:

Question:
- name required|string|max:120
- email required|email|max:160
- phone nullable|string|max:40
- question required|string|max:3000

Showing:
- name required|string|max:120
- email required|email|max:160
- phone nullable|string|max:40
- preferred_date nullable|date
- preferred_time nullable|string|max:80
- message nullable|string|max:3000

Optional anti-spam:
- Add a hidden honeypot field named website.
- If website is filled, silently return success without storing.
