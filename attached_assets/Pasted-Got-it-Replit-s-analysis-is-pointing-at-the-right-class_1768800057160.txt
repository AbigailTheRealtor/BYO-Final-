# ✅ PART A — Fix Services “Next” Button (Make it re-render-proof)

## Problem (confirmed)

Livewire re-renders trigger `removeWizardEventListeners()` which clones buttons → **event listeners are lost**. Current logic conditionally re-attaches handlers based on `currentServiceType`, which becomes `null` on Edit, so handlers never reattach.

### ✅ Required fix (stronger than “default to full_service”):

**Stop relying on “attach once” + “service type must be known.”**
Instead, use **event delegation** (one listener on a stable parent) so cloning/rerendering cannot break it.

---

## Step A1 — Implement event delegation (single listener that never breaks)

In the Services tab script (where initializeFullService lives):

1. Identify a wrapper element that persists across rerenders (examples):

* `#buyer-agent-wizard`
* `.wizard-container`
* the main form container

2. Attach **one** delegated click handler ONCE, guarded to prevent duplicates.

### Add this helper (top-level, not inside initializeFullService)

```js
function attachWizardDelegatedHandlers() {
  if (window.__buyerWizardDelegatedHandlersAttached) return;
  window.__buyerWizardDelegatedHandlersAttached = true;

  document.addEventListener('click', function (e) {
    const nextBtn = e.target.closest('[data-wizard-next]');
    const backBtn = e.target.closest('[data-wizard-back]');

    if (nextBtn) {
      e.preventDefault();
      e.stopPropagation();
      // call the same function Create uses to go to next step
      window.buyerWizardNextStep?.();
      return;
    }

    if (backBtn) {
      e.preventDefault();
      e.stopPropagation();
      window.buyerWizardPrevStep?.();
      return;
    }
  }, true);
}
```

### Then expose the existing next/back logic as global functions

Wherever Create currently advances tabs (the code inside the Next handler), wrap it:

```js
window.buyerWizardNextStep = function () {
  // existing Next logic EXACTLY as Create uses
};

window.buyerWizardPrevStep = function () {
  // existing Back logic EXACTLY as Create uses
};
```

✅ This eliminates the “listener lost due to cloning” problem entirely.
