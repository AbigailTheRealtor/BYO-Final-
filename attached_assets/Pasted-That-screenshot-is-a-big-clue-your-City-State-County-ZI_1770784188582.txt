That screenshot is a **big clue**: your City/State/County/ZIP inputs are being populated with the literal string/boolean **`false`** on edit.

That only happens when the edit component is doing something like:

* setting the Livewire property to `false` (boolean) instead of `''` / `null`, **or**
* using a fallback like `?? false` / `?: false` when hydrating, **or**
* the Blade is printing a boolean (or a “truthy” check) into the input value.

So this is **not** just “not saving” — it’s **wrong hydration defaults**.

# 1) Root Cause (Must Fix)

The edit hydration is assigning boolean `false` (or string `"false"`) to the location fields.

### Common offending patterns (dev must remove)

* `$this->property_city = $draft->property_city ?? false;`
* `$this->property_city = $draft->property_city ?: false;`
* `$this->property_city = isset($draft->property_city) ? $draft->property_city : false;`
* `value="{{ $property_city ?? false }}"` in Blade
* `@if($property_city)` logic that then prints `false`

✅ **Requirement:** Location fields must hydrate to **string or null**, never boolean.

---

# 2) Required Fix — Normalize location values on load (mount/loadDraft)

## A) In the Landlord Edit component `mount()` (or loadDraft method)

Replace any `false` fallbacks with `null` or `''`.

### Required logic (exact behavior)

* If saved value is missing → set to `''` (empty string) or `null`
* If saved value is boolean false → convert to `''`
* If saved value is the string `"false"` → convert to `''`

✅ **Safeguard:** normalize both booleans and strings.

**Normalization rule (must apply to all four fields):**

* `false`, `"false"`, `0`, `"0"` (if those appear) should not render as text in these inputs.

---

# 3) Required Fix — Ensure Blade input binds to Livewire property only

### Requirement

City/State/County/ZIP inputs must be purely:

* `wire:model="property_city"` etc.

🚫 Do not also set a manual `value="..."` attribute that might print `false`.

**Safeguard**
If Blade currently has something like:

* `<input value="{{ $property_city }}">`
  Remove it and rely on Livewire binding.

---

# 4) Required Fix — Save correct keys (so edit loads real values)

Once hydration no longer shows false, confirm these fields are actually saved to the same keys you load:

✅ Must be included in both create and edit payloads:

* `property_city`

**Safeguard**
Do not mix keys like:

Pick one canonical set (recommended: `property_*`) and use everywhere.

---

# 5) Additional Safeguard — Do not allow booleans into these fields when saving

In the save method (draft save + edit save), normalize before persisting:

* trim whitespace
* convert booleans to `null`
* if input is `false` or `"false"` → store `null` or empty string

This prevents bad data from ever entering the DB again.
