## 2) Root Cause Fix (Global): Stop “live” re-rendering from wiping options

### REQUIRED: Replace `wire:model.live` / `wire:model` with `wire:model.defer` for multi-select and conditional triggers

In **EVERY tab** where you have:

* checkbox lists
* multi-select dropdowns
* “Needed: Yes/No” toggles that reveal other fields
* view preferences
* acceptable property conditions
* amenities/features
* offered lease term
* any field that conditionally reveals UI

✅ Change:

* `wire:model` → `wire:model.defer`
* `wire:model.live` → `wire:model.defer`
* `wire:model.lazy` is ok, but prefer `defer` for stability.

Example:

```html
<select wire:model.live="view_preferences_needed">...</select>
```

Becomes:

```html
<select wire:model.defer="view_preferences_needed">...</select>
```

Example checkboxes:

```html
<input type="checkbox" wire:model="view_preferences" value="{{ $opt }}">
```

Becomes:

```html
<input type="checkbox" wire:model.defer="view_preferences" value="{{ $opt }}">
```

**Why:** `live` causes immediate server round-trips and re-renders while the user is clicking, which is why the UI “flashes then disappears.”

## 3) REQUIRED: Add stable `wire:key` wrappers around every conditional block

For every section that shows/hides based on another field (ex: “View Preferences Needed = Yes”), wrap it with a stable key:

```html
<div wire:key="tenant-view-preferences-block">
  @if($view_preferences_needed === 'Yes')
    ... options ...
  @endif
</div>
```

Do this for each conditional section across tabs:

* View preferences
* Acceptable property conditions
* Non-negotiable features/amenities
* Offered lease term other
* Any “Needed” follow-ups (pets, parking, etc.)

**Why:** Without stable keys, Livewire may tear down and recreate the DOM in a way that drops interactivity.

---

## 4) REQUIRED: Prevent “flash then vanish” caused by Alpine initialization timing

If any tab uses Alpine (`x-data`, `x-show`, etc.), add `x-cloak` to the container:

```html
<div x-data x-cloak>
  ...
</div>

<style>[x-cloak]{display:none !important;}</style>
```

Apply specifically to option-heavy areas where you see the flash.

## 5) REQUIRED: Stop “updated()” handlers from resetting arrays/options

In the Tenant create Livewire component, search for methods like:

* `updated($propertyName)`
* `updatedXxx()`
* `hydrate()`
* `mount()` that reassigns option arrays each render
* any code that does `$this->options = ...` inside render cycle

### Fix pattern:

* Ensure all “options arrays” are only assigned in `mount()`, NOT in `render()` and NOT in `updated()` unless necessary.

✅ Example (correct):

```php
public function mount() {
  $this->viewPreferenceOptions = config('...') // once
}
```

❌ Example (causes flash/disappear):

```php
public function render() {
  $this->viewPreferenceOptions = ...
  return view(...)
}
```

If you must compute options dynamically, compute into a local variable in blade or use a cached property that doesn’t get reset.

## 6) Fix “Next / Tab navigation” race conditions (creation flow)

If options disappear when switching tabs, your tab click likely triggers validation + re-render that clears state.

### REQUIRED:

* On “Next” button in create flow wizard:

  * use `type="button"`
  * use `wire:click.prevent`
  * disable during loading

```html
<button type="button"
        wire:click.prevent="nextStep"
        wire:loading.attr="disabled"
        wire:target="nextStep">
  Next
</button>
```

And ensure `nextStep()` does **not** reset tab fields:

* Do not reinitialize arrays
* Do not blank properties unless intentionally
