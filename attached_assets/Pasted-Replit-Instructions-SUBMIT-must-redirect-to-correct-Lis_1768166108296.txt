Fix Common Reasons Listings Don’t Show After Submit

For EACH tab query, confirm and correct these, WITHOUT changing UI:

1) Status filter mismatch

If tab query only shows Active, but submit creates Pending, it will look like it disappeared.

✅ REQUIRED:

Either include the submitted status in the tab query, OR

Ensure submit sets the correct status that the tab expects.

Do not rename statuses. Just align the filter with what submit actually sets.

2) Type discriminator mismatch (if used)

If tab query filters type=2 but submit saves type=null or type=1, you’ll see 0 results.

✅ REQUIRED:

Ensure each submit sets the correct type value expected by that tab query.

3) Ownership / guard mismatch

If tab query filters by user_id and submit saves with a different user/guard, it won’t show.

✅ REQUIRED:

Ensure submit saves with the same user_id/created_by field used by the list queries.

4) Date-based filtering

If any list query filters by listing_date “today,” it will hide records.

✅ REQUIRED:

Remove listing_date gating from list visibility. Visibility should depend on correct listing type + status + ownership rules.
