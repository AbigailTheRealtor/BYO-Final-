PART 3 — TERMS MATCH BLOCK (NEW)
==================================================

Required exact label:

Display directly under Offered Services.

Required format:

Only show segments that exist.

--------------------------------------------------
DEFINITIONS — DO NOT CHANGE LOGIC
--------------------------------------------------

Reuse existing comparison data. Do NOT invent new counting logic.

Matched
- term exists on both sides AND values match exactly

Changed
- field existed in the prior comparison baseline and was modified to a different value
- in practical display terms: same field exists on both sides, but the value changed from the prior version / counter baseline

Missing
- user/listing defined the term
- agent did NOT include it

Added
- agent included a term
- user/listing did NOT define it

IMPORTANT TERMINOLOGY RULE:
- In MAIN BID COMPARISON / VIEW BID style screens, use comparison language like:
  - Mismatch
  - Missing
  - Extra
- In COUNTER HISTORY / COUNTER TERMS style screens, use negotiation-history language like:
  - Changed
  - Added
  - Removed

So:

A. Main bid comparison/private compensation screens:
- field-level row labels should use Mismatch where the values do not align

B. Counter bidding history / counter terms:
- field-level history badges should use Changed / Added / Removed

Do NOT collapse these concepts into one label everywhere.
