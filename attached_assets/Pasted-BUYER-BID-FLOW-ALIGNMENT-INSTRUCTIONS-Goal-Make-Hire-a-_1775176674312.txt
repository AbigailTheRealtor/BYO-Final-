1. SERVICES MUST AUTOPOPULATE FROM THE BUYER LISTING
==================================================

Problem to fix:
When an agent bids on a Hire a Buyer’s Agent listing, the “Services the Buyer Requests from Their Agent” section must auto-populate from the exact services the Buyer selected on the original listing, same as Tenant and Landlord. It must not be blank, stale, mixed, or replaced with a different catalog.

Required behavior:
- When buyer creates listing and checks requested services, those selected services become the baseline requested services for that listing.
- When agent opens bid form, those same selected services must already be shown in the same format used for Tenant/Landlord.
- When buyer later views the bid, the requested services comparison must reference the buyer’s original selected services.
- When counter bids occur, the latest counter terms must carry forward the correct buyer requested services display and comparison logic.

Important:
This is NOT a new independent services list.
It is the buyer listing’s requested services baseline, same concept as Tenant and Landlord.

Implementation rules:
- Reuse the same data loading pattern used in Tenant flow for listing-selected services.
- Reuse the same filtering/sanitizing logic used in Landlord and Seller fixes so only valid buyer services tied to the buyer property type are shown.
- Ensure stale services from another role/property type are never injected.
- Filter loaded services against the current Buyer service catalog before rendering.
- Preserve section headings, grouping, emoji headings, order, and checked state exactly as buyer listing creation defines them.

Expected label:
“Services the Buyer Requests from Their Agent”

If “Other – Specify additional services as needed” exists in buyer flow and was selected, preserve the exact same logic and display behavior currently used in buyer listing creation/edit/view.
