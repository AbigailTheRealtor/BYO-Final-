### Objective

Any field that exists across multiple listing types must render with:

* the **same label**
* the **same formatting**
* the **same merge rules** (Yes/No + child fields)

* the **same hide/show logic**

🚫 This is SHOW-page only. Do NOT change create/edit logic or DB.

# 1) GLOBAL RULES (APPLY EVERYWHERE)

## 1.1 Formatting (mandatory)

* **Money:** `$800,000` (currency + commas)
* **Percent:** `5%` (no trailing decimals unless user entered them)
* **Dates:** `Feb 5, 2026`
* **Yes/No:** display exactly `Yes` or `No` (never `1/0`, never blank)
* **Lists:** render as **pills** for multi-selects; use comma list only inside a single merged line (see rules below)
* **Hide empties:** If value empty → do not render the row at all
* **Suppress placeholders:** If stored value starts with `Example:` or equals known placeholder copy → treat as empty and hide row

## 1.2 “Other” rule (mandatory)

If a selection includes **Other** and user typed custom text:

* Display **only the custom text**
* Do **not** display the word “Other”
* If custom text is empty → hide that “Other” value entirely

## 1.3 Dependent field rule (mandatory)

If a parent field is **No**, hide all dependent/child rows.
If a parent field is **Yes**, display parent and merge child values into the parent row when possible (rules below).

# 2) STANDARD MERGE PATTERNS (USE THESE HELPERS)

Create view-layer helper functions (or a Presenter class) and use these everywhere:

### 2.1 Yes + Count

`formatYesCount(parentYesNo, count, suffix)`

* If parent = Yes and count exists → `Yes (X suffix)`
* If parent = Yes and count empty → `Yes`
* If parent = No → `No` and hide count row

**Example:**
Carport Needed + # spaces → `Yes (3 Spaces)`

### 2.2 Yes + List

`formatYesList(parentYesNo, list[])`

* If parent = Yes and list has values → `Yes (A, B, C)`
* If parent = Yes and list empty → `Yes`
* If parent = No → `No` and hide list row

**Example:**
Pool Needed + types → `Yes (Private, Community)`

### 2.3 Yes + Text

`formatYesText(parentYesNo, text)`

* If parent = Yes and text exists → `Yes (text)`
* If parent = Yes and text empty → `Yes`
* If parent = No → `No` and hide text row

### 2.4 Normalize list values

`normalizeList(list[])`

* Remove blanks
* Remove placeholder “Example: …”
* Replace “Other” with custom text if present

## 3.4 Pets / Animals Group (compact but complete)

### Pets Allowed

* Parent: `Pets Allowed` or `Pets` (Yes/No)
* Children (only show if Yes):

  * `Number of Pets`
  * `Type of Pets` (multi-select or text)
  * `Breed of Pets` (text)
  * `Weight of Pets (lbs)` (number)

**Required rendering (consistent across all):**
Option A (clean + compact): merge into one line + optional detail lines.

**Line 1 (always):**

* **Pets Allowed:** `Yes (3)` OR `Yes` OR `No`

**Optional detail lines (only if values exist):**

* **Pet Type:** Dog, Cat
* **Breed:** Lab
* **Weight:** 50 lbs

🚫 Do not show “Number of Pets” as a separate line if already merged into `Pets Allowed: Yes (3)`.

### Service Animal

* Parent: `Service Animal` (Yes/No)
* Child: (if you have any detail fields) show only if Yes

**Render:**

* **Service Animal:** Yes/No

### Emotional Support Animal

* Parent: `Emotional Support Animal` (Yes/No)
* Render Yes/No
