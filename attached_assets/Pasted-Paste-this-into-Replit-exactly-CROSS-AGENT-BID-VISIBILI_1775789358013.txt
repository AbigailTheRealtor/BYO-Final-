**CROSS AGENT BID VISIBILITY RULE — ALL 4 HIRE FLOWS**
**SELLER / BUYER / LANDLORD / TENANT**
**READ FULLY BEFORE MAKING ANY CHANGES**

## Goal

Apply one consistent privacy rule across all 4 Hire an Agent listing types:

* Hire a Seller’s Agent
* Hire a Buyer’s Agent
* Hire a Landlord’s Agent
* Hire a Tenant’s Agent

### Final visibility rule

#### Listing owner can:

* see all agent bid cards
* click **View Bid**
* click **View Counter Terms**
* see each agent’s full bid details
* see each agent’s full counter history / counter terms

#### Agents can:

* fully view **their own** bid and own counter flow
* see a **limited summary only** for other agents’ bids on that same listing

### Limited summary agents may see for other agents

Only this information may be visible from other agents’ bids:

* **Offered Services: X/X matched**
* **Terms Match: X/X matched**
* **— affects match score**
* **Match Summary**
* **Original Match %**
* **vs. Listing Owner’s Original Request**
* **Services %**
* **Terms %**
* **Counter Match %**
* **vs. Your Latest Counter**
* **Services %**
* **Terms %**
* **Added services or terms do not increase either score**

### Agents must NOT see from other agents

* full offered service checklist
* full terms content
* full bid amounts / commission details / fee details
* full counter terms
* counter bid history
* private notices intended for that bid owner
* action buttons for another agent’s bid
* View Bid button for another agent’s bid
* View Counter Terms button for another agent’s bid
* any full text of Additional Terms or custom text fields

# PART 6 — STABLE AGENT ALIAS MAP

Build one stable alias map per listing page and reuse it everywhere on that page.

## Rules

* only actual agent bidders get aliases
* exclude listing owner user_id
* same real agent keeps same alias everywhere on that page
* do not regenerate aliases per section, per accordion, or per modal

Use actual collection names from the page, but logic should be equivalent to:

```php
@php
$agentAliasMap = [];
$agentIndex = 1;

$uniqueAgentBidders = collect($bids)
    ->filter(fn($b) => $b->user_id !== $auction->user_id)
    ->sortBy([
        ['created_at', 'asc'],
        ['id', 'asc'],
        ['user_id', 'asc'],
    ])
    ->pluck('user_id')
    ->unique()
    ->values();

foreach ($uniqueAgentBidders as $agentUserId) {
    $agentAliasMap[$agentUserId] = 'Agent ' . $agentIndex;
    $agentIndex++;
}
@endphp
```

Then everywhere on that page that prints the bidder label, use:

```php
{{ $agentAliasMap[$bid->user_id] ?? 'Agent' }}
```

### Remove bad numbering sources

Do not use:

* `$loop->iteration`
* per-section counters
* numbering inside filtered subsections
* independent alias maps in accordions / modals / counter history sections

That is what causes the same real agent to become Agent 1 in one place and Agent 2 in another.
