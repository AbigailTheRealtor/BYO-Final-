## Task: Add “Optional Document Sharing (Recommended)” section to Acknowledgement Summary

We are adding a new section to the **Acknowledgement Summary** flow that appears when a user accepts an agent bid.

This applies to:

* Buyer
* Tenant
* Seller
* Landlord

This is the point where users may optionally share trust-related supporting documents with their **selected agent only**.

Important:

* **ID is not uploaded earlier in the listing flow**
* ID upload happens only here, at the Acknowledgement Summary stage
* this section must remain optional
* this section must not block the acceptance flow

---

## Product goal

Add a clean, trust-building step at the right moment in the user journey.

This section should feel:

* optional
* professional
* reassuring
* useful
* not overwhelming

It should help users share supporting documents **after they have chosen an agent**, when trust and intent are highest.

---

## Section title

Use this exact title:

### **Optional Document Sharing (Recommended)**

---

## Introductory text

Add this short helper text directly under the section title:

> Sharing these documents can help your selected agent move forward faster and improve the overall process.

Add this privacy note below that:

> Documents shared here will only be visible to you and your selected agent.

Do not use language that implies mandatory verification.

---

## Core UX rules

* This section appears on the Acknowledgement Summary page
* It is optional
* User can upload documents or skip
* User must be able to continue without uploading anything
* Documents are private and not shown publicly
* Documents are only visible to:

  * the user
  * the selected agent tied to the accepted bid/listing

This section should look polished and distinct, but not scary or compliance-heavy.

Recommended layout:

* soft border
* light background
* rounded corners
* generous padding
* clear separation from the rest of the page

Suggested visual feel:

* premium SaaS

* not red/danger styling

### B. Top of section

Order:

1. Section title
2. short helper text
3. privacy note

### C. Upload fields

Each upload item should be its own clean block with:

* field label
* short helper text
* upload button/input
* file name preview after upload if selected
* replace/remove option if supported

### D. Footer actions within this section

At the bottom of the section, provide:

* **Skip for Now** button
* continue/acknowledgement action remains part of the page flow

Do not make users feel trapped inside this section.

## Behavior rules

* User can click Skip for Now and continue

### Privacy behavior

Documents must not be visible:

* to other agents

* the selected/accepted agent connected to that acknowledgement flow
