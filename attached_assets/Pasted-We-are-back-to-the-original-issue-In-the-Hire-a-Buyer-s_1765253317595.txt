2. Check the Buyer tab files for property_type logic

Open these files (all under resources/views/livewire/hire-buyer-agent/buyer-agent-auction-tabs/commission-based/):

purchasing-terms.blade.php

services.blade.php

additional-details.blade.php

broker-compensation.blade.php

buyer-info.blade.php

In each file:

Search for any use of property_type or “Income” or “Residential”.
Examples of things that will break Income:

@if ($property_type === 'Residential')
    ...content...
@endif

@if ($property_type !== 'Income')
    ...content...
@endif


Replace those with a condition that includes Income OR just remove the filter if the content should be the same:

@php
    $type = strtolower($property_type ?? '');
    $isResidentialOrIncome = in_array($type, ['residential', 'income', 'income property']);
@endphp

@if ($isResidentialOrIncome)
    {{-- existing tab content goes here, unchanged --}}
@endif


Or, if there’s no reason to hide the content by property type at all, simply remove the @if filter and always render the tab content.

At the very top of each of those files, add a temporary debug line to verify they are actually being rendered:

<div style="background:#ff0; padding:4px; font-weight:bold;">
    DEBUG BUYER TAB LOADED: {{ basename(__FILE__) }} |
    property_type = {{ $property_type ?? 'null' }}
</div>


When I refresh /hire/agent/auction/buyer and choose Income, I should see those yellow debug strips inside each tab.

If I don’t see them, the tab partial is not being hit (then check the includes again in tenant-agent-auction.blade.php).

If I see the debug strip but no main content, the @if condition inside the tab is still excluding Income.
