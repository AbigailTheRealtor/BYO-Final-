# PART 8 — STANDARDIZE VIEW BID COUNTER STATUS MESSAGES ACROSS ALL 4 ROLES

## Current inconsistency

### Tenant View Bid currently says things like:

* `Your counter offer has been submitted. Waiting for the listing owner to respond.`
* `The agent has responded to your counter offer. Review and respond in Counter Bidding History below.`

### Seller / Buyer / Landlord currently say things like:

* `You have submitted a counter offer for this bid.`
* `Test User has submitted a counter offer.`

## Need decision + implementation

We need one consistent format across all 4 roles.

## Recommended standard

Use the clearer, role-aware Tenant style as the universal standard:

### If current viewer submitted the latest counter:

* `Your counter offer has been submitted. Waiting for the other party to respond.`

More specifically:

* owner side waiting on agent:
  `Your counter offer has been submitted. Waiting for the agent to respond.`
* agent side waiting on listing owner:
  `Your counter offer has been submitted. Waiting for the listing owner to respond.`

### If the other party submitted the latest counter:

* `The [agent / listing owner] has responded to your counter offer. Review and respond in Counter Bidding History below.`

This is better than:

* `You have submitted a counter offer for this bid.`
* `Test User has submitted a counter offer.`

because it is clearer about:

* who submitted last
* who is waiting
* what to do next

## Action

Standardize the **View Bid** counter-state messages across:

* Buyer
* Seller
* Landlord
* Tenant

using this role-aware format.

* `Counter Offer Sent. You sent a counter offer on this bid. Review it in Counter Bidding History below.`

### Then seller clicks View Bid:

* `You have submitted a counter offer for this bid.`

* who submitted latest counter
* who is waiting

* show waiting message

If latest counter submitter != current viewer:

* show received/respond message
* show response actions
