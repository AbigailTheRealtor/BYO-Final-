#1013 - Landlord Offer Listing Multiselect, Placeholder, and Draft/Edit Icon Audit + Fix

Scope:
Landlord Offer Listing only:
- /offer-listing/landlord
- /offer-listing/landlord/edit/{id}

Do not touch:
- Seller
- Buyer
- Tenant
- Hire Agent
- Limited Service
- routes
- controllers
- schema
- validation
- save logic
- stored meta keys

What & Why
Landlord Create/Edit Offer Listing still has UI issues:
1. Multi-select fields close after each selection instead of allowing multiple selections at once.
2. Some “Other” placeholders still say “Describe...” instead of platform standard “Enter title (e.g., Example).”
3. On draft/edit listings, icons flash briefly and disappear. View icon works, but many other field icons do not stay visible.

Required Fixes

1. Match Seller multiselect behavior
Audit every true multi-select field on Landlord Offer Listing create/edit.

Expected:
- User can select multiple options without dropdown closing after each click.
- Use Select2 closeOnSelect: false for true multiselect fields.
- Saved values still reload correctly.
- No duplicate Select2 containers after tab switches or Livewire updates.

Apply to both:
- offer-landlord-listing.blade.php
- offer-landlord-listing-edit.blade.php
- any Landlord tab partials where Select2 multiselects initialize

2. Placeholder cleanup
Audit Landlord Offer Listing create/edit for placeholders beginning with:
- Describe...

Replace with platform format:
Enter [field title] (e.g., Example)

Rules:
- Must start with “Enter”
- Must include the field title or clear field purpose
- Must use “(e.g., Example)” with comma after e.g.
- Capitalize example words where appropriate
- Text-only change
- Do not change labels, tooltips, field names, bindings, or logic

3. Complete draft/edit icon audit and fix
Problem:
On Landlord draft/edit, icons flash and disappear. View icon remains correct, but other icons vanish.

Audit:
Compare Landlord Create vs Landlord Edit field-by-field.
Check for:
- Missing .input-cover wrappers
- Missing .has-icon
- Missing .has-select-icon for Select2 fields
- Missing/incorrect data-icon attributes
- Icons injected before Select2 replaces the field
- Livewire re-render removing injected icons
- Duplicate/fragile icon cleanup logic
- Select2 initialization order causing icons to disappear

Expected:
- Icons render immediately on draft/edit page load.
- Icons remain after tab changes.
- Icons remain after Livewire updates.
- Icons remain after Select2 initialization/reinitialization.
- View icon is not the only icon showing.
- No duplicate icons.
- Select2 text/placeholder does not overlap icons.

Likely fix direction:
- Align Landlord Edit icon initialization with Landlord Create.
- Ensure addIconsToInputs() runs after Select2 initialization.
- Ensure addIconsToInputs() runs on:
  - DOMContentLoaded / livewire:load
  - Bootstrap tab shown event
  - Livewire message.processed
  - draftLoaded event if used
- Use existing sentinel class such as .data-icon-rendered to prevent duplicates.
- Do not hardcode duplicate icons.

Relevant files to audit:
- resources/views/livewire/offer-listing/landlord/offer-landlord-listing.blade.php
- resources/views/livewire/offer-listing/landlord/offer-landlord-listing-edit.blade.php
- resources/views/livewire/offer-listing/offer-landlord-tabs/
- Any shared Select2/icon helper only if proven to be the root cause

Done looks like:
- All Landlord true multiselect fields stay open while selecting multiple values.
- All “Describe...” placeholders are changed to “Enter [field title] (e.g., Example).”
- Landlord draft/edit icons no longer flash and disappear.
- Icons match Create page field-for-field.
- Icons display across all tabs on page load, tab change, Livewire update, and draft load.
- No duplicate icons.
- Select2 placeholders/selected text do not overlap icons.
- Saved draft values still reload correctly.
- Next/Back and Save Draft/Save Edit still work.
- No console errors.
