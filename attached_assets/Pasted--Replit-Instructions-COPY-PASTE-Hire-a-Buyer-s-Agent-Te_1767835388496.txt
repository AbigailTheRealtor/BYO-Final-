TASK 1 — GLOBAL FIX: ANY NUMERICAL INPUT MUST SUPPORT COMMAS
=========================================================
Issue:
Some numeric fields do not accept comma formatting and/or clear user input (ex: "2,500" becomes invalid or wipes).

Expected (MUST APPLY TO ALL NUMERIC FIELDS IN THIS BUYER LISTING FLOW):
- Any numerical input should allow the user to type or paste comma-separated values (e.g., "1,000", "25,000").
- The UI should display numbers with commas (at least on blur; optionally while typing if cursor-safe).
- The stored value should be clean numeric text (recommended: digits + optional decimals; NO commas).
- This must apply to ALL numeric inputs used in:
  - Buyer listing create/edit
  - Agent bid create/edit
  - Counter offers (Buyer and Agent)
  - View Bid / View Counter Terms
  - Listing view display

Implementation (MUST DO):
1) Identify all numeric inputs in the Hire a Buyer’s Agent flow (fees, days, dollar amounts, percentages, deposits, etc.).
2) Replace any direct parsing that breaks on commas:
   - DO NOT do `Number(value)` or `parseFloat(value)` on a string that may contain commas.
3) Standardize ALL numeric inputs to use ONE shared input handler/component:
   - If currently `type="number"`, change to `type="text"` (or equivalent) and enforce numeric rules with sanitization.
   - Keep the SAME field keys/bindings.

Create shared helpers (ONE source of truth):
A) cleanNumericInput(raw, { allowDecimal })
- Remove commas and all non-numeric chars (allow one decimal point if allowDecimal=true)
- Examples:
  - "1,000" -> "1000"
  - "$25,000.50" -> "25000.50"
  - "10%" -> "10"

B) formatNumericDisplay(clean, { decimalsMode })
- Insert commas for thousands in display ONLY
- Preserve decimals if present
- Examples:
  - "1000" -> "1,000"
  - "25000.50" -> "25,000.50"

Behavior rules (REQUIRED):
- OnChange (typing): keep a DISPLAY string in local state; DO NOT clear the input mid-typing.
- OnBlur or OnSubmit: normalize:
  - `stored = cleanNumericInput(display)`
  - save `stored` to the existing field key
- OnLoad: display formatted:
  - `display = formatNumericDisplay(stored)`

NOTE:
- This task applies to ALL numeric fields in this Buyer listing flow, not only Buyer’s Broker Lease Fee.
