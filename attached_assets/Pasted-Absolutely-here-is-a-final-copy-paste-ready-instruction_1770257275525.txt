# Replit Instructions — Improve Financing Section Hierarchy & Term Readability

**Applies To:** Hire a Buyer’s Agent → **Actual Listing Display**
**Property Types:** All Buyer listings where Financing is shown
(Residential, Income, Commercial, Business, Vacant Land)

## PURPOSE

Improve clarity, hierarchy, and readability of the **Financing Details** section on the **actual listing display** by:

1. Making **Financing Details** a clearly separated, high-level section (matching **Purchasing Terms**), and
2. Adjusting the **term text color** so financing terms are more readable and authoritative (slightly darker gray).

This is a **presentation-only** update.

---

## 1) SAFEGUARDS (NON-NEGOTIABLE)

1. **UI-only changes**

   * Do NOT change database schema
   * Do NOT modify stored values
   * Do NOT change business logic or conditional logic

2. **No text or order changes**

   * Do NOT change wording of any financing terms
   * Do NOT change the order of fields
   * Do NOT add or remove any fields

3. **Scope control**

   * Apply only to **actual listing display**
   * Do NOT change:

     * Create Listing screen
     * Edit form inputs
     * Validation
     * Tooltips
     * PDFs (unless they already follow the same header pattern)

4. **Backward compatibility**

   * Must work for existing listings, drafts, edits, bids, and counters without re-saving

---

## 2) REQUIRED CHANGE #1 — Financing Details Must Be a True Section Header

### Issue

On the actual listing display, **Financing Details** blends into the financing terms below it and does not visually read as a major section.

### Required Fix

Style **Financing Details** exactly like **Purchasing Terms** so it is clearly a **section header**, not a field label.

### Header Styling Requirements

* Same font size as **Purchasing Terms**
* Same font weight (bold)
* Same color (black or near-black)
* Same spacing:

  * Margin above and below
* Same divider treatment (if Purchasing Terms uses a divider line)

**Important rules:**

* ❌ Do NOT add a colon after “Financing Details”
* ❌ Do NOT convert it into a field/value row
* ✅ It must visually read as a grouping header only

---

## 3) REQUIRED CHANGE #2 — Darken Financing Term Text (Not Black)

### Issue

The financing **terms/values** under Financing Details are currently too light and visually blend into the background.

### Required Fix

Darken the financing term text **slightly**, while preserving hierarchy.

### Correct Text Hierarchy

* **Section headers** (e.g., Financing Details, Purchasing Terms):

  * Black or near-black
* **Field labels** (with colons):

  * Dark gray
* **Field values**:

  * Dark gray (same tone or one step lighter than labels)

### Color Guidance (Do Not Hardcode If Tokens Exist)

* Current light gray → **too light**
* Target a “legal body text” gray:

  * Example range: `#374151` – `#4B5563`
* Do NOT use pure black for term values

### Formatting Rules

* Keep existing:

  * Font size
  * Line spacing
  * Label/value structure
* All labels must continue to end with a colon
  **Label:** Value
* Do NOT bold values unless already bolded elsewhere for emphasis
