## 3. Duplicate event handlers may be firing

```javascript

   ...
});
```

If duplicate handlers exist, one may set the full array while another overwrites with a partial value.

## 4. Livewire state may be overwriting the DOM with stale data

Compare:

* current DOM selected values
* current Livewire property value

* the same field appears twice in the DOM
* duplicate IDs/classes exist
