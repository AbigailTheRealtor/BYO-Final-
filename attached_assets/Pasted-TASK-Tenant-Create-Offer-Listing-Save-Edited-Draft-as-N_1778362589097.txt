TASK: Tenant Create Offer Listing — Save Edited Draft as New Draft, Not Overwrite Original

GOAL:
When a user opens/resumes a saved Tenant Create Offer Listing draft and makes edits, clicking Save Draft should create a new draft record instead of overwriting the original draft.

SAFEGUARDS:
- Audit first before editing.
- Tenant Create Offer Listing only.
- Full Service only. Limited Service is legacy/frozen and must not be touched.
- Do NOT touch Buyer, Seller, Landlord, Hire Agent, routes, validation, migrations, database schema, or listing submit/final publish logic.
- Do NOT delete or mutate existing saved drafts.
- Do NOT change field names, wire:model bindings, tab logic, icons, placeholders, or UI layout unless strictly required for draft save behavior.

CURRENT ISSUE:
Opening an existing Tenant draft and saving changes appears to overwrite/update that same draft.

DESIRED BEHAVIOR:
- If user starts a new Tenant listing and clicks Save Draft:
  create one draft normally.
- If user opens/resumes an existing Tenant draft and clicks Save Draft after editing:
  create a NEW draft record/version.
- Original draft must remain unchanged and still accessible.
- The newly saved draft should contain the edited values.
- Final Submit behavior must remain unchanged.

AUDIT:
1. Locate Tenant Create Offer Listing Livewire component and draft save/load methods.
2. Identify how the app tracks the currently loaded draft ID.
3. Identify whether Save Draft currently performs updateOrCreate, update, or save against an existing draft ID.
4. Confirm which method is used only for draft saving, not final submission.

IMPLEMENTATION:
Update Tenant draft-saving logic so that when a loaded/resumed draft ID exists, Save Draft does NOT update that draft. Instead:
- copy the current component state into a fresh draft record
- do not carry over the original draft primary key
- optionally store parent/source draft ID only if an existing safe column already exists
- do not add migrations or new columns unless explicitly approved

VERIFY:
1. Create a Tenant draft.
2. Reopen/resume that draft.
3. Change a visible field.
4. Click Save Draft.
5. Confirm there are now two Tenant drafts:
   - original draft with original values
   - new draft with edited values
6. Confirm final Submit still works.
7. Confirm Buyer, Seller, Landlord, and Limited Service behavior is unchanged.

DONE LOOKS LIKE:
Editing and saving a resumed Tenant Create Offer Listing draft creates a separate new draft while preserving the original draft unchanged.
