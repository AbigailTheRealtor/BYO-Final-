#941 - Remove Direct Hire Seller Agent Records from Hire a Seller's Agent Search

URL:
 /search/seller-agent-needed?page=1

What & Why:
Direct Hire / Offer Preset Seller Agent records are appearing under the public “Hire a Seller’s Agent” listing search. They do not belong there. The Hire a Seller’s Agent search page should show only normal marketplace Seller Agent Needed listings, not Direct Hire agent preset/offer-preset records.

Hard safeguards:
- Do NOT delete any records.
- Do NOT change database schema, migrations, models, validation, or saved field names.
- Do NOT break Direct Hire routes/pages.
- Do NOT affect Buyer, Landlord, or Tenant searches unless the same filtering issue is confirmed there and specifically documented first.
- Do NOT hide valid Hire a Seller’s Agent marketplace listings.
- This should be a query/filter fix only.

Audit first:
1. Identify the controller/method powering:
   /search/seller-agent-needed?page=1
2. Identify how normal Hire a Seller’s Agent listings are marked.
3. Identify how Direct Hire / Offer Preset Seller Agent records are marked.
4. Confirm the exact meta key, workflow_type, source_type, listing_type, or route flag that distinguishes them.
5. Report the exact file and query line before editing.

Fix:
- Update the Seller Agent Needed search query to exclude Direct Hire / Offer Preset Seller Agent records.
- Prefer filtering by the most reliable existing field/meta value, such as workflow_type, listing_source, listing_type, or direct_hire flag.
- If normal marketplace records use workflow_type = hire_agent, include only those.
- If Direct Hire records use a different marker, exclude that marker.
- Do not rely only on title text like “Direct Hire” unless no better data marker exists.

Done looks like:
- /search/seller-agent-needed?page=1 no longer shows Direct Hire – Seller Agent listings.
- Valid Hire a Seller’s Agent marketplace listings still appear.
- Direct Hire Seller Agent pages still open from their correct direct-hire URLs.
- No records are deleted.
- Pagination still works.
- Search filters still work.
- Git diff only touches the relevant Seller Agent Needed search query/view/controller file.
- Buyer, Landlord, and Tenant search pages are unchanged unless separately audited and approved.

Testing:
- Create or locate one normal Hire a Seller’s Agent listing and confirm it appears.
- Create or locate one Direct Hire Seller Agent / Offer Preset listing and confirm it does not appear on /search/seller-agent-needed.
- Confirm the Direct Hire listing still works from its own direct URL.
- Confirm no errors in logs.
