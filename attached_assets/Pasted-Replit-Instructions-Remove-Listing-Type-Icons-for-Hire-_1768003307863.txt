Step 2 — Identify how Seller icons are being added

Seller will almost always match one of these patterns:

Pattern A — Icons hardcoded in option labels (most common)

Example:

<option value="Bidding Period">🔨 Bidding Period</option>
<option value="Traditional">📄 Traditional</option>


✅ Fix:

Remove emoji/icon characters from the label text only

Keep values identical

Correct:

<option value="Bidding Period">Bidding Period</option>
<option value="Traditional">Traditional</option>

Pattern B — Icons injected via Select2 templateResult

Example:

templateResult: (state) => {
  if (state.id === "Bidding Period") return $('<span><i class="..."></i> Bidding Period</span>');
}


✅ Fix (Seller-only instance):

Remove the <i> / <svg> part

Return text-only label

Pattern C — Icons injected via React/JSX option renderer

Example:

{option.value === "Bidding Period" && <HammerIcon />}
<span>{option.label}</span>


✅ Fix:

Remove the icon component render

Keep <span>{option.label}</span>

Pattern D — Icons added via CSS pseudo-element

Example:

.seller-listing-type .option-bidding::before { content: "🔨"; }


✅ Fix:

Remove or override the pseudo-element rule for Seller Listing Type only
