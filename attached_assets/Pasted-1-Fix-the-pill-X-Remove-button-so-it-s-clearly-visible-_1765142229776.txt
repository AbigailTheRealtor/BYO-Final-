1) Fix the pill “X / Remove” button so it’s clearly visible

Current Issue
On the Acceptable Cities and Acceptable Counties pills, the close/remove “X” icon is present but blends into the blue pill background, so users can’t easily see where to click to remove a selection.

Required Fix

Make the “X” icon the same white color as the pill text.

Ensure the “X” is clickable and has a comfortable hit area.

HTML example (for reference)

<span class="byo-pill">
  Largo, FL
  <button type="button" class="byo-pill-remove">×</button>
</span>


CSS example

.byo-pill {
  display: inline-flex;
  align-items: center;
  gap: 0.35rem;
  padding: 0.35rem 0.75rem;
  border-radius: 999px;
  background: #007bff;   /* current blue */
  color: #ffffff;
}

.byo-pill-remove {
  border: none;
  background: transparent;
  cursor: pointer;
  font-size: 0.9rem;
  line-height: 1;
  color: #ffffff !important;  /* match pill text color so it's visible */
  padding: 0 0.25rem;          /* small click area for mobile */
}


JS behavior

Clicking .byo-pill-remove should remove that city/county from the selected list and re-render the pills.

2) Auto-populate Acceptable State from City or County

Goal
When a user selects a City or a County, the Acceptable State field should automatically populate (but only if it’s currently empty).

Backend Requirements

Ensure both cities and counties API responses include state_code:

{
  "id": 1,
  "name": "Largo",
  "state_code": "FL",
  "county_id": 12,
  "county": "Pinellas County"
}

Frontend Logic

When a City is selected:

function onCitySelected(city) {
  selectedCities.push(city);

  // Auto-set State if empty
  if (!selectedStates.length) {
    selectedStates.push(city.state_code);  // or assign to single state field
  }
}


When a County is selected:

function onCountySelected(county) {
  selectedCounties.push(county);

  if (!selectedStates.length) {
    selectedStates.push(county.state_code);
  }
}


Important Rule

Only auto-populate State when it is currently empty.

If the user manually edits the State field, do not overwrite it again based on city/county selections.

3) Auto-populate Acceptable Counties when a City is selected

Goal
When a user selects a City, automatically add its corresponding County to the “Acceptable Counties” list if it is not already there.

Backend Requirements

The cities API endpoint (e.g., /api/cities?q=Largo) should return:

[
  {
    "id": 100,
    "name": "Largo",
    "state_code": "FL",
    "county_id": 12,
    "county": "Pinellas County"
  }
]

Frontend Logic
function onCitySelected(city) {
  selectedCities.push(city);

  // Auto-set State if empty
  if (!selectedStates.length) {
    selectedStates.push(city.state_code);
  }

  // Auto-add County if not already selected
  const alreadyHasCounty = selectedCounties.some(
    (c) => c.id === city.county_id
  );

  if (!alreadyHasCounty && city.county_id) {
    selectedCounties.push({
      id: city.county_id,
      name: city.county,
      state_code: city.state_code,
    });
  }
}
