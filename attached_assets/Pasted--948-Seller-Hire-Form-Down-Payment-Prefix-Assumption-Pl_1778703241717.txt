What & Why
Four targeted follow-up fixes to the /hire/agent/auction/seller commission-based flow. These correct:
1. Missing default $ prefix behavior on the Down Payment gap field.
2. Stray $ symbol in the Assumption Fee placeholder.
3. Amenities multiselect closing after each selection.
4. Property Features multiselect closing after each selection.

Done looks like
- Down Payment to Cover the Gap shows the $ prefix box by default on a fresh form, identical to Assumption Fee.
- Switching the dropdown to % removes the prefix and appends a % suffix instead.
- Assumption Fee amount placeholder reads:
  Enter assumption fee amount (e.g., 5000)
  with no leading $.
- Amenities and Property Features stay open while selecting multiple values.
- Amenities and Property Features now behave exactly like:
  - Appliances Included
  - Special Sale Provision
  - Offered Financing/Currency
- No duplicate Select2 containers appear after tab switching or Livewire updates.
- Saved draft values reload correctly.
- Buyer/Landlord/Tenant forms remain unaffected.

Out of scope
- Routes
- Migrations / schema
- Validation rules
- Limited Service flows
- Seller Offer Listing architecture
- Any unrelated Select2 fields
