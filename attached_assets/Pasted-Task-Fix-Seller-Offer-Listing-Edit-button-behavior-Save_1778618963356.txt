Only work on the Seller Full Service Create/Edit Offer Listing flow.

Issue:

Expected behavior:
1. On Seller Offer Listing CREATE:
   - Keep existing “Save Draft” behavior unchanged.

2. On Seller Offer Listing EDIT:

   - The button should be available from every tab.

   - The user should not need to reach the final submit tab to update the listing.
   - After saving, show the same kind of success confirmation/flash already used elsewhere, such as “Listing updated successfully.”
   - Do not create a duplicate listing.
   - Do not create or overwrite a draft record.
   - Stay on the edit page unless existing app behavior already redirects after update.
