This is a CRITICAL persistence fix for Agent Presentation & Promotional Materials across ALL listing types / roles.

Problem:
When a bid is edited, uploaded marketing materials can disappear from the bid.
Also, if materials were pre-saved in the user profile and used to pre-load the bid, they may disappear after later edits unless manually re-uploaded right before submit.

This must be fixed so uploaded materials persist correctly.

---

## GOAL

For ALL roles/listing types, make sure Agent Presentation & Promotional Materials persist through:

* initial bid submit
* bid edit/update
* repeated edits
* pre-saved profile assets used to pre-load a bid
* manually uploaded files added during bid creation
* manually uploaded files added during later edits

If a file is already attached to the bid OR referenced from the user profile for that bid, it must remain on the bid unless the user intentionally removes/replaces it.

---

## APPLIES TO

All listing types / roles:

* Tenant
* Buyer
* Seller
* Landlord

And all relevant bid flows:

* create bid
* edit bid
* counter bid if the same materials section is reused there
* preloaded profile assets

This applies to all Agent Presentation & Promotional Materials items, including but not limited to:

* Business Card
* uploaded marketing materials
* uploaded presentation materials
* uploaded flyers
* uploaded digital ads
* uploaded listing presentation files
* any other file-based promotional material section currently supported

## PHASE 2 — FIX PERSISTENCE RULES

Implement these rules for ALL roles:

RULE A — Existing bid files must persist
If a bid already has attached Business Card or Marketing/Promotional Materials,
and user edits the bid without removing/replacing them,
those files must remain attached after save.

RULE B — Preloaded profile assets must persist once used in a bid
If a pre-saved profile asset is used to pre-load a bid,
that bid must continue to show those assets after edit/save unless the user intentionally removes/replaces them.

RULE C — Empty edit submissions must NOT wipe files
If no new files are uploaded during edit,
the save/update logic must preserve the existing stored file metadata.

RULE D — New uploads should merge/replace intentionally only
If user uploads new files:

* apply the intended replace/append behavior already supported by the UX
* but do not accidentally wipe unrelated existing file groups

RULE E — Same rule across all roles
Tenant, Buyer, Seller, Landlord must all preserve files the same way.
