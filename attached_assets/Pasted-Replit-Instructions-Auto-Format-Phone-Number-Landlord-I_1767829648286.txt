Replit Instructions — Auto-Format Phone Number (Landlord Information Tab)
Goal

When a user types a phone number, the input should automatically format as:
555-555-5555
…and should work whether they type:

5555555555

555-555-5555

(555) 555-5555

555 555 5555

The system should store the phone consistently (recommended: digits only), but display formatted.

SAFEGUARDS (MUST FOLLOW)

DO NOT:

delete/rename any database columns, migrations, models, or API fields

change any labels, tooltips, placeholder text, or UI layout

change validation rules for other fields

break existing edit/listing display logic

MUST:

apply formatting only to the phone input field

keep existing save flow intact

keep backwards compatibility with any previously saved phone values

Expected Behavior (UI)
While typing

User types 5 → shows 5

Types 555 → shows 555

Types 5555 → shows 555-5

Types 5555557 → shows 555-555-7

Types 5555555555 → shows 555-555-5555

Pasting

Pasting (555) 555-5555 should become 555-555-5555

Deleting

Backspace should behave normally (no “cursor jumping” or getting stuck)

Implementation (Frontend)
Step 1 — Add a formatter function (no external libs required)

Create a helper function in the same component file or a shared utils file:

Formatting rules:

Strip all non-digits

Limit to 10 digits

Insert dashes after digit 3 and 6

Return formatted string

Pseudo-logic:

digits = value.replace(/\D/g, '').slice(0, 10)

If digits.length <= 3 → return digits

If digits.length <= 6 → return digits.slice(0,3) + '-' + digits.slice(3)

Else → return digits.slice(0,3) + '-' + digits.slice(3,6) + '-' + digits.slice(6)

Step 2 — Update the Phone Number input onChange

Where the phone field is defined in Landlord Information Tab:

On input change:

take user input

format it

set the displayed value to formatted result

✅ This ensures the dash appears automatically.

Saving Rules (Data Integrity)
Recommended storage (best practice)

Store digits only in the database (example: 5555555555)

Display as formatted in UI (555-555-5555)

On submit:

Convert formatted string to digits only (replace(/\D/g,''))

Save that value to the existing phone column (same key)

✅ This prevents inconsistent formats in DB and keeps search/validation easier.

Backwards compatibility

If older records already include dashes or parentheses:

On load, format the value before displaying:

display = formatPhone(existingValue)

Validation Rules (Must Add/Confirm)

Phone number is valid if:

it contains exactly 10 digits (US format)

If optional field:

allow blank

If required:

show existing error style/message (do not change copy)

Apply Everywhere This Field Appears

Make sure the same component/logic is used for:

Create Hire a Landlord’s Agent listing

Edit listing (if Landlord can edit)

Any place Landlord phone appears as editable (if any)

Any view screens can simply format for display

✅ Best safeguard: put the phone input into a reusable component like PhoneInput and reuse it everywhere.

QA Checklist (Must Pass)

Typing 5555555555 results in 555-555-5555

Pasting (555) 555-5555 results in 555-555-5555

Input blocks >10 digits

Backspace works normally

Saved value is consistent (digits only recommended)

Existing saved phones still display correctly
