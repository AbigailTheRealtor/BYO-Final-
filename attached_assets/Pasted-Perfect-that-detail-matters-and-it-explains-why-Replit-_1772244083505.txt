Perfect, that detail matters — and it explains why Replit keeps “missing” the Income fields.

If those 3 fields only appear when you select **Income Property** under **Acceptable Property Type**, then they’re inside a conditional block like:

`@if(in_array('Income Property', $acceptable_property_type)) ...`

That creates two common failure modes in Livewire:

1. **They are never saved** because the edit component doesn’t have properties/saveMeta for them (so they vanish after you click Edit or load Draft)
2. **They are saved, but immediately cleared** when Livewire re-hydrates and the conditional block isn’t active at the exact moment the component loads, so defaults overwrite the meta

So we need to make Replit do two precise fixes:

* Ensure these fields are **always loaded into properties BEFORE any updated* handlers run**
* Ensure they are **saved even if the conditional UI block isn’t currently visible**, as long as `acceptable_property_type` includes Income Property

If you don’t have asArrayMeta helper, implement local safe decode:

* If string → wrap into [string]
