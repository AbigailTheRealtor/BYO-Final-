Implementation requirement

Do not rely on events or frontend navigation. The submit method must return the redirect response.

Buyer submit → Buyer detail route

PART 3 — Ensure Submit Fires Exactly Once (prevents delays & double notifications)

Even if redirect exists, it can fail if submit is firing multiple times or only firing after retries.

REQUIRED standard wiring (use ONE pattern consistently)

Preferred:

No wire:click="submit" on submit button

Also add locking:

Submit button: wire:loading.attr="disabled" + wire:target="submit"

✅ Prevents multi-click and stops “must click multiple times” issues.
