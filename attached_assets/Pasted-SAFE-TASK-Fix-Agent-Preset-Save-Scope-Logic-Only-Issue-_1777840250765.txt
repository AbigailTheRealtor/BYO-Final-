SAFE TASK — Fix Agent Preset Save Scope Logic Only

Issue:
The Agent Preset scope dropdown is not respecting the selected scope.

Current bad behavior:
- “This preset only” saves changes across every role/property type.
- “All Buyer presets” saves across all roles.
- Same issue happens for Tenant, Seller, Landlord.
- The last saved profile content appears when sharing any role profile.

Correct behavior:
1. “This preset only”
   - Save public Agent Profile content ONLY to the current preset.
   - Example: Buyer + Residential only updates Buyer Residential.
   - Must NOT update Buyer Commercial, Seller Residential, Tenant, Landlord, etc.

2. “All [Current Role] presets”
   - Save public Agent Profile content ONLY to presets matching the current role.
   - Example: All Buyer presets updates:
     - Buyer Residential
     - Buyer Income
     - Buyer Commercial
     - Buyer Business
     - Buyer Vacant Land
   - Must NOT update Tenant, Seller, or Landlord presets.

3. “All roles and property types”
   - Save public Agent Profile content across all role/property-type presets.

Important:
The scope applies ONLY to public Agent Profile sections.

Always keep these preset-specific only, regardless of selected scope:
- Offered Services
- Other Services
- Broker Compensation
- Agency Agreement Terms
- Referral Fee / Cooperation Terms
- Any role/property-specific compensation fields

Do NOT:
- Change database schema unless absolutely required and approved first
- Change dashboard UI
- Change Hire This Agent flow except reading correct preset data if needed
- Change service catalogs
- Change broker compensation wording
- Change listing/bid/counter logic
- Make services or compensation global
- Save one role’s services/compensation into another role
- Touch non-agent preset areas

Files to inspect first:
- routes related to /agent/presets
- AgentPresetController
- AgentPreset model
- preset save/update request handling
- preset sharing/profile preview logic
- Blade/Livewire component for the dropdown shown in screenshot

Audit first:
1. Find the form field name/value for the scope dropdown.
2. Find where scope is interpreted during save.
3. Identify why updates are being applied to every role/property type.
4. Identify which fields are public Agent Profile fields vs preset-specific fields.

Implementation requirements:

A. Define allowed scope values clearly:
- this_preset_only
- all_current_role_presets
- all_roles_and_property_types

If the current dropdown uses different values, map them safely without breaking the UI.

B. Determine current preset context from the preset being edited:
- current role/user type
- current property type

Do NOT trust hidden fields alone if the preset record already contains role/property_type.

C. Public Agent Profile fields may be copied based on scope.
These may include fields like:
- agent overview
- qualifications
- experience
- bio
- marketing/presentation/promotional materials
- contact/profile display fields
- public profile sections

D. Preset-specific fields must ONLY update the current preset:
- services
- other_services
- broker compensation
- agency agreement terms
- referral fee/cooperation terms
- compensation fields
- agreement timeframe
- termination fee
- retainer fee
- protection period
- any role/property-specific terms

E. Save behavior:
If scope = this_preset_only:
- Update only the current preset ID.

If scope = all_current_role_presets:
- Update only presets where user_id/agent_id matches current agent AND role/user_type matches current preset role.
- Do not filter only by property type.
- Do not update other roles.

If scope = all_roles_and_property_types:
- Update all presets for the same agent/user only.
- Only copy public profile fields.
- Do not copy services or compensation fields.

F. Shared profile / Hire This Agent preview:
- Ensure the shared link loads the correct preset by ID/slug/role/property type.
- It should not pull “latest saved” global profile data if a specific preset was shared.
- Buyer Residential shared profile must show Buyer Residential preset-specific services/compensation.
- Seller, Tenant, and Landlord shared profiles must show their own preset data.

5. Test shared profile links:

   - Tenant Residential link shows Tenant Residential preset.
   - Seller Residential link shows Seller Residential preset.
   - Landlord Residential link shows Landlord Residential preset.
   - No link should show the last-saved preset from another role.
