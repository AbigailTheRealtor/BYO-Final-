Replit Instructions — Dashboard Bid Visibility, Actions & Notifications

Applies to: Hire a Tenant’s Agent (Residential & Commercial)
Profiles involved:

Tenant User Profile: tenant@exp.com

Agent Profile: testagent2@bidyouroffer.com

CONTEXT (IMPORTANT — DO NOT SKIP)

The platform has two separate dashboards that must remain distinct:

Tenant Dashboard
Example profile: tenant@exp.com
This is the User/Tenant profile that creates Hire a Tenant Agent listings and accepts, counters, or rejects Agent bids.

Agent Dashboard
Example profile: testagent2@bidyouroffer.com
This is the Agent profile that submits bids and must be able to see bid status updates, counters, acceptances, and the Accepted Bid Summary.

All logic below must respect these two different profile types and dashboards.

ISSUE SUMMARY

Currently:

A notification appears briefly when a bid is accepted, but does not persist

The Agent cannot find:

which bid was accepted

the Accepted Bid Summary

the current bid status

“Recent Notices” shows No notifications found

Bid actions (Accept / Counter / Reject) appear to be tied to the listing, not the individual bid

Counter actions can incorrectly set bids to “Inactive” with no visibility

Agents must manually click their own name or return to listings to infer what happened

REQUIRED BEHAVIOR — DASHBOARD VISIBILITY
1. Tenant Dashboard (tenant@exp.com)

When logged in as the Tenant/User:

Every Agent bid must be visible inside the Tenant dashboard

Each bid must show:

Agent name (or Agent # if anonymized)

Bid status: Pending / Countered / Accepted / Rejected

Buttons to Accept / Counter / Reject THAT specific bid

Actions taken here must apply only to the selected bid_id

2. Agent Dashboard (testagent2@bidyouroffer.com)

When logged in as the Agent:

A. Recent Notices (Dashboard Home)

The following must appear under Recent Notices and remain clickable until dismissed:

“Your bid was accepted” → links to Accepted Bid Summary

“Your bid was countered” → links to counter details

“Your bid was rejected”

Any new counter requiring Agent action

Each notice must include:

Listing ID

Bid reference (or Agent Bid Summary link)

B. My Bids Section

Under My Bids, each bid row must clearly show:

Listing ID

Bid Status:

Pending

Countered

Accepted

Rejected

A clickable link to:

View full bid

View Accepted Bid Summary (if accepted)

Once a bid is Accepted, the Agent must be able to access:

The Accepted Bid Summary document

Signature status (Tenant / Agent)

PDF download (after both parties sign)

PER-BID ACTION LOGIC (CRITICAL)
3. Accept / Counter / Reject MUST BE BID-SPECIFIC

Do NOT tie actions to the listing as a whole.

Required logic:

Each bid has its own bid_id

Accept / Counter / Reject actions update only that bid

Other bids on the same listing remain unaffected

Example:

If a listing has 2 Agent bids, the Tenant must be able to accept one and reject or counter the other independently.

4. Counter Flow Fix

Current issue:

Sending a counter can set the bid to “Inactive” with no visibility

Required fix:

Replace “Inactive” with explicit Countered status

When a counter is sent:

Bid status becomes Countered

Both Tenant and Agent can view the counter terms

Agent dashboard shows a notification

Agent can respond directly from their dashboard

AGENT EDIT RULES

Agent may edit their bid only while status = Pending

Once status becomes:

Countered

Accepted

Rejected
→ the bid becomes locked, except through the counter-response workflow

ACCEPTED BID SUMMARY ACCESS

When a bid is Accepted:

Tenant is immediately redirected to the Accepted Bid Summary page

Agent receives a persistent notification

Both dashboards must link to the same Accepted Bid Summary

Admin must also receive a copy for records

SAFEGUARDS (MUST FOLLOW — VERY IMPORTANT)

DO NOT:

merge Tenant and Agent dashboards

rename or delete existing database columns

change listing creation flows or UI

alter Buyer, Seller, or Landlord flows

remove listing-page Accept / Counter / Reject logic

DO:

scope all actions by bid_id

persist notifications until dismissed

ensure dashboard actions behave the same as listing-page actions

ensure Agent and Tenant dashboards show real-time bid status updates
