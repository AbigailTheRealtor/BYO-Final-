# WHAT’S BROKEN

When user clicks the listing photo field and selects a photo from their computer:

* the file picker/folder opens
* then it dings/freezes / does not continue
* user cannot finish selecting the file or submit

This is usually caused by one or more of these:

* duplicate click handlers attached to the file input or its label/button
* a hidden file input being triggered multiple times
* Livewire rerendering the file input while the OS picker is open
* `wire:model` file input sitting inside a reactive block that keeps rerendering
* JS preventing default or re-triggering `.click()` on the file input
* nested labels/buttons over the input causing event conflicts

## STEP 2 — Remove duplicate JS click triggers

Inspect the active blade and any inline JS for the photo field.

### Remove/disable these patterns if present

* a button that calls `input.click()` more than once
* both `<label for="photo">` **and** JS `click()` on the same element
* event listeners that reopen the input on `change`
* handlers that call `preventDefault()` unnecessarily on the file upload trigger
* repeated event binding inside `message.processed` without guard

### Goal

## STEP 3 — Prevent Livewire from rerendering the file input during selection

If the file input sits inside a highly reactive section, Livewire may rerender it while the file dialog is open.

### Fix

* Wrap the surrounding uploader UI in a stable container
* Do **not** keep rebinding/replacing the actual file input during every render
* If needed, isolate the input using `wire:key` so it stays stable
* Do not put the input inside a repeatedly rebuilt Select2-style JS init block

### Important

Do **not** use `wire:ignore` on the actual file input if Livewire needs to receive the upload.
Instead, stabilize the parent container and stop JS from retriggering it.
