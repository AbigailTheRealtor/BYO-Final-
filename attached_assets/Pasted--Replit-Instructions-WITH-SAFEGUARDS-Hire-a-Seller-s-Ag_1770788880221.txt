# FIX 1 — Seller Financing does NOT auto-populate on Edit

## Required behavior

When a user edits a listing (or reopens a draft):

* Seller Financing selection/toggle must reload
* Any Seller Financing sub-fields must reload
* All dependent inputs must be visible if they were previously triggered

### Implementation requirements

#### A) Canonical keys (must match create + save + load + edit)

Use one key set for Seller Financing:

* `seller_financing` (boolean or selection)
* plus the exact sub-fields your form collects (examples)

  * `seller_financing_down_payment`
  * `seller_financing_interest_rate`
  * `seller_financing_term_years`
  * `seller_financing_balloon`
  * `seller_financing_notes`
    (Use your actual existing keys; do not invent new ones unless your DB is missing them.)

✅ **Safeguard:** No separate keys for edit vs create.

#### B) Edit hydration must set BOTH values and UI show/hide booleans

In Seller Edit component `mount()` / `loadDraft()`:

1. Set `$this->seller_financing` from saved value
2. Set each `seller_financing_*` subfield from saved values
3. Set a boolean like `showSellerFinancingFields = true` when seller financing is enabled/selected

🚫 If you only set the values but not the show boolean, the UI will hide the section and the user thinks it didn’t save.

#### C) Edit save must persist the same keys

In the edit update method:

* save `seller_financing` and all `seller_financing_*` subfields
* merge-save so other-tab saves don’t clear them

# FIX 2 — “Provide digital photo enhancements” sub-options not showing on Edit (and Other text missing)

## Required behavior

If the user selected **Provide digital photo enhancements**, the Edit screen must show:

### Select enhancement types (options MUST be available and selected items must be checked):

* Basic edits (brightness, contrast, cropping)
* Twilight conversion (convert daytime photo to sunset look)
* Object removal (e.g., cars, trash cans, furniture, etc.)
* Virtual twilight photography
* Color correction or sky replacement
* Other

If **Other** is selected:

* show the “Other” text input
* populate it with the saved custom text

---

## Implementation requirements

### A) Store three values (parent + types + other text)

Persist:

1. `digital_photo_enhancements_selected` (boolean parent)
2. `digital_photo_enhancement_types` (array/JSON)
3. `digital_photo_enhancement_other_text` (string)

✅ **Safeguard:** enhancement types must be stored as JSON array and hydrated as array.

---

### B) Edit hydration must load values AND force sub-question UI to appear

In Seller Edit component `mount()` / `loadDraft()`:

1. `$this->digital_photo_enhancements_selected = (bool) saved_parent;`
2. `$this->digital_photo_enhancement_types = saved_array ?? [];`
3. `$this->digital_photo_enhancement_other_text = saved_text ?? '';`

Then set UI toggles:

* if parent selected → `showDigitalEnhancementTypes = true`
* if “Other” in types → `showDigitalEnhancementOther = true`

🚫 Do not rely on `updatedDigitalPhotoEnhancementTypes()` only.

---

### C) Edit Blade must reuse the same sub-question partial as Create

The Edit form must render the exact same checkbox list and “Other” textbox component/partial used on Create.

✅ **Safeguard:** Do not use a simplified “selected services summary” renderer on Edit.

---

### D) Save logic (draft + edit) must persist types and other text

On save:

* write parent boolean
* write selected types array
* if Other selected, write other text
* if Other not selected, clear other text (recommended)

✅ Merge-save safeguard applies.
