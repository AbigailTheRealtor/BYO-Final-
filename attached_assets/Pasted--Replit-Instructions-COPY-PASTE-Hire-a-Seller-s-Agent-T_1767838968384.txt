=========================================================
TASK 1 — TAB 6: BROKER COMPENSATION TOGGLE BEHAVIOR (CLEAR PRIOR INPUTS)
=========================================================
Requirement:
When entering compensation, if the user switches between:
- Flat Fee ↔ Percentage, OR
- switches between different compensation field types/options, OR
- switches property types (Residential ↔ Commercial),
THEN the previous input values for the now-hidden selection MUST be cleared.

Why:
Prevents conflicting/inaccurate listings where multiple fee types appear at once.

Implementation:
1) Identify the toggle(s) that control the commission type (e.g., $ vs %, flat vs %, dropdown option changes).
2) On change of the selection, immediately:
   - CLEAR values for inputs that are no longer applicable/visible
   - Keep ONLY the active selection’s values
3) Apply the same clearing logic when property type changes (Residential/Income/Vacant Land ↔ Commercial/Business).
4) Ensure this clearing logic runs in:
   - Listing Create/Edit
   - Agent Bid Create/Edit
   - Counter Offer screens (both parties)

Safeguard:
- Clear only the related fields for the toggled section (do not wipe unrelated listing data).
- Do not rename keys; set them to null/empty string using existing keys.

=========================================================
TASK 2 — NUMERIC INPUTS: ALLOW COMMAS + STOP “AUTO-DELETING” VALUES (SELLER FLOW)
=========================================================
Issue:
Some numeric inputs do not accept commas and/or clear values when commas are typed/pasted.

Expected:
- Any numerical input in Seller Broker Compensation should allow commas (e.g., “2,500”, “25,000.50”).
- Store clean numeric value (digits + optional decimals; no commas).
- Display formatted with commas (at least on blur/load).
- Must work in Create/Edit, Bids, Counters, and View screens.

Implementation (Single source of truth):
1) Standardize Seller compensation numeric inputs to use ONE shared helper/component:
   - `cleanNumericInput(raw, allowDecimal)`
   - `formatNumericDisplay(clean)`
2) DO NOT parse raw strings with commas using Number() directly.
   - Always clean commas before parsing/validating.
3) Prefer `type="text"` with sanitization over `type="number"` if `type="number"` is causing wipe/reset behavior.
4) OnChange: keep user-typed display string; DO NOT clear mid-typing.
5) OnBlur/Submit: normalize and save clean value under the same existing field key.
6) OnLoad/View: format the stored value for display.
