ssue: “Hire a Buyer’s Agent → Income Property” – Tabs 3–7 blank

Residential, Commercial, Business, and Vacant Land all work.

For Income, Tabs 3–7 are blank and the console shows:

Uncaught (in promise) TypeError: can't access property "$wire",
this.components.componentsById[componentId] is undefined
    initializeFullService buyer:4224


The Livewire root component is mounted correctly for Income (<div class="container pt-5 pb-5" wire:id="...">), so the bug is inside the Buyer Livewire component, probably in the view that renders Tabs 3–7 and the JS that initializes Services.

1) Check the Blade view for property-type conditions

In the Blade / Livewire view that powers Hire a Buyer’s Agent (the one that holds all the tabs):

Find the part that renders Purchasing Terms, Broker Compensation, Services, Additional Details, Buyer Info.

There is probably a conditional like:

@if ($property_type === 'residential')
// ...
@elseif ($property_type === 'commercial')
// ...
@elseif ($property_type === 'business')
// ...
@elseif ($property_type === 'vacant_land')
// ...
@endif


or a switch ($property_type).

👉 Add the missing branch for 'income' for each tab, especially Services, and make sure it uses the correct partial / Livewire child component.

Example (pseudo-code):

@elseif ($property_type === 'income')
    @include('buyer.tabs.income-purchasing-terms')
    @include('buyer.tabs.income-compensation')
    @include('buyer.tabs.income-services')
    @include('buyer.tabs.income-additional-details')
    @include('buyer.tabs.income-buyer-info')
@endif


Right now it looks like there is no Income-specific include for Services (and possibly the other tabs), so nothing is rendered and the JS initializer can’t find its child component.

2) Make sure the Services component for Income exists & matches the initializer

For the Services tab:

Confirm that an Income version exists and is referenced correctly, for example:

@livewire('buyer.full-service-services', [
    'propertyType' => 'income',
    // other props...
], key('buyer-full-service-services-income'))


or

@include('buyer.services.income-full-service')


Whatever pattern you used for Residential/Commercial/Business/Vacant Land, duplicate it for Income and keep the naming consistent.

The important part is:

The child Livewire component for Income Services must actually be mounted so Livewire registers it in componentsById.

The componentId passed into initializeFullService(componentId) for Income must correspond to that child component.

3) Guard the JS initializer so it doesn’t crash the whole component

In the JS file where initializeFullService is defined (the one referenced as buyer:4224):

function initializeFullService(componentId) {
    const componentStore = this.components && this.components.componentsById
        ? this.components.componentsById
        : null;

    if (!componentStore || !componentStore[componentId]) {
        console.warn(
            'FullService initialization skipped – Livewire component not found for',
            componentId
        );
        return;
    }

    const component = componentStore[componentId];
    const $wire = component.$wire;

    // existing initialization logic...
}


This prevents the app from throwing a fatal error if, for some reason, the component isn’t present for a given property type.

4) Notification permission warning (secondary issue)

There is also:

The Notification permission may only be requested from inside a short
running user-generated event handler.


Somewhere in the Buyer JS, Notification.requestPermission() is being called on page load or an auto-init. That needs to be moved into a click handler instead, e.g.:

document
  .getElementById('enable-notifications-button')
  .addEventListener('click', () => {
      if (!('Notification' in window)) return;

      Notification.requestPermission().then(result => {
          console.log('Notification permission:', result);
      });
  });


This is a warning, not the cause of the blank tabs, but it should still be cleaned up.
