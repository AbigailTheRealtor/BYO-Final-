Replit Instructions — Make Section Titles PERFECTLY Centered + Reduce Header Bar Height

Applies to: Hire a Tenant’s Agent listing display (Residential + Commercial)
Primary file: resources/views/hire_tenant_agent/view.blade.php
(If shared across other listing displays, apply the same class/CSS there too)

1) Add a dedicated wrapper + class around EVERY section title (do not rely on generic padding)

Show Replit this exact pattern and have them apply it to each section header:

<div class="listing-section-header">
  <div class="listing-section-title">Listing Details:</div>
</div>


Do this for:

Listing Details

Property Preferences

Leasing Terms

Pre-Screening

Services

Additional Details

Broker Compensation & Agency Agreement Terms

Tenant’s Info

✅ Important: The title must be inside .listing-section-title (not an h1/h2/h3/h4) to avoid default heading margins interfering.

2) Add THIS CSS (this is what actually fixes the “title sits too low” problem)

Add to your main CSS file (or a <style> block scoped to this view):

/* SECTION HEADER BAR */
.listing-section-header{
  display: flex;
  align-items: center;          /* true vertical centering */
  justify-content: flex-start;  /* keep left aligned */
  height: 56px;                 /* set a consistent bar height */
  padding: 0 18px;              /* no top/bottom padding (prevents drift) */
  border-bottom: 1px solid #ddd;
}

/* TITLE TEXT */
.listing-section-title{
  margin: 0;
  padding: 0;
  line-height: 1;               /* prevents baseline drift */
  display: block;
}


✅ This forces the header bar to a clean height and centers the text exactly.

3) Remove/undo the previous “min-height / padding-top / padding-bottom” changes on old header selectors

Replit previously added things like:

min-height: 2.5rem

padding-top/bottom: 0.75rem

display:flex on generic containers like .card-header or h4

Tell them:

Undo/Remove any prior header rules that target:

.card-header (globally)

h2/h3/h4 inside headers

any “symmetrical padding” used to fake centering

Because those are what’s creating the oversized header bar + drift.

4) Clear cache + hard refresh (required)

Run:

php artisan view:clear
php artisan cache:clear


Then hard refresh:

Windows: Ctrl + Shift + R

Mac: Cmd + Shift + R

Result you should see

Header bar is shorter

“Listing Details:” is perfectly centered vertically

No more “3/4 down” look

Same result for every section header
