- If the project already has referral-like routes or middleware, inspect first and report before adding duplicates.

PHASE 4 — REFERRAL CAPTURE ONLY

1. Inspect routes and choose the correct public web routes file used by this project.

2. Add a dedicated public GET route:
- /invite/{code}
Named route:
- referral.capture

3. Create a new controller:
- app/Http/Controllers/ReferralController.php

4. Add method:
- capture(Request $request, string $code)

Required behavior:
A. Find an active referral link in agent_referral_links by code.
B. If the code is invalid or inactive:
   - redirect safely to the main landing page or homepage
   - optionally flash a neutral message
   - do not crash

C. If the code is valid:
   - check whether session already contains referral attribution:
     - referral.agent_id
     - referral.code
     - referral.captured_at
     - referral.visit_id

D. First-click-wins rule:
   - if session already has a referral.agent_id, do not overwrite it
   - still redirect safely
   - do not create a duplicate referral_visits record for the same request if attribution is already locked in session

E. If session has no referral yet:
   - store in session:
     - referral.agent_id = referral link’s agent_id
     - referral.code = referral link code
     - referral.captured_at = now()
   - create a referral_visits row with:
     - agent_id
     - referral_code
     - session_id (if available from session)
     - ip_address
     - user_agent
     - landing_url (current full URL is fine)
   - after creating the row, store referral.visit_id in session
   - increment click_count on agent_referral_links by 1

5. Redirect target:
- redirect to the main public landing page used by this project
- if there is ambiguity, inspect existing public entry points and choose the safest neutral landing page
- report which route/path was chosen

IMPLEMENTATION RULES:
- No middleware unless clearly necessary.
- Do not add cookie persistence yet unless already idiomatic in this codebase.
- Keep the controller small and isolated.
- Match project coding style.
- Use null-safe/session-safe handling.
- Do not expose internal database errors to users.
