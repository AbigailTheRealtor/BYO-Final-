A) FIX: Accordion flashes open then instantly closes
Likely cause (common)

The accordion toggle is being triggered twice due to:

nested click handlers (header + icon),

a link/button inside the header causing navigation,

collapse IDs not unique,

or a re-render immediately collapsing (Livewire / Alpine / JS conflict).

Required Fix
A1) Ensure unique collapse IDs per bid

In the bid loop, set IDs using the bid id:

Toggle target: #bidCollapse-{{ $bid->id }}

Collapse container id: bidCollapse-{{ $bid->id }}

Do not use $loop->index alone if the DOM rerenders.

A2) Prevent double-toggle event bubbling

If the arrow/icon is inside a clickable header, stop propagation on the icon:

Example:

<button type="button"
        onclick="event.stopPropagation();"
        data-bs-toggle="collapse"
        data-bs-target="#bidCollapse-{{ $bid->id }}">
  ▲
</button>


If using Livewire:

Make sure the arrow button is type="button"

Add wire:click.prevent and do NOT also have a data-bs-toggle on the parent.

Only one mechanism controls collapse:

Either Bootstrap collapse OR Livewire/Alpine — not both.

✅ Requirement: Pick one and remove the other for this accordion only.

A3) If Livewire is re-rendering and collapsing immediately

Wrap the collapse content in wire:ignore.self so Livewire updates don’t snap-close it:

<div id="bidCollapse-{{ $bid->id }}"
     class="collapse"
     wire:ignore.self>
   ...
</div>


This is a very common fix when a Livewire update happens right after click.

✅ Acceptance: Clicking the arrow keeps the bid expanded until user collapses it.
