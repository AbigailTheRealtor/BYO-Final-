Required behavior:

1. Traditional Listing behavior
If listing type is NOT Bidding Period:

  - Landlord: Desired Lease Price

If listing type IS Bidding Period:
Show a new section at the top of the Sale Terms / Leasing Terms tab:

For Seller:
Section title:
Bidding Period Pricing

Fields:
- Desired Sale Price
- Starting Price / Opening Bid
- Reserve Price
- Buy Now Price

For Landlord:
Section title:
Bidding Period Lease Pricing

Fields:
- Desired Lease Price
- Starting Rent / Opening Offer
- Reserve Rent
- Lease Now Price

3. Default logic
When user enters Desired Sale Price:
- Auto-fill Buy Now Price with the same amount if Buy Now Price is empty.

When user enters Desired Lease Price:
- Auto-fill Lease Now Price with the same amount if Lease Now Price is empty.

Do not overwrite Buy Now Price / Lease Now Price after the user manually edits it.

4. Reserve field logic
- Reserve Price and Reserve Rent should be optional.
- Do not make reserve required.
- Do not expose hidden/public logic yet.
- Only collect/store the value for now.

Landlord:
- Place Bidding Period Lease Pricing section at the top of Leasing Terms tab.

7. Tooltips
Add tooltip icons for every new field.

Tooltip text:
- Desired Sale Price:
  The Seller’s preferred sale price for the property.
- Starting Price / Opening Bid:
  The price where bidding begins for a Bidding Period Listing.
- Reserve Price:
  The minimum sale price the Seller may be willing to accept. This field is optional.
- Buy Now Price:
  A price that may allow a Buyer to immediately secure the property if the Seller accepts those terms.
- Desired Lease Price:
  The Landlord’s preferred lease price or rental amount.
- Starting Rent / Opening Offer:
  The rent amount where bidding begins for a Bidding Period Listing.
- Reserve Rent:
  The minimum rent amount the Landlord may be willing to accept. This field is optional.
- Lease Now Price:
  A rent amount that may allow a Tenant to immediately secure the property if the Landlord accepts those terms.
