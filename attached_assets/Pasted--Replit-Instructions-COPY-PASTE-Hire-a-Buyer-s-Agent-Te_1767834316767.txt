========================================
TASK 1 — FIX “Buyer’s Broker Lease Fee” FLAT FEE INPUT DELETING NUMBERS
========================================
Issue:
- In Buyer’s Broker Lease Fee, when user selects Flat Fee, typed values are being cleared/deleted.
- Comma formatting is not consistently supported (example: "2,500" causes deletion or fails).

Expected:
- Flat Fee input must NOT delete/clear user-entered numbers.
- Input must accept comma-formatted numbers (e.g., "2,500", "25,000", "2500.00").
- System should SAVE a clean numeric value (recommended: digits + optional decimals, no commas).
- System should DISPLAY formatted with commas.

Implementation (MUST DO):
1) Find the Buyer’s Broker Lease Fee → Flat Fee input handler.
2) Remove any logic that does:
   - Number(value) or parseFloat(value) on a string containing commas WITHOUT cleaning commas first
   - if NaN then set value to "" (this is what causes “auto delete”)
3) Ensure the input is treated as TEXT, not a numeric-only browser input that strips commas:
   - Use `type="text"` (or your framework equivalent) and sanitize/validate manually.

4) Add these shared helper functions (put in a shared util or the Buyer comp terms component):

   A) cleanMoneyInput(raw)
   - Removes all characters except digits and a single decimal point.
   - Removes commas safely.
   - Limits to a reasonable length (optional).
   Example behavior:
   - "2,500" -> "2500"
   - "$2,500.50" -> "2500.50"

   B) formatMoneyDisplay(clean)
   - Adds commas for thousands in display ONLY.
   - Example:
   - "2500" -> "2,500"
   - "2500.50" -> "2,500.50"

5) Input behavior:
   - OnChange (typing): DO NOT clear the field if parsing fails mid-typing.
     Keep a DISPLAY string in state (what user sees).
   - OnBlur OR OnSubmit: normalize:
     - `stored = cleanMoneyInput(display)`
     - validate `stored` is numeric if required
     - save `stored` under the SAME existing key used today

6) OnLoad (edit/view/bids/counters):
   - If stored value is "2500" or "2500.50", display as formatted:
     - `display = formatMoneyDisplay(stored)`
   - If old records contain commas or symbols, run clean→format before displaying.

IMPORTANT:
- Do NOT change the saved key/field name for Buyer’s Broker Lease Fee.
- Do NOT change any other fee calculation logic—only fix input parsing + persistence.
