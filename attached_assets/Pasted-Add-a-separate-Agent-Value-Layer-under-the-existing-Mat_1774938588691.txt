Add a separate **Agent Value Layer** under the existing Match Score on the active Tenant bid/detail view.

### Goal

Keep Match Score exactly as-is, but add a clear visual indicator when the agent has included services beyond the Tenant’s original request.

### Required behavior

Under the Services Match section, add a new line that highlights the number of extra services offered by the agent.

Use the existing canonical services comparison output already used for:

* Services Match denominator
* matched count
* extra count
* missing count

Do not create a separate new comparison pipeline.

### Display format

Show something like:

⭐ **Extra Value Added: 7 Services**

or, if you prefer cleaner wording:

⭐ **+7 Extra Services Offered**

Use the existing extra count already being calculated. Do not inflate or change the Services Match percentage.

### Transparency rule

Add a short note or tooltip that makes this clear:

> Extra services were included by the Agent beyond the Tenant’s original request. These do not increase the match score but may provide additional value.

### Important rules

* Do **not** include extra services in Match %
* Do **not** hide or reduce visibility of missing services
* Do **not** let the extra-services display replace the existing `Extra (Added by Agent)` count unless you are simply restyling the same information
* Do **not** modify the underlying scoring logic

### Scope

Tenant bid/detail view only for this step.

Do not modify Buyer, Seller, or Landlord yet unless a clearly shared presentation component is the direct root cause and can be safely updated without side effects.

### Visual style

Use a subtle but noticeable highlight:

* gold/yellow star icon
* slightly emphasized text
* should stand out, but not overpower Match Score

### Conditional display

* If extra count > 0, show the Agent Value Layer
* If extra count = 0, do not show the star line

### Verification

After the change, verify:

* Match Score percentage remains unchanged
* Missing services still display normally
* the extra-services value shown in the new star line matches the existing extra count exactly
* no change to underlying services comparison logic
