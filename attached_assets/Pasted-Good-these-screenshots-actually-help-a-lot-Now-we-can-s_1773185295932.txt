# The real problem

The issue is **only in CREATE mode (new listing)**.

The logic that decides **when to show sections after a user clicks a selection** is failing.

This is almost always caused by one of these:

### 1️⃣ Select2 value not updating the Livewire property

When you click:

```
Seller Financing
Assumable
Crypto
Exchange
Lease Option
Other
```

Select2 visually shows the tag but **Livewire doesn't update the property yet**, so the conditional `@if` blocks never re-render.

---

### 2️⃣ JS is toggling the wrong element

The handler may be running but targeting the wrong selector.

Example:

```
$('#seller-financing-assumable-section')
```

but the real section id might be:

```
seller-financing-assumable-terms-section
```

---

### 3️⃣ The handler is bound to the wrong element

Select2 replaces the `<select>` with its own container.

If the event is bound to:

```
$('.select2')
```

instead of:

```
$('#offered_financing')
```

the handler won't fire correctly.

---

# Why it works in drafts

Draft mode loads saved values and the Blade condition like this works:

```
@if(in_array('Assumable', $offered_financing))
```

So the sections appear.

But during creation, **that condition is never triggered**.

---

# The real fix (this is what Replit must do)

Tell them to **stop trying to toggle sections with JS** and instead make Livewire handle the state.

They should do **this instead**:

### Step 1 — Ensure the field is bound

The select must be:

```php
<select wire:model="offered_financing" multiple>
```

and

```php
<select wire:model="sale_provision" multiple>
```

---

### Step 2 — Add Livewire watcher

In the component:

```php
public function updatedOfferedFinancing()
{
    // force rerender
}

public function updatedSaleProvision()
{
    // force rerender
}
```

This ensures the component rerenders when the selection changes.

---

### Step 3 — Sync Select2 to Livewire

JS must push values into Livewire:

```javascript
$('#offered_financing').on('change', function () {
    @this.set('offered_financing', $(this).val());
});

$('#sale_provision').on('change', function () {
    @this.set('sale_provision', $(this).val());
});
```

---

### Step 4 — Remove manual `.show()` / `.hide()` logic

Let Blade handle visibility:

```
@if(in_array('Seller Financing', $offered_financing))
    ... section ...
@endif
```

---

# Why this works

This removes the fragile system:

```
Select2 → JS → jQuery show/hide → DOM guessing
```

and replaces it with:

```
Select2 → Livewire property → Blade condition → correct render
```

which is **much more reliable**.
