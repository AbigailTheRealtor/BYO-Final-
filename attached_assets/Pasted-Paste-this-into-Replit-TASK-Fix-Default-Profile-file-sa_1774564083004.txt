**TASK: Fix Default Profile file saving for Agent Presentation & Marketing Materials**

## Issue

Under **Agent Presentation & Marketing Materials**, uploaded files are **not saving into the Saved Default Profile**.

We need to make sure uploaded files are saved correctly as part of the Default Profile system for **all roles and all property types**:

* Seller
* Buyer
* Landlord
* Tenant

and:

* Residential
* Income
* Commercial
* Business
* Vacant Land

---

## Scope

This fix is **only** for the **Saved Default Profile** logic tied to **Agent Presentation & Marketing Materials**.

Do **not** change unrelated bid logic, listing creation, listing edit pages, or validation unless required for this file-saving fix.

---

1. **Trace the exact save/load files first**

   * Find the real route/component/model/service used for:

     * saving a bid as default profile
     * loading/editing a saved default profile
     * applying a saved default profile back into a new bid

2. **Do not assume file storage structure**

   * First confirm how uploaded files are already being stored in the bid flow:

     * file path
     * media table

     * any other storage reference

3. **Reuse the existing upload storage pattern**

   * Do not invent a different file storage system for default profiles if the bid form already has a working one

5. **Do not try to prefill native file inputs**

   * Instead, preserve and display saved file references, file names, or links when loading saved defaults

6. **Do not duplicate files unnecessarily**

## Required fix

When an agent clicks **Save as Default Profile**, the following must also save from **Agent Presentation & Marketing Materials**:

* uploaded files
* uploaded file references
* file paths / media references / attachment references
* any related saved asset entries already supported by the current bid architecture

This must work for:

* Seller
* Buyer
* Landlord
* Tenant

and all property types.

---

## Required behavior

### 1. Save to Default Profile

If the agent has uploaded files under **Agent Presentation & Marketing Materials** and clicks **Save as Default Profile**:

* those uploaded files must be included in the saved default profile
* the default profile must retain the correct file reference(s)
* this must work on both:

  * create new default profile
  * update existing default profile

### 2. Load in Default Profile Editor/Dashboard

When opening the saved default profile later:

* the saved uploaded files must still appear as existing saved assets/references
* the user should be able to keep them as-is
* the user should be able to replace them if the UI already supports replacement
* the user should be able to remove them if the UI already supports removal

### 3. Apply to New Bid

When a saved default profile is applied to a new bid:

* the uploaded file references from **Agent Presentation & Marketing Materials** must also carry over correctly
* do not attempt to visually prefill the raw file input itself
* instead show the saved file name/link/reference in the same way existing saved uploads are shown elsewhere in the app

---

## Important implementation rule

Use the **same storage pattern already used by the current bid upload system**.

Examples:

* if bids use a media table, use that
* if bids use JSON file paths, use that
* if bids use an attachment model, use that

Do **not** create a conflicting file storage method just for default profiles.
