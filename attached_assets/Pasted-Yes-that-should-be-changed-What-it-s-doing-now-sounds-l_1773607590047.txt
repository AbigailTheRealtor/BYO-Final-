Yes — that should be changed.

What it’s doing now sounds like this flow:

1. Submit finds the first missing required field
2. it takes the user there
3. user fills it in
4. then the UI kicks them back to the last tab
5. they have to click Submit again

That is clunky.

The better behavior is:

* on Submit, take the user to the **first missing required field**
* after they fill that field, **stay in the current workflow**
* if another required field is still missing, go to the **next missing required field**
* only return to the final tab once **all required fields are complete**
