## Task: Final polish for Profile Settings — improve password feedback and strengthen Delete Account confirmation

The Profile Settings revamp is mostly complete and should not be redesigned further right now.

1. make password update feedback clearer
2. make Delete Account confirmation safer

Do not change the page structure again.

---

## Part 1 — Improve password update feedback

### Current issue

Right now the settings form allows non-password profile fields to save even if the user entered the wrong current password.

That behavior is acceptable, but the feedback must be crystal clear so the user does not think their password was updated when it was not.

### Goal

When the user submits the form:

* if profile fields save successfully, show a clear success message
* if password change fails because the current password is incorrect, show a separate clear warning/error message
* user must immediately understand:

  * profile info was saved
  * password was **not** updated

### Required UI behavior

Support two messages at once when applicable:

#### Example

* **Success:** Profile settings updated successfully.
* **Warning/Error:** Password was not changed because the current password entered was incorrect.

Do not bury the password warning in tiny text or a generic flash message.

### Requirements

* the password failure message must be visible and easy to understand
* it must not imply the entire save failed if other profile fields saved successfully
* if only password fields were submitted and the current password is wrong, show only the password error
* if password update succeeds, show a clear success message confirming that too

### Safe implementation guidance

* keep profile save logic and password save logic separated conceptually
* preserve the current “do not block all profile saves” behavior unless there is a compelling reason to change it
* improve the returned feedback/messages only

### QA checklist

* [ ] profile-only update shows success
* [ ] profile + wrong current password shows success for profile plus clear password warning
* [ ] password-only with wrong current password shows clear error
* [ ] password-only with correct current password shows password success
* [ ] user is never left unsure whether their password changed

## Part 2 — Strengthen Delete Account confirmation

### Current issue

Delete Account currently uses a simple confirm flow.

That is functional, but we want a safer confirmation to prevent accidental deletion.

### Goal

Require a typed confirmation before account deletion proceeds.

### Required behavior

In the Delete Account section:

Before the final delete action runs, require the user to type:

```text
DELETE
```

exactly, then allow account deletion.

### Recommended UX

In the danger section, show:

* explanatory warning text
* confirmation input field
* delete button disabled until the typed value matches `DELETE`

or, if keeping a confirm step:

* user clicks Delete Account
* a modal or confirmation panel appears
* user must type `DELETE`
* final delete button becomes enabled only after correct input

### Requirements

* do not allow a one-click delete
* do not rely on browser `confirm()` alone
* use a clear and intentional confirmation flow
* preserve current soft-delete/logout behavior unless there is a reason not to

### Suggested warning text

Use something like:

> This action will deactivate your account and log you out. To confirm, type DELETE below.

### Safeguards

* keep delete account as a soft-delete if that is the current behavior
* do not permanently erase records unless that is already your intentional design
* preserve logout after delete
* preserve any current route/controller logic unless only minor changes are needed for the stronger confirmation flow

### QA checklist

* [ ] delete button is not immediately destructive
* [ ] user must type DELETE to proceed
* [ ] delete action stays blocked if text does not match
* [ ] soft-delete still works correctly
* [ ] user is logged out after successful delete
* [ ] no accidental deletions from stray clicks
