Bug 1 Fix — Stop Autofill Contamination:
Problem:
edit() autofills empty PROFILE_FIELDS from the most recently updated other preset. Those fallback values look real in the form, then save() permanently writes them into the current preset even when scope = This preset only.

Task:
1. Remove or disable the autofill fallback at AgentPresetController lines 118–145.
2. Empty profile fields on a preset should remain empty unless the agent enters values or intentionally uses a scope option.
3. save() should only save actual submitted values for the current preset.
4. Do not copy fallback values into profile_data.

Preferred behavior:
- “This preset only” updates only the current preset.
- “All [Role] presets” copies only approved PROFILE_FIELDS to that role’s presets.
- “All roles and property types” copies only approved PROFILE_FIELDS to all presets.
- Services, other_services, broker compensation, agency terms, and referral terms always remain preset-specific.

Bug 2 Fix — Stable Public Profile Selection:
Problem:
AgentProfileController::show() chooses the most recently updated preset with bio, so /agent/{id} changes depending on whichever preset was saved last.

Task:
1. Stop selecting primary profile purely by most recently updated preset.
2. Use a stable priority order instead.
3. Recommended priority:
   - Prefer Buyer Residential if it exists and has profile content
   - Else Seller Residential
   - Else Landlord Residential
   - Else Tenant Residential
   - Else first preset with public profile content
   - Else first preset for that agent
4. Do not let last updated order control public profile identity.

Do NOT change:
- /agent/{id}/{role}/{property_type} or equivalent role/property-specific public route if it already loads the exact preset correctly.
