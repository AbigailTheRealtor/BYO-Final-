## PART A — “Other” still not showing on ACTUAL LISTING

This is almost always because:

1. “Other” text is saved in one place but the **actual listing display** is rendering from a *different* model/field, or
2. the display code assumes **array**, but DB stores **JSON string**, or
3. the display code filters out “Other” or never appends the `*_other` text.

### A1) Find the exact blade that renders the **actual Tenant listing**

In Replit search, search these (one at a time):

* `Hire a Tenant`
* `tenant_agent_auctions.show`
* `Tenant Listing Details`
* `non_negotiable`
* `offered_lease_term`
* `resources/views` + `tenant`

**You MUST edit the blade/partial that renders the public/actual listing** (not the create/edit form).

Typical locations:

* `resources/views/tenant-agent-auctions/show.blade.php`
* `resources/views/auctions/tenant/show.blade.php`
* `resources/views/components/listing-display/*.blade.php`
* a shared partial like `resources/views/partials/listing-fields.blade.php`

> If you fix the form but not this blade/partial, it will “never show” no matter how many times you try.

### A2) Add a SAFE “decode helper” inline (no refactors)

In the actual listing display blade, **right before** rendering those two fields, paste this:

```php
@php
    $decodeToArray = function ($value) {
        if (is_array($value)) return $value;
        if (is_null($value) || $value === '') return [];
        if (is_string($value)) {
            $decoded = json_decode($value, true);
            return is_array($decoded) ? $decoded : [$value];
        }
        return [];
    };
@endphp
```

This prevents the “it’s JSON but I treat it like an array” bug.

## PART B — Edit Listing “Acceptable Property Conditions” flashes + scrolls tab up/down

This is caused by Livewire re-rendering the whole section quickly while Alpine/CSS loads.

### B1) Wrap the options UI in `x-cloak` + stable `wire:key`

In the EDIT tab blade where acceptable property conditions are shown:

```html
<div wire:key="tenant-acceptable-property-conditions">
  <div x-data x-cloak>
    {{-- your acceptable property conditions UI here --}}
  </div>
</div>

<style>[x-cloak]{ display:none !important; }</style>
```

### B2) Stop rapid re-renders: use `wire:model.defer`

If it’s checkboxes or multi-select, change from:

* `wire:model` (or `wire:model.live`)
  to:
* `wire:model.defer`

Example:

```html
<input type="checkbox"
       wire:model.defer="acceptable_property_conditions"
       value="{{ $opt }}">
```

### B3) If you’re using a JS plugin (select2/tomselect/etc.)

Plugins often re-init and cause the “flash then scroll”. In that case you must isolate:

```html
<div wire:ignore>
  <!-- plugin-controlled select here -->
</div>
```

and then sync value manually via events. (Only do this if you confirm you’re using a JS plugin.)
