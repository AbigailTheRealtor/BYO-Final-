## Required behavior for `View Bid`

When a user clicks **View Bid** from the **user-side grouped “Bids on My Listings” pages** under:

* Hire Tenant’s Agent
* Hire Landlord’s Agent
* Hire Buyer’s Agent
* Hire Seller’s Agent

it must open the **exact full bid-detail view** already shown when the user clicks **View Bid** from inside the listing flow.

### Meaning

It should open the screen/state that shows:

* **Private Compensation & Agreement Terms**
* Match Summary
* Agent Overview & Qualifications
* Services
* full bid details for that single agent bid

It must **not** just go to the generic listing page top.

* Hire Tenant’s Agent

* **View Bid** must link directly to the existing bid-detail experience for that bid

* Do not send users to a generic listing page without opening/focusing the bid detail

### Also

* the listing header/title may still link to the listing if needed
* but **View Bid** is specifically for the single-bid detail experience

Reuse the exact existing bid-detail presentation already in the product.
