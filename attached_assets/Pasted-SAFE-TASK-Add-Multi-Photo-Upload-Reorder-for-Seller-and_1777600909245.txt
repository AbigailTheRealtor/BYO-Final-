SAFE TASK — Add Multi-Photo Upload + Reorder for Seller and Landlord Offer Listings

GOAL:
Seller and Landlord Offer Listing create/edit forms should support multiple property photos, not just one photo, and users should be able to reorder photos.

SCOPE:
Offer Listings only:
- Create Seller Offer Listing
- Edit Seller Offer Listing if edit form exists
- Create Landlord Offer Listing
- Edit Landlord Offer Listing if edit form exists
- Photos, Tours & Documents tab only

DO NOT:
- Do not touch Buyer/Tenant hire-agent forms.
- Do not touch Agent Presets.
- Do not touch Broker Compensation.
- Do not touch services.
- Do not touch MLS/tax/HOA/flood fields.
- Do not delete existing photos.
- Do not run destructive migrations.
- Do not rename existing columns.
- Do not break existing video tour / virtual tour / document uploads.
- Do not change unrelated tabs.

REQUIREMENTS:

1. Multiple Photo Upload
- Property photo input should allow multiple files.
- User can upload multiple photos at once.
- Accept image types only:
  jpg, jpeg, png, webp
- Enforce the platform’s existing max file size.
- Store photos using the existing storage path/convention if one already exists.

2. 50 Photo Limit
- Maximum total property photos = 50.
- Existing photos + newly selected photos cannot exceed 50.
- If exceeded, show clear validation:
  “You may upload up to 50 property photos.”
- Do not silently discard files.
- Do not delete existing photos.

3. Helper Text
Add helper text near the property photo upload field:
“You may upload up to 50 property photos. Photos should be clear, relevant, and accurately represent the property.”

4. Photo Preview
- Show uploaded/existing photos as thumbnails.
- Each photo should have:
  - Preview thumbnail
  - Remove/Delete button
  - Reorder controls

5. Reorder Photos
Allow user to change photo order.
Acceptable implementation:
- Drag-and-drop if already available in project, OR
- Move Up / Move Down buttons for safer MVP.

Required behavior:
- First photo should be treated as primary/cover photo.
- Reordered photo order must save and reload correctly.
- Do not lose photos during reorder.

6. Existing Photos
- Existing single-photo records must still display.
- If current data is stored as one photo, safely normalize display to an array/list.
- Do not require migration unless absolutely necessary.
- If a migration is needed, stop and report why before making it.

7. Save / Reload
Verify:
- Multiple photos save.
- Multiple photos reload after draft/save.
- Deleted photos stay deleted.
- Reordered photos keep order.
- Existing video/document fields still work.

FILES TO INSPECT FIRST:
- SellerOfferListing Livewire component
- LandlordOfferListing Livewire component
- Seller photos/tours/documents blade
- Landlord photos/tours/documents blade
- Any existing photo processing methods:
  - processPendingPhotoUploads()
  - updatedNewPropertyPhotos()
  - deletePropertyPhoto()
  - property_photos / newPropertyPhotos / listingDocuments fields

REPORT BEFORE EDITING:
- Current photo storage field(s)
- Whether photos already support arrays in backend
- Whether edit forms already load existing photos
- Whether migration is needed

AFTER EDITS VERIFY:
- Seller create: upload 3 photos, save, reload, reorder, delete one.
- Landlord create: upload 3 photos, save, reload, reorder, delete one.
- 50-photo limit works.
- Helper text appears.
- No console errors.
- No backend errors.

REPORT BACK:
- Exact files changed
- Whether migration was needed
- How photos are stored
- How reorder works
- Verification results
- Any follow-up needed
