# 1) Fix the immediate crash (Minimum Annual Net Income)

### Location

`resources/views/buyerAgentAuctionDetail.blade.php` around line ~654

### Replace this (unsafe)

```php
{{ number_format($auction->get->minimum_annual_net_income) }}
```

### With this (safe-cast)

```php
{{ is_numeric(str_replace([',', '$', ' '], '', (string) $auction->get->minimum_annual_net_income))
    ? number_format((float) str_replace([',', '$', ' '], '', (string) $auction->get->minimum_annual_net_income))
    : '' }}
```

✅ This prevents the crash **even if it’s a formatted string**.

# 3) Cleaner + best-practice fix (recommended): add a reusable helper

Instead of repeating long logic everywhere, add one helper and use it across listing views.

## Option A (Laravel helper function)

Create a helper like:

**`app/Support/Format.php`**

```php
namespace App\Support;

class Format
{
    public static function money($value): string
    {
        if ($value === null) return '';

        $clean = str_replace([',', '$', ' '], '', (string) $value);

        if ($clean === '' || !is_numeric($clean)) return '';

        return '$' . number_format((float) $clean, 0);
    }

    public static function number($value, int $decimals = 0): string
    {
        if ($value === null) return '';

        $clean = str_replace([',', '$', ' '], '', (string) $value);

        if ($clean === '' || !is_numeric($clean)) return '';

        return number_format((float) $clean, $decimals);
    }
}
```

Then in blade:

```php
{{ \App\Support\Format::number($auction->get->minimum_annual_net_income) }}
```

✅ Safeguard: display-layer only, no DB changes.
