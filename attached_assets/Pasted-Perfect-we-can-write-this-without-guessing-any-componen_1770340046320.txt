## ✅ REQUIREMENT

When a user **edits** a listing and clicks **SUBMIT**, the system must:

1. **Save successfully**, then
2. **Immediately redirect to the public listing “Show” page** for that same listing
3. (Optional) Keep the success message, but it should display **on the Show page**, not the Edit page

This must apply to:

* **Hire a Tenant’s Agent → Edit → Submit**
* **Hire a Buyer’s Agent → Edit → Submit**

## ✅ SAFEGUARDS (DO NOT BREAK ANYTHING)

1. **Do NOT rename DB tables/columns.**
2. **Do NOT change Create Listing behavior.** (Edit-only fix)
3. **Do NOT change Save Draft behavior.**
4. **Do NOT change tab logic, validation rules, tooltips, labels, or field options.**
5. **Do NOT change service rendering logic.** (Not part of this fix)
6. Only modify the **final “Submit” on Edit** post-save behavior.

* It’s in an “Edit” page flow (route/view contains `edit`)

**Safeguard:** If it’s create-submit, **do not touch it**.

## STEP 4 — MAKE SURE IT ONLY HAPPENS ON **EDIT + SUBMIT**

Many edit pages have **Save Draft** and **Submit**.

### If the code has a draft flag

Look for anything like:

* `$isDraft`
* `$draft`
* `$status`
* `$submitType`

Then enforce:

```php
if ($isDraft) {
    // Keep existing draft behavior EXACTLY
    session()->flash('success', 'Draft saved successfully!');
    return; // or whatever it currently does
}

// Only redirect on real SUBMIT
session()->flash('success', 'Listing updated successfully!');
return redirect()->to($showUrl);
```

**Safeguard:** Do not change how draft is determined; only branch based on the existing variable/logic.

## STEP 5 — SUCCESS MESSAGE SHOULD SHOW ON THE SHOW PAGE

The flash message will survive the redirect automatically **only if** the Show page renders it.

Search in the Show page template for:

* `session('success')`
* `session('message')`

If it’s missing, add a minimal block near the top:

```php
@if (session('success'))
  <div class="alert alert-success">
    {{ session('success') }}
  </div>
@endif
```

**Safeguard:** If your UI uses a custom alert component already, use that instead—do not add duplicate alerts.
