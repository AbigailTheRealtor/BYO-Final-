We need to add a Referral Fee (%) negotiation term to the agent-side Hire Agent system only.

Objective

Implement Referral Fee (%) as an agent-to-agent negotiable term that appears only when the listing was created from the agent dashboard Hire Agent flow.

This feature is not for regular users/clients.

That means:

a tenant, buyer, seller, or landlord using the normal user dashboard should not see or negotiate Referral Fee
only agents creating Hire Agent listings from the agent dashboard should have this field
only agents bidding on those agent-created Hire Agent listings should see and negotiate this term
Core product rule

Referral Fee (%) is:

an agent-to-agent term
visible to bidding agents on agent-created Hire Agent listings
negotiable inside the existing bid / counter / accept flow
separate from client compensation
not paid by the client

Do not create a separate bidding system or separate negotiation flow.

It must live inside the existing Hire Agent listing/bid/counter flow, but only for agent-created Hire Agent listings.

Scope rules

This change must apply only to:

agent dashboard Hire Agent create/edit flow
agent-created Hire Agent listings
bids submitted by agents on those agent-created Hire Agent listings
counteroffers / summaries / term comparison for those agent-created Hire Agent listings

This change must NOT apply to:

user dashboard Create Hire Agent flow
regular tenant / buyer / seller / landlord listings created by end users
user-side bid cards
user-side counter flows
user-side agreement sections
public/client-facing forms

In short:

YES

Agent-created Hire Agent listings → Referral Fee exists and is negotiable by agents

NO

User-created Hire Agent listings → no Referral Fee field at all

Keep the input simple:
