* Property Type → Livewire Component → Blade tab file → Listing detail Blade file

## 2) Binding audit (prove the state updates)

Pick ONE field (Amortization Type) and show:

* The exact input markup (`wire:model`)
* A debug dump proving the Livewire property changes when you type/select
  (e.g., log/dump on updated hook)

## 3) Save audit (prove what is written)

On **Save Draft**, log:

* `$this->offered_financing`
* `$this->offered_financing_details`
* The exact payload written to the model (before save)

Then prove DB result:

* Show the stored value of `offered_financing_details` for the listing ID after save

## 4) Rehydrate audit (prove what is loaded)

On **Reopen Draft / Edit**, log:

* `$model->offered_financing_details` (as loaded from DB)
* `$this->offered_financing_details` (after mount)
* Confirm nothing resets it to `[]` afterward

## 5) Model + casting audit (serialization)

Confirm the model backing Buyer auctions has:

* `offered_financing_details` cast to array/json
* included in fillable/guarded rules as needed
* no mutators wiping/transforming the data

## 6) Listing render audit (prove the view reads the right source)

In the **public listing Blade**, confirm:

* It reads from `offered_financing_details`
* It prints the missing fields show-if-filled
* Provide screenshot where data exists in DB and displays on listing
