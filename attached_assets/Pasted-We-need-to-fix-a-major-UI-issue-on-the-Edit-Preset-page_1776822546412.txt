Problem

That means this is likely a full-page hydration / re-render / Alpine / Livewire state issue affecting the entire preset form.

We need a read-only audit first, then a small surgical fix.

What I strongly suspect

This is probably one of these patterns:

A. Alpine state resets after hydration

Example:

server renders full form
Alpine initializes with default collapsed/hidden state
entire form body disappears
B. Livewire re-render wipes values/visibility

Example:

first paint is correct
Livewire updates component
bound properties or visibility checks go empty/false
C. Main Blade conditional is too aggressive

Example:

form only renders when some array/field is truthy
second pass evaluates false
everything disappears

Stabilize the entire Edit Preset page so:

all sections remain visible

This is a rendering/hydration fix only.

Implementation rules

After the audit, apply the smallest safe fix.

1. If Alpine is controlling section visibility

Make all major preset sections default to visible/open on initial load.

Do not let Alpine initialize the page into a collapsed/empty state after first paint.

2. If Livewire is re-rendering the full form

Do not let the whole form body get wiped by a reactive re-render.

Use the smallest safe approach, such as:

stabilizing mount data
avoiding a second reset
or applying wire:ignore / wire:ignore.self only where appropriate

Do not broadly disable reactivity unless absolutely necessary.

3. If Blade conditionals are hiding the form

Remove or loosen the conditional so the form shell and saved values always render.

No preset edit page should fully collapse just because a subsection is empty.

4. Add a fallback empty-state only where needed

If a subsection truly has no values, show:

the empty fields
or a lightweight “No values set yet” helper

Do not hide the entire section.
