# 2) Fix “Other” Text Disappears (View Preferences Needed → Other)

### Likely cause

The UI shows an “Other” textbox, but the value is not actually bound to a persisted field (or it’s being reset on re-render / tab switch). 

### Required implementation (Livewire pattern)

**A. Add a dedicated persisted field for the “Other” text**

* If you store View Preferences as an array/JSON:

  * `view_preferences` (array)
  * `view_preferences_other` (string)  ✅ **must exist and be saved**

**B. Livewire component**

* Ensure the textbox uses something like:

  * `wire:model.defer="view_preferences_other"`

**C. saveDraft() must explicitly save it**

* In the Buyer “Save Draft” action, assign:

  * `$model->view_preferences_other = $this->view_preferences_other;`

**D. mount()/loadDraft() must reload it**

* When loading an existing draft:

  * `$this->view_preferences_other = $model->view_preferences_other;`

**E. Prevent “Other” from being wiped when unchecked**

* Only clear `view_preferences_other` if the user unselects “Other” AND you intentionally want to clear it.
* Otherwise, keep it stored.

# 6) Fix “Additional Details” Not Saving on Save Draft

* `$model->additional_details = $this->additional_details;`

**D. If it’s inside conditional UI**

* Add a stable `wire:key` to prevent Livewire from re-creating the textarea and losing state.
