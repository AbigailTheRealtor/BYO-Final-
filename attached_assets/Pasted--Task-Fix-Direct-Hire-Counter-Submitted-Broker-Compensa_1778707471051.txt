# Task: Fix Direct Hire Counter + Submitted Broker Compensation formatting and Referral Fee visibility

Problem:
In the Direct Hire flow, the Broker Compensation and Agency Agreement Terms display is inconsistent between:

Counter page:
/hire/agent/direct/{agentId}/{role}/{propertyType}/counter

Submitted page:
/hire/agent/direct/{agentId}/{role}/{propertyType}/submitted

Example:
Counter:
https://3ce01d2c-f79e-44c2-b048-2e8dc64316ef-00-tu7aronbf5wk.spock.replit.dev/hire/agent/direct/142/seller/residential/counter

Submitted:
https://3ce01d2c-f79e-44c2-b048-2e8dc64316ef-00-tu7aronbf5wk.spock.replit.dev/hire/agent/direct/142/seller/residential/submitted

The Counter Review tab is currently showing combined/raw values like:
- "Percentage of the Total Purchase Price + 6"
- "Yes + Percentage of the Rent Due Each Rental Period + 10"
- "Yes + 2000"
- "Yes + 500 + Applied toward final compensation"

But Submitted correctly formats these as:
- "6% of Total Purchase Price"
- "Yes"
- separate Leasing Fee row
- "Yes ($2,000)"
- "Yes ($500)"
- separate Retainer Application row

Counter Review tab should match Submitted formatting.

Also, Referral Fee is still appearing for a Tenant/consumer user on Submitted page. Referral Fee should NEVER appear for consumer users. It should only appear for logged-in Agent users, and when Agent is logged in, Referral Fee must be in its own separate Referral Fee section/tab — not mixed into Broker Compensation and Agency Agreement Terms.

Affected flow:
Direct Hire only:
- /hire/agent/direct/{agentId}/{role}/{propertyType}/counter

Required fixes:

Part 1 — Counter Review formatting must match Submitted formatting
The Counter Review tab’s Broker Compensation section must display formatted rows the same way the Submitted page displays them.

Do not build Broker Compensation rows by concatenating multiple cc[] input values with " + ".

Instead:
- Use the same canonical server-side formatting logic used for Submitted wherever possible.
- If live Review tab updates are needed before submit, either:
  A. use JS display logic that mirrors the same formatting rules exactly, or
  B. simpler/preferred: keep the Review tab Broker Compensation section server-rendered from the current effective mapped values and update only after tab changes if existing architecture supports it safely.
- The final displayed values should match Submitted format for each role/property type.

Examples that must be corrected:
Seller residential:
- seller_purchase_fee_type + seller_purchase_fee_percent should display:
  "6% of Total Purchase Price"
  NOT "Percentage of the Total Purchase Price + 6"

- buyer_broker_commission_structure + buyer_broker_commission_percent should display as separate meaningful rows:
  "Seller’s Broker to Compensate Buyer’s Broker from Seller’s Broker Commission"
  and
  "3% of Total Purchase Price"
  NOT a combined raw string

- interested_in_lease + seller_leasing_fee_type + seller_leasing_fee_percent should display:
  "Interested in Offering a Lease Agreement: Yes"
  and separate leasing fee row:
  "10% of the Rent Due Each Rental Period"
  NOT "Yes + Percentage of the Rent Due Each Rental Period + 10"

- early_termination_fee = Yes + amount should display:
  "Yes ($2,000)"
  NOT "Yes + 2000"

- retainer_fee = Yes + amount + application should display:
  "Yes ($500)"
  and "Retainer Application: Applied toward final compensation"
  NOT "Yes + 500 + Applied toward final compensation"

Apply the same principle across Buyer, Seller, Landlord, Tenant and Residential/Commercial.

- Counter
