## What Replit Should Check/Fix (most common root causes)

### 1) API is not returning the fields (projection/serializer issue)

Even if the DB has the values, the listing display might be using:

* a `select` list (Prisma/SQLAlchemy/Supabase select)
* a DTO/serializer “allowed fields” list
* a GraphQL selection set
* a “safeListing” mapper that strips fields

**Fix:** add the existing keys to the response mapping.
**Safeguard:** add-only, no renames.

Examples of what to look for (not code, just what to search):

* `select: { ... }` missing `city/county/state/zip_code`
* `toPublicListing(listing)` omits fields
* `serializeListing()` omits fields
* `ListingDTO` missing properties
* `fields = [...]` whitelist missing them

### 2) Frontend is rendering a different object than the one that has the address

Sometimes the display uses:

* `listing.propertyDetails.*` but the fields are stored on `listing.*`
* or the reverse

**Fix:** map the correct path *without moving data*.
**Safeguard:** don’t “normalize” or refactor; just read the correct path.

### 3) Residential vs Commercial display components diverged

Residential view might have it, commercial might not (or vice versa), or both missing.

**Fix:** add the same 4 rows to both display components (or the shared “PropertyDetails” component if it exists).
**Safeguard:** minimal change; avoid refactor unless a shared component already exists.
