* Loan Duration value must **persist** when:

  * moving between tabs
  * saving draft
  * editing
  * reloading the page
* The value must remain visible and stored until the user changes or clears it.

## Likely Cause (For Replit)

This usually happens when:

* tab navigation re-mounts the component and state isn’t re-hydrated from the draft/model
* conditional rendering resets the bound variable
* `updated()` / `hydrate()` / `mount()` logic overwrites the field with null
* the field isn’t included in the saved payload for drafts/submissions

## Required Fix (Implementation Requirements)

### 1) State Persistence Across Tabs

* Ensure **Loan Duration** is bound to a **stable form state property** that is not reinitialized to null when changing tabs.

  * do not overwrite existing entered values with defaults

### 2) Conditional Rendering Must Not Reset Value

* If Loan Duration is shown only when Seller Financing is selected:

  * hiding/showing the field must **not** clear its value automatically
  * only clear the value if the user explicitly changes Offered Financing away from Seller Financing **and** you intentionally clear dependent fields (if you do this, it must be consistent and deliberate)

### 3) Draft + Submit Payload Must Include Loan Duration

* Confirm Loan Duration is included in:

  * save draft payload
  * submit payload
  * edit payload (update)
* Confirm backend stores it in the correct column/key and returns it on load.

### 4) Form Load / Edit Load

* When editing an existing listing or reloading a draft:

  * if Seller Financing is selected, Loan Duration must repopulate from stored data.
