# Fix “Other” Summary Duplication in Direct Hire Counter Review Pages

## Problem

In all Direct Hire counter review summaries, when a user selects:

* `Preferred Communication Method = Other`
* `Top Priority = Other`
* (and any similar “Other + custom text” fields)

the summary currently shows BOTH:

* the literal value `Other`
* AND the custom text field underneath

Example (bad):

* Preferred Communication Method: Other
* Preferred Communication (Other): In Person

Desired behavior:

* Preferred Communication Method: In Person

and DO NOT show:

* “Other”
* “(Other)” labels
* duplicate rows

This issue exists in:

* Seller Direct Hire Counter
* Buyer Direct Hire Counter
* Tenant Direct Hire Counter
* Landlord Direct Hire Counter

---

# Required Behavior

## If dropdown value !== "Other"

Show normal summary row.

Example:

* Preferred Communication Method: Phone

---

## If dropdown value === "Other"

DO NOT display:

* the word `Other`
* separate `(Other)` summary rows

Instead:

* replace the primary field value with the custom text input value

Example:

BAD:

* Preferred Communication Method: Other
* Preferred Communication (Other): In Person

GOOD:

* Preferred Communication Method: In Person
