# 1) Define the Problem Precisely (So Dev Doesn’t Patch the Wrong Layer)

### A) Draft not saving / not restoring

This happens when:

* follow-up fields are not bound (`wire:model`) OR
* they are bound but not included in `saveDraft()` OR
* the save logic overwrites the “financing details” object (drops nested keys) OR
* mount/loadDraft doesn’t rehydrate nested objects before render

### B) Listing not showing

This happens when:

* public listing view is only printing the “selected financing types” and not the follow-up answers OR
* it’s reading the wrong key/field name (common when details are stored in JSON)

## A) Offered Financing selections (the checkboxes)

## B) Offered Financing follow-up details (the conditional questions)

### Safeguard

# 3) Fix Draft Persistence (Save + Reload)

## Step 3.1 — Ensure every follow-up input is bound to a stable Livewire property

For each follow-up question, confirm it is using something like:

* `wire:model.defer="offered_financing_details.seller_financing.<field_name>"`
* `wire:model.defer="offered_financing_details.exchange_trade.<field_name>"`
* etc.

### ⚠️ IMPORTANT

Use `defer` so tab changes don’t wipe state mid-render.

---

## Step 3.2 — Save Draft must persist BOTH the selections and the details

In the Buyer form Livewire component `saveDraft()`:

* Save:

  1. `offered_financing` selections
  2. `offered_financing_details` JSON

### ✅ Merge safeguard (CRITICAL)

When saving, do **not** do:

* `$model->offered_financing_details = $this->offered_financing_details;` **IF** `$this->offered_financing_details` can be partial/empty due to conditional rendering.

Instead:

* load existing stored details first
* merge with current details
* then save merged result

**Merge Rule**

* Existing keys stay unless user actually changed them
* New keys add
* Only clear a section if user unchecks that financing type AND you intentionally clear (optional)

---

## Step 3.3 — Load Draft must rehydrate BOTH selections and details BEFORE render

In `mount()` / `loadDraft()` for Buyer:

* `$this->offered_financing = normalizeToArray($model->offered_financing)`
* `$this->offered_financing_details = normalizeToObject($model->offered_financing_details)`

### Safeguard

* Initialize `offered_financing_details` with empty objects for each section so conditionals render cleanly:

  * `seller_financing: {}`
  * `exchange_trade: {}`
  * etc.

# 5) Prevent the “Conditional UI Wipe” Bug (Very common cause)

If those follow-up blocks are inside `@if(in_array('Seller Financing', $offered_financing))`, the moment offered_financing becomes a string or changes ordering, the follow-up block unmounts and Livewire loses state.

✅ Fix:

* Normalize `offered_financing` to an array on mount (same normalize helper as earlier)
* Wrap conditional sections with stable `wire:key` so Livewire doesn’t recreate DOM and drop input state

Example safeguard:

* `wire:key="seller-financing-section"`
