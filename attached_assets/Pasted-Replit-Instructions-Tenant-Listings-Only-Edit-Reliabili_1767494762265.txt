A) FIX — “Save Draft” Should NOT Publish a Listing
Issue

When a user clicks Save Draft, the system is currently treating it like a published listing and showing it in the main listings area.

Required Behavior

Save Draft creates/updates a record in Draft status.

Drafts must NOT appear in:

the public listings feed

the user’s active listings list (unless there is a separate “Drafts” tab/section)

any area where agents can view/bid on listings

Drafts should appear ONLY in a dedicated section:

“Drafts” tab, or

Drafts filter, or

Dashboard widget “Draft Listings”

Data Rules (Must be consistent)

Add/confirm a single source of truth field, for example:

status = draft | active | closed | archived
or

is_draft = true/false

Save Draft should always:

set status = draft (or is_draft=true)

set published_at = null (if you track publish time)

prevent the listing from entering any “bidding/visible” state

Filtering Rules (Critical)

Wherever listings are queried for display (dashboard listings, browse pages, agent bidding views):

Only show listings where:

status = active (or is_draft=false)

Drafts must be excluded by default.

Acceptance Criteria

✅ Save Draft does not show in Active Listings

✅ Agents cannot see or bid on Drafts

✅ Drafts are accessible in a Draft area only (or by editing continuation link)
