# 1) City, County, State, Zip Code do not save on draft (Landlord)

### Root cause (most common)

These fields are either:

* not included in the `saveDraft()` `fill()` / assignment
* not `fillable` on the model (if using `$model->fill($data)`)
* not included in `rules`/`validated` payload for draft save
* being overwritten/reset in `mount()` after loading the model

### Fix

## A) Ensure DB columns exist and are correct

Confirm the landlord table has columns (or equivalents):

* `city`
* `county`
* `state`
* `zip` (or `zip_code`)

**Safeguard:** do not rename; only map to what exists.

## B) Ensure Livewire properties exist and are bound

In the Landlord Livewire component, confirm public properties:

* `$city, $county, $state, $zip` (or your naming)

In Blade, confirm inputs use:

* `wire:model.defer="city"` etc.

## C) Ensure `saveDraft()` persists them

Inside `saveDraft()` (Landlord only), explicitly set them on the model:

* either via `$listing->city = $this->city;` etc
* or include them in a `$payload` used for `fill()`

**Safeguard:** do not rely on submit validation; drafts should still save partial.

## D) Ensure `mount($id)`/draft-load restores them

When opening a draft:

* load the model
* assign into Livewire properties:

```php
$this->city = $listing->city;
$this->county = $listing->county;
$this->state = $listing->state;
$this->zip = $listing->zip;
```

**Safeguard:** do not re-default these after assignment (watch for “set defaults” blocks that run after load).

# 2) “Other” inputs not re-populating (multiple fields)

This is the same class of issue across your fields:
✅ The “Other text” is saved somewhere, but on draft-load it’s not being assigned back to the Livewire property that the input is bound to.

### Required pattern for every “Other” field

For each field that has an “Other” option:

1. a main multi-select/dropdown property (array/string)
2. a separate `*_other` text property
3. saveDraft writes **both**
4. mount/load assigns **both**
5. the Blade `@if` condition shows the input when `Other` is selected, and the input binds to `*_other`

# 3) Personal Photo does not save on draft (Landlord)

### Root causes (most common)

* photo path isn’t being persisted to DB on draft save
* file upload stored temporarily but never moved to permanent storage
* on draft-load, Livewire property is empty because it expects an uploaded file object, not a saved path

### Fix (recommended approach)

## A) Persist the stored file path in the landlord table

Ensure there is a column like:

* `personal_photo_path` (or whatever you use)

**Safeguard:** If column already exists with a different name, use it. No renames.

## B) On Save Draft, if a new upload exists, store it and save path

In `saveDraft()`:

* if `$this->personal_photo` is a Livewire upload:

  * store it in a landlord-specific directory
  * save returned path to `personal_photo_path`

If no new upload, keep existing `personal_photo_path` (don’t wipe it).

## C) On draft-load, set a “preview URL” from stored path

You can’t restore the upload object itself, but you can restore display by:

* keeping `$this->personal_photo_path = $listing->personal_photo_path;`
* render `<img src="{{ Storage::url($personal_photo_path) }}">`

**Safeguard:** Do not require re-upload just to view the existing draft photo.

## D) Ensure delete works (if your UI supports it)

If “Delete photo” exists:

* set `personal_photo_path` to null
* delete file from disk (optional; or soft keep)
