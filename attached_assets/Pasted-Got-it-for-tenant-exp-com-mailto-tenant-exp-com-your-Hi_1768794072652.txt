# Phase 1 — Fix “Edit Won’t Submit” (Most Common Causes)

## 1) Confirm the Edit route exists in `routes/tenant.php`

**File:** `routes/tenant.php`

Make sure you have **both**:

* the **GET** route for the edit page
* the **POST/PATCH** route OR Livewire handles submit in-component (no separate POST route needed)

### If using Livewire page/component for edit:

You’ll typically have something like:

```php
Route::get('/buyer-agent/{id}/edit', EditBuyerAgentListing::class)->name('tenant.buyer-agent.edit');
```

✅ **Safeguard:** Don’t duplicate route names in `web.php` or `landlord.php` or you’ll get weird behavior.

### If using controller update route:

Make sure update route exists and matches the form method:

```php
Route::patch('/buyer-agent/{id}', [BuyerAgentController::class, 'update'])->name('tenant.buyer-agent.update');
```

## 2) Verify the Edit Blade actually submits to something

If it’s Livewire, your form must have:

✅ **Required**

```html

```

🚫 **Bad (common mistake)**

* `wire:submit="update"` (can cause full reload / double submit issues)

* button is not type submit

✅ The button must be:


# Phase 2 — Fix Validation Errors That Prevent Submit

## 5) Prevent “required on create” fields from blocking edit

This is a huge reason edits won’t submit: rules that are valid for Create can block Edit if some fields are hidden by logic.

### Safeguard strategy (recommended):

**Use two rule sets:**

* `rulesForCreate()`
* `rulesForEdit()`

Where `rulesForEdit()` only requires what is visible/needed on edit.

Example structure:

```php
protected function rulesForEdit(): array
{
    return [
        'title' => 'required|string',
        // only require conditional fields if their toggle is ON:
        'lease_option_terms' => $this->interested_in_lease_option ? 'required|string' : 'nullable|string',
    ];
}
```

✅ **Critical:** Do NOT require hidden conditional fields.

---

## 6) Stop type mismatches (arrays/JSON) that throw errors on validation

If you validate `array` but the model returns JSON string, Livewire may fail.

✅ Fix with casts on the model:

```php
protected $casts = [
  'acceptable_special_sale_provisions' => 'array',
  'financing_types' => 'array',
];
```

Or decode in mount (only if you cannot cast).

---

## 7) Ensure the Edit component loads all properties before validating

If your `mount()` is missing fields, validation sees them as null and fails.

✅ In Edit `mount($id)`:

* set toggles first
* then dependent fields
* ensure array fields become arrays, not strings

# Phase 4 — Confirm Tenant Route/Middleware isn’t blocking PATCH/Livewire requests

## 9) Tenant middleware / guard check (very likely in tenant.php)

In `routes/tenant.php` you probably have something like:

```php
Route::middleware(['auth', 'tenant'])->group(function () {
   ...
});
```

If your Edit submit triggers a 403/419/redirect, check:

### 419 Page Expired (CSRF)

Usually from non-Livewire POST forms missing:

```html
@csrf
@method('PATCH')
```

Livewire forms don’t need those, but controller forms do.

### 403 Forbidden (policy/ownership)

Make sure your Edit query uses `findOrFail` with tenant ownership scope:

```php
$listing = BuyerAgentListing::where('tenant_id', auth()->id())->findOrFail($id);
```

✅ This also prevents random edits by other users.

# Phase 5 — Make Sure Updates Actually Save (and don’t null out fields)

## 10) Update only the fields you actually manage

If a field doesn’t exist as a property in Edit, don’t set it to null accidentally.

✅ Best practice:

```php
$data = $this->only([
  'field_one',
  'field_two',
  // ...

$this->listing->update($data);
```
