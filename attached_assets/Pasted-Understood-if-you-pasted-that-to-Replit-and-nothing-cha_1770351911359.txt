When opening a saved landlord draft, these do not persist:

* City, County, State, Zip do not save
* Appliances Included: Other text not restored
* Desired Lease Term: Other text not restored
* Personal photo not restored

### Residential ONLY

* Landlord’s Broker Lease Fee not restored
* Rent Includes: Other text not restored

### Commercial ONLY

* Tenant Pays + Owner Pays: Other text not restored
* Terms of Lease not restored

## 5) REQUIRED: Fix must be done in 2 places (Landlord only)

### A) Save Draft

In landlord `saveDraft()`:

* explicitly persist every listed field (no “it’s already in $data” assumptions)

### B) Draft Load

In landlord `mount($id)` / `loadDraft($id)` / edit initializer:

* explicitly assign every listed field back into Livewire state

**This is the most common reason “Other” inputs don’t reappear**: the DB has the value, but Livewire doesn’t repopulate the `*_other` property.

---

## 6) REQUIRED: Cast all landlord multi-select fields to array (Landlord model only)

In the Landlord model add/verify:

* appliances_included => array
* rent_includes => array
* tenant_pays => array
* owner_pays => array
* (any other landlord multi-select fields used in conditions)

Without casts, `in_array('Other', …)` fails after draft reload, and the input won’t show.

**Safeguard:** only update Landlord model casts.

## 7) Personal photo draft fix (Landlord only)

Drafts can’t restore Livewire’s temporary upload object. So:

✅ Required approach:

* On saveDraft: store the upload to disk and save `personal_photo_path` in DB
* On draft load: set `$this->personal_photo_path` and render preview via Storage URL
* Never wipe path on draft save unless user explicitly deletes photo

**Safeguard:** do not change photo logic in other flows.
