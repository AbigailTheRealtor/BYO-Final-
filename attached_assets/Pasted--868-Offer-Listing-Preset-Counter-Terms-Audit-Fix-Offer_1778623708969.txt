#868 — Offer Listing Preset Counter Terms — Audit & Fix (Offer Listing ONLY)

Critical safeguards:

- Do NOT modify Limited Service code
- Do NOT change database schema, migrations, models, or table structure
- Do NOT rename routes unless absolutely required and proven by audit
- Do NOT remove Match Score or comparison UI from Hire Agent marketplace flows
- Do NOT refactor shared counter-term architecture
- Do NOT rename Livewire properties
- Do NOT globally modify shared CSS classes
- Shared Hire Agent behavior is the source of truth and must remain unchanged
- Any shared file edits MUST be gated behind:
  $isOfferListing
  or
  workflow_type === 'offer_listing'
- Prefer additive conditional logic over replacements/refactors
- If any shared file cannot safely support Offer Listing-only branching, STOP and report before editing

Task:

Audit and fix the Offer Listing Preset “Request Changes / Counter Terms” flow ONLY.

The Offer Listing flow currently shares code with the Hire Agent marketplace counter-term system. Several Offer Listing behaviors are broken or incorrectly inheriting marketplace UI/validation. All fixes must remain isolated behind isOfferListing guards so Hire Agent marketplace behavior remains completely unchanged.

- required indicators visible

- values preserved after failed validation

2. Buyer and Tenant counter flows show a Client Details tab containing:
   - Full Name *
   - Phone Number *
   - Email Address *
   - Areas of Interest *

- required indicators visible
- wire:model bindings present

- values preserved after failed validation

- Match Score still visible

- all current marketplace behavior preserved
