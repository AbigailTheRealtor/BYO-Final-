Replit Instructions — Remove “View Competing Bids” + Add Match Score to Competing Bid Cards (Bidding Period Listings)
GOAL

For Bidding Period / Timed listings, once an agent has submitted a valid bid, the page must:

REMOVE the “View Competing Bids” button (no separate reveal step)

DISPLAY anonymized competing bids automatically (submit-to-unlock still applies)

On each competing bid card (Agent 1, Agent 2, etc.), show:

Match Score (overall % badge)

Broker Terms match % + X/Y fields matched

Services match % + X/Y services matched

Footer label: “Compared to Your Bid”

Maintain strict privacy:

Only Broker Terms, Services, and Match Score are shown

No identities, no timestamps, no counters, no extra fields

This is for bidding period listings after an agent has made a bid.

SAFEGUARDS (MUST FOLLOW)
DO NOT:

rename/delete any database columns, migrations, models, or existing field keys

change bidding timer logic, submission validation, or counter logic

expose any personally identifying agent info in competing bids (name, email, brokerage, license, phone, profile)

show timestamps, edit history, or “last updated at” to competing agents

show counteroffer content to other agents

show any bid fields outside:

Broker Compensation & Agency Agreement Terms (summary only)

Offered Services (summary only)

Match Score module

display scoring formulas, weights, or per-field/service itemization

add rankings (“best”, “highest score”) or sorting by score

MUST:

enforce Submit-to-Unlock: competing bids visible only after the viewing agent submits a valid bid

keep anonymized labels (Agent 1/2/3…) unique per listing

show Match Score as a snapshot tied to submitted bids (no draft recalcs)

keep changes additive only (UI + safe queries)

SECTION 1 — UI Change: Remove “View Competing Bids” Button
Current UI

Button: View Competing Bids

User clicks to reveal competing bids

Required UI

Remove/hide the button entirely for bidding period listings

Competing bids appear automatically once unlocked

Implementation Rule

If:

listing is Bidding Period AND

viewer is an Agent AND

viewer hasSubmittedValidBid(listing_id, agent_id) is TRUE
Then:

render the competing bids section immediately (no button)
Else:

show locked message (see Section 2)

SECTION 2 — Submit-to-Unlock Gate (Still Required)
If agent has NOT submitted a valid bid:

Show a locked message only:

Bidding Period: Submit your bid to view anonymized competing bids (Broker Terms, Services, and Match Scores only).

Do NOT show any competing bid cards.

If agent HAS submitted:

Show anonymized competing bid cards automatically (Agent 1/2/3…).

SECTION 3 — Add Match Score to Each Competing Bid Card (Required)
Competing Bid Card Must Display (Read-only)

Each anonymized card must display ONLY:

A) Match Score (Full Module)

Overall Match Score badge: overall_percent%

Broker Terms: broker_comp_percent% + (X/Y fields matched)

Services: services_percent% + (X/Y services matched)

Footer: Compared to Your Bid

B) Offered Services Summary

“Offered Services: {count} Services” (or your existing summary)

C) Broker Compensation Summary (Allowed only if it’s part of Broker Terms summary)

Keep your existing “Broker Compensation Summary” section if it stays within the allowed scope and contains no extra bid fields.

✅ Allowed examples:

Tenant’s Broker Commission Structure (Included / Out-of-pocket / etc.)

Tenant’s Broker Commission Fee (as a format, e.g., “100% of Monthly Rent”)

❌ Not allowed:

agent identity

submission timestamps

notes

negotiation/counter terms

SECTION 4 — Match Score Calculation Rule (Snapshot-Based)
Baseline Label

Replace any “Compared to Your Original Bid” text with:

Compared to Your Bid

Baseline Definition

“Your Bid” = the viewing agent’s latest submitted bid for that listing.
Draft edits do not change baseline until re-submit.

Score Source of Truth

For competing bid cards, Match Score must be computed/stored as a snapshot tied to:

competitor’s latest submitted bid version, compared against viewer’s latest submitted bid version

Recalculation Triggers

Recalculate Match Score snapshot only when:

viewer submits/re-submits their bid

competitor submits/re-submits their bid

Do NOT recalc continuously on page refresh for every viewer; cache/store snapshots where possible.

SECTION 5 — No Timestamps + No “Last Bidder” Callout (Recommended)
Required

Do NOT show exact dates/times to competing agents.

Recommended Cleanup

Remove or avoid messaging like:

“Agent 1 was the last bidder.”

Reason: it implies timing/sequence and encourages gaming.

Instead, show:

“Bidding Period: Anonymized competing bids are visible after you submit.”

SECTION 6 — Query / Data Loading Rules (Privacy Safe)

When fetching competing bids for display:

return anonymized agent labels

return ONLY:

broker terms summary fields

offered services counts/summary

match score snapshot payload (overall + comp + services + counts + label)

Do not include:

agent_id in client-visible payload (only internal)

emails/names

timestamps

counters
