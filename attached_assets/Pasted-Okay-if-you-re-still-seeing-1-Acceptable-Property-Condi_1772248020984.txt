At this point the most reliable fix is to **stop depending on Select2 → Livewire syncing directly** (which is fragile with `wire:ignore`, morphdom, re-inits, and edit route switching), and instead use a **hidden JSON bridge** that Livewire always saves/loads correctly.

This is the “bulletproof” pattern I use when Select2 refuses to behave:

* Select2 controls the visible widget
* JS writes the selection into a hidden input as **JSON**
* Livewire saves the JSON string (always)
* Livewire decodes JSON into the real array fields (always)
* Blade uses the real array fields to decide whether to show “Other” input (always)

### Add these helper methods inside the class

```php
private function decodeJsonArray($value): array
{
    if (is_array($value)) return $value;
    if (!is_string($value) || trim($value) === '') return [];
    $decoded = json_decode($value, true);
    return is_array($decoded) ? $decoded : [];
}

private function encodeJsonArray($value): string
{
    $arr = is_array($value) ? $value : $this->decodeJsonArray($value);
    $arr = array_values(array_filter($arr, fn($v) => $v !== null && $v !== '' ));
    return json_encode($arr);
}
```

### Add these updated handlers (CRITICAL)

These make Livewire react even if Select2 is wire:ignore.

```php
public function updatedConditionPropBuyerJson($value)
{
    $this->condition_prop_buyer = $this->decodeJsonArray($value);
}

public function updatedNumberOfUnitTypeJson($value)
{
    $this->number_of_unit_type = $this->decodeJsonArray($value);
}

public function updatedPropertyItemsJson($value)
{
    $this->property_items = $this->decodeJsonArray($value);
}
```

# 3) Ensure SAVE uses the array fields (and normalizes “Other”)

In BOTH save methods (where you call saveMeta), BEFORE saving buyer meta keys, add:

```php
// Always decode from JSON bridge (source of truth)
$this->condition_prop_buyer = $this->decodeJsonArray($this->condition_prop_buyer_json);
$this->number_of_unit_type  = $this->decodeJsonArray($this->number_of_unit_type_json);
$this->property_items       = $this->decodeJsonArray($this->property_items_json);

// Normalize "Other" token for property_items
$items = $this->property_items ?? [];
$items = array_values(array_filter($items));
$otherText = trim((string)($this->other_property_items ?? ''));

if ($otherText !== '' && !in_array('Other', $items)) {
    $items[] = 'Other';
}
if ($otherText === '' && in_array('Other', $items)) {
    $items = array_values(array_diff($items, ['Other']));
}
$this->property_items = $items;
$this->property_items_json = $this->encodeJsonArray($items);
```

Then save meta like normal:

* `condition_prop_buyer`
* `number_of_unit_type`
* `property_items`
* `other_property_items`

This prevents the “listing shows Other but not the custom text” problem.

# 5) JS: On change, write JSON into hidden Livewire models (not @this.set arrays)

In BOTH:

* `resources/views/livewire/hire-buyer-agent/hire-buyer-agent.blade.php`
* `resources/views/livewire/hire-buyer-agent/hire-buyer-agent-edit.blade.php`

Inside `initializeFullService()` (or equivalent), add these THREE bindings (namespaced to avoid stacking):

```js
function setJsonModel(modelName, arr) {
    const json = JSON.stringify(arr || []);
    // Update the hidden input bound to wire:model
    const $input = $(`input[wire\\:model\\.defer="${modelName}"]`);
    if ($input.length) {
        $input.val(json).trigger('input');
    } else {
        // fallback if defer selector differs
        @this.set(modelName, json);
    }
    console.log('[JSON SET]', modelName, json);
}

// CONDITIONS
if ($('#condition_prop_buyer').length) {
    if (!$('#condition_prop_buyer').hasClass('select2-hidden-accessible')) {
        $('#condition_prop_buyer').select2({ width: '100%' });
    }
    $('#condition_prop_buyer').off('change.jsonCond').on('change.jsonCond', function () {
        setJsonModel('condition_prop_buyer_json', $('#condition_prop_buyer').val() || []);
    });
}

// INCOME UNIT TYPE
$('.number_of_unit_type').each(function () {
    const $el = $(this);
    if (!$el.hasClass('select2-hidden-accessible')) {
        $el.attr('multiple', 'multiple'); // force
        $el.select2({ width: '100%' });
    }
    $el.off('change.jsonUnit').on('change.jsonUnit', function () {
        setJsonModel('number_of_unit_type_json', $el.val() || []);
    });
});

// PROPERTY ITEMS
if ($('#property_items').length) {
    if (!$('#property_items').hasClass('select2-hidden-accessible')) {
        $('#property_items').select2({ width: '100%' });
    }
    $('#property_items').off('change.jsonItems').on('change.jsonItems', function () {
        setJsonModel('property_items_json', $('#property_items').val() || []);
    });
}
```

### IMPORTANT

Keep your existing Select2 init guards if you want, but these JSON writes MUST exist.
