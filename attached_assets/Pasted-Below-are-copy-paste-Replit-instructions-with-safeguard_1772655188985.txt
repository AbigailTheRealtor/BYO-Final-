This issue is usually caused by **Select2 visually selecting**, but **Livewire not receiving/saving** the correct array value (or receiving it, then normalization wipes it).

### Required behavior

When the user selects/unselects options, you must update:

1. The real Livewire array property (example: `condition_prop_buyer`)
   AND
2. Any JSON bridge hidden input (if you use one)

### Implementation requirement (safest approach)

In the Select2 `change` handler for this field, do BOTH:

* Set the Livewire property directly using `@this.set('condition_prop_buyer', valuesArray)`
* If you use a hidden JSON field, also update it: `@this.set('condition_prop_buyer_json', JSON.stringify(valuesArray))`

**Safeguard:** Do not rely on only the hidden JSON input OR only DOM events. Do both, every time, in all 3 modes: create / edit / draft.
