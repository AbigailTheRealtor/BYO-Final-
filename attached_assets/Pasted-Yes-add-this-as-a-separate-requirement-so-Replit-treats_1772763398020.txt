### One-Click Next Button Rule

The **Next** button must work with **one click only** for:

* new listing creation
* saved drafts
* edit listing mode

### Required behavior

When user clicks **Next**:

1. validate only the current tab fields that are actually visible and required
2. save/sync current tab values before moving forward
3. move to the next tab immediately on the first click if validation passes
4. if validation fails, stay on the current tab and show a visible notification of missing required fields

### Must prevent

* tab flicker
* delayed tab switching
* multiple-click navigation
* click being ignored because of Livewire re-render timing
* Select2 or JS reinitialization interrupting the tab change
* hidden fields blocking the click
* draft/edit mode requiring extra clicks due to missing autopopulated values

### Safeguard implementation rule

Do **not** rebuild tab logic from scratch.

Only add the minimum guard logic needed so the **Next** button consistently works on first click in:

* create flow
* draft flow
* edit flow

If the user clicks **Next** and a required field is missing, show a visible message such as:

**Please complete the required fields before continuing.**

* creating a new Hire a Tenant’s Agent listing
