### What we are implementing

## Step B1) Put Select2 init in ONE place only

Create / confirm a single global JS file:

`public/js/select2-stable.js`

It should include:

* a guarded initializer
* tab event handlers
* a Livewire hook that re-inits only missing Select2 (not destroy)

### Required guarded initializer behavior

* Never call `destroy()` in hooks
* Only init if NOT already `.hasClass('select2-hidden-accessible')`

---

## Step B2) Add tab-shown handler (this fixes hidden tab rendering)

If you use Bootstrap tabs, add:

* Event: `shown.bs.tab`

Logic:

* When a tab becomes visible, find any `select.select2` inside it
* Initialize only those not already initialized

**This is the missing piece that caused your “broken” Select2 fields.**

---

## Step B3) Restore a *guarded* Livewire hook (but do NOT reinit everything)

Add one Livewire hook:

* `Livewire.hook('message.processed', ...)`

Inside it:

* initialize only selects that exist AND are not initialized
* do not call any `initializeFullService()` that re-inits the entire page

This prevents the “flash” while still recovering from Livewire DOM morphs.

---

## Step B4) Fix wire:ignore rules (very important)

### Static-option Select2 fields

✅ Use:

```html
<div wire:ignore>
  <select class="select2">...</select>
</div>
```

### Dynamic-option selects (options change based on Livewire state)

❌ Do NOT wrap with `wire:ignore` (it blocks option updates)
✅ Either:

* don’t use Select2 for truly dynamic selects, OR
* use `wire:ignore.self` **ONLY if** Livewire changes siblings but not the select options
* or rebuild via JS manually when Livewire updates options (more complex)

**Safeguard rule:** if the select’s `<option>` list changes when Property Type changes, it’s dynamic → do not `wire:ignore` it.
