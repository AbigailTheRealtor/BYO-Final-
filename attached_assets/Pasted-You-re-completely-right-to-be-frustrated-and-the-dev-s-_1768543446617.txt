# 2) Root cause is almost certainly THIS (specific)

When you use Select2 (or any JS-enhanced select) with Livewire, you must treat it as **wire:ignore** and manually sync values with `@this.set(...)`.

If they left the Offered Financing multiselect as Select2 with just `wire:model="offered_financing"`, Livewire will often **not reliably update** the property, which means:

* The conditional blocks mount/unmount unpredictably
* Follow-up inputs can get reset or never sync
* Draft saves will miss follow-up fields

**wire:key wrappers alone won’t fix Select2 sync.**
wire:key helps with DOM stability; it does not fix JS widgets bypassing Livewire.

# 3) The fix that actually works (and is standard)

## Replit Instructions (copy/paste for dev) — “Select2 + Livewire Correct Integration”

**Applies to: Offered Financing/Currency multiselect** and any JS-enhanced selects inside those conditional follow-ups.

### A) Wrap Select2 fields in wire:ignore

In the Blade where Offered Financing select is rendered:

```html
<div wire:ignore>
  <select id="offered_financing" multiple>
    ...
  </select>
</div>
```

### B) Initialize Select2 and explicitly sync to Livewire on change

In the same view (or a JS file loaded for that page), add:

```js
document.addEventListener('livewire:load', function () {
  const el = $('#offered_financing');

  function initSelect2() {
    if (!el.length) return;

    el.select2();

    el.off('change').on('change', function () {
      const values = $(this).val() || [];
      Livewire.find(el.closest('[wire\\:id]').attr('wire:id'))
        .set('offered_financing', values);
    });
  }

  initSelect2();

  // Re-init after Livewire updates (important)
  Livewire.hook('message.processed', (message, component) => {
    initSelect2();
  });
});
```

### C) Rehydrate Select2 value when loading draft/edit

After `loadDraft()` / `loadAuctionData()` sets `$this->offered_financing`, dispatch a browser event:

```php
$this->dispatchBrowserEvent('hydrate-offered-financing', [
  'values' => $this->offered_financing
]);
```

Then in JS:

```js
window.addEventListener('hydrate-offered-financing', (e) => {
  const el = $('#offered_financing');
  el.val(e.detail.values || []).trigger('change.select2');
});
```

✅ This makes the conditional blocks show correctly and stay consistent.
