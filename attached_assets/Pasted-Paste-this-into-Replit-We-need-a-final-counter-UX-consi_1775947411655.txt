# FINAL MESSAGE STANDARD

Use only these two messages everywhere counter state is shown:

### If the viewer submitted the latest counter:

**Counter Offer Sent.**

### If the other party submitted the latest counter:

**Counter Offer Received.**

## Remove all other variants

Remove or replace all versions of:

* `Test User has submitted a counter offer.`
* `You have submitted a counter offer for this bid.`
* `Your counter offer has been submitted. Waiting for the listing owner to respond.`
* `The agent has responded to your counter offer.`
* `The agent has submitted a counter offer. Review and respond...`
* `Awaiting the seller's response.`
* `Awaiting the landlord's response.`
* `Awaiting the tenant's response.`
* `Awaiting the buyer's response.`
* any mention of `Counter Bidding History below`
* any `counter-back has been submitted` wording in status messages

Do **not** change the button label **Counter Back**. Only remove `counter-back` from status sentences.

# FINAL ACTION STANDARD

## Bid Card — when a counter exists

### If the viewer submitted the latest counter:

Show directly on the **bid card**:

* **View Counter Terms**
* **Edit Counter Terms**
* keep **View Bid**

Do **not** show:

* Accept
* Counter Back
* Reject

Because the viewer is waiting.

---

### If the other party submitted the latest counter:

Show directly on the **bid card**:

* **View Counter Terms**
* **Accept**
* **Counter Back**
* **Reject**
* keep **View Bid**

Do **not** show:

* Edit Counter Terms

Because this is a response state.
