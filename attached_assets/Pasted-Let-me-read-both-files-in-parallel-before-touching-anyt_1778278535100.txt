Task #656 — Change Report: Buyer Edit/Draft Icon & Select2 Parity Fix
One file changed: resources/views/livewire/offer-listing/buyer/offer-buyer-listing-edit.blade.php

No other files touched. No migrations. No schema changes. No Seller/Tenant/Landlord flows altered.

Change 1 — Replace addIconsToInputs() with the sentinel-based implementation
Root cause of icon duplication: The old function used parentNode (one level up only), a :scope > .input-icon dedup guard, no sentinel class, no file-input skip, and no orphan cleanup. On every Livewire message.processed re-render it injected a fresh icon because the guard was checking DOM ancestry rather than a stable sentinel class — making duplicates inevitable as the DOM was morphed repeatedly.

New implementation (identical to Buyer Create and the already-fixed Tenant Edit):

Feature	Old	New
Selector	.has-icon	.has-icon[data-icon] (both required)
Traversal	input.parentNode	input.closest('.input-cover') (depth-independent)
File skip	none	if (input.type === 'file') return
Orphan cleanup	none	removes any .input-icon:not(.data-icon-rendered) before the check
Dedup guard	:scope > .input-icon present	.data-icon-rendered sentinel class present
Sentinel class on injected icon	none	data-icon-rendered added
Insert position	before input	wrapper.firstChild
Calling addIconsToInputs() any number of times is now fully idempotent.

Change 2 — CSS: z-index: 10 on .input-cover .input-icon
Added z-index: 10 to the existing rule. Without it, icons can render behind Select2 overlay elements in some browsers. This property was present in Buyer Create and Tenant Edit but missing from the Edit blade.

Change 3 — CSS: Select2 icon-padding rule
Added:

.input-cover.has-select-icon .select2-container .select2-selection {
    padding-left: 44px !important;
}

Without this, Select2 multi-select fields that have an icon render their text flush-left, directly overlapping the icon. Present in Buyer Create (line 271) and Tenant Edit (line 299) but entirely absent from Edit.

Change 4 — shown.bs.tab icon reinjection
Added inside the existing DOMContentLoaded block, after the initial addIconsToInputs() call:

document.querySelectorAll('[data-bs-toggle="tab"]').forEach(tab => {
    tab.addEventListener('shown.bs.tab', function() {
        setTimeout(addIconsToInputs, 300);
    });
});

The 300 ms delay lets Select2 reinitialise on the newly-visible tab before icons are stamped.

Change 5 — draftLoaded event listener
Added after the livewire:load handler:

window.addEventListener('draftLoaded', function() {
    setTimeout(addIconsToInputs, 300);
});

Re-stamps icons after draft data is synced into the DOM via the draftLoaded browser event, matching the pattern in Buyer Create (line 1181).
