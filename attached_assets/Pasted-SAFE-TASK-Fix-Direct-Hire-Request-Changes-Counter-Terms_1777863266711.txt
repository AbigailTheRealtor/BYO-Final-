SAFE TASK — Fix Direct Hire Request Changes / Counter Terms Flow

Issue:
On the Hire Me / “Review This Agent’s Proposed Terms” page, clicking “Request Changes / Counter Terms” currently routes to an intermediate counter overview page showing:

- Listing Information
- Bid Information
- “No counter terms have been submitted...”
- Submit Counter Terms button

Correct behavior:
Clicking “Request Changes / Counter Terms” should go directly into the editable 3-tab Counter Terms form.

Scope:
- Direct Hire / Hire Me review page
- Request Changes / Counter Terms button
- Existing counter terms route/controller/view
- Offered Services tab in counter form
- Additional Terms default behavior

Do NOT:
- Change Agent Preset save logic
- Change service catalog wording
- Change broker compensation field names
- Change database schema unless absolutely required and reported first
- Change Offer Listing flows
- Change public Agent Profile routing
- Rebuild the counter system

--------------------------------------------------
PART 1 — Button Routing
--------------------------------------------------

On the Hire Me review page:

“Request Changes / Counter Terms” should:
1. Create/save the Direct Hire listing and bid if needed.
2. Redirect directly to the editable counter form page with the 3 tabs:
   - Broker Compensation & Agency Agreement Terms
   - Additional Details
   - Offered Services

It should NOT stop on the intermediate counter overview page.

If the current flow requires first creating the bid and then clicking “Submit Counter Terms,” bypass that intermediate screen and redirect straight to the same destination that “Submit Counter Terms” currently opens.

Return:
- Current route used by the button
- Intermediate route it currently lands on
- Final editable counter form route
- Updated route used by button

--------------------------------------------------
PART 2 — Additional Terms Must Be Blank
--------------------------------------------------

When opening the counter form from “Request Changes / Counter Terms”:

- Additional Terms should be blank by default.
- Do NOT auto-fill Additional Terms from the Agent preset.
- Do NOT auto-fill Additional Terms from the original bid unless this is editing an already-submitted counter.
- Only populate Additional Terms if the user previously saved/submitted a counter.

This applies to the editable counter form only.

Do NOT change read-only proposed terms display.

--------------------------------------------------
PART 3 — Offered Services Counter Behavior
--------------------------------------------------

In the Counter Terms form → Offered Services tab:

Correct behavior:
- Show only the services the Agent selected in their preset/proposal.
- These services should appear as editable checkboxes.
- The client/user may uncheck any services they do not need.
- Do NOT show the entire recommended service catalog.
- Do NOT show services the Agent did not select.
- This prevents users from thinking the Agent is offering services they did not include.

Additional Services:
- Keep an Additional Services textarea/input for services the user wants to request beyond the Agent’s selected services.
- These are client requests only and should be part of the counter.

Agent preset Other/Additional Services:
- If the Agent included preset other_services, show those as selected/editable services if the current counter form supports it.
- If not, show them clearly as Agent-included services and allow user to remove/request changes safely.

--------------------------------------------------
PART 4 — Preserve Existing Accept Flow
--------------------------------------------------

Do not break “Accept & Submit Hire Request.”

Accept should:
- Use the Agent’s preset selected services
- Use the Agent’s proposed compensation/agreement terms
- Not require service checkboxes from the read-only review page

--------------------------------------------------
PART 5 — Testing
--------------------------------------------------

Test Seller Direct Hire:
1. Open Seller Residential Hire Me review page.
2. Click Request Changes / Counter Terms.
3. Confirm it goes directly to editable 3-tab counter form.
4. Confirm it does NOT stop at “No counter terms have been submitted” overview page.
5. Confirm Additional Terms is blank.
6. Confirm Offered Services tab shows only Agent-selected services.
7. Confirm services are checkboxes and can be unchecked.
8. Confirm unselected services not selected by Agent do NOT appear.
9. Confirm Additional Services request field exists.
10. Submit counter and verify it saves.

Repeat quick smoke test for:
- Buyer Residential
- Landlord Residential
- Tenant Commercial
