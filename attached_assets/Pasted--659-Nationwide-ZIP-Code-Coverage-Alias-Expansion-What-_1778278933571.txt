#659 — Nationwide ZIP Code Coverage & Alias Expansion

What & Why:
The platform now has nationwide city coverage (32,145 cities) after Task #653, but ZIP-code coverage and municipality alias mapping are still incomplete/inconsistent. Some ZIP codes resolve to only one USPS-preferred municipality even when multiple real cities share the ZIP (example: Treasure Island ZIPs 33706/33707 currently resolve only to Saint Petersburg). This task expands and normalizes nationwide ZIP-code coverage so ZIP → City → County → State autofill behaves consistently across the U.S.

Goal:
Create production-grade nationwide ZIP-code coverage and ZIP-to-city relationships without affecting existing listings, drafts, or application logic.

STRICT SAFEGUARDS:
- DATA-ONLY TASK unless absolutely required for normalization utilities.
- DO NOT modify Livewire components.
- DO NOT modify Blade files.
- DO NOT rewrite autocomplete logic.
- DO NOT alter Select2 behavior.
- DO NOT touch users, listings, bids, drafts, accepted bid summaries, or messages.
- DO NOT truncate us_zip_codes.
- DO NOT delete existing ZIP rows unless duplicates are confirmed safe.
- DO NOT run migrate:fresh.
- DO NOT reseed unrelated tables.
- DO NOT perform legacy cleanup.
- Preserve existing ZIP IDs where possible.
- Use dry-run/report-first if available.
- STOP and report before any destructive operation.

Task #653 expanded us_cities from 971 → 32,145 and added missing cities nationwide. Task #658 expands county coverage. ZIP data still has municipality alias gaps and potentially incomplete nationwide coverage.
