PHASE B — CONDITIONAL SELLER DOCUMENT UPLOADS

Goal:
In Create/Edit Seller Offer Listing → Tax, Legal, HOA & Disclosures, if the user selects “Yes” for a document/disclosure question, show a file upload directly under that specific question.

Scope:
- Seller Offer Listing only
- Tax, Legal, HOA & Disclosures only
- Create + Edit flows
- Use existing EAV/meta pattern
- Use existing file upload pattern if one already exists

DO NOT touch:
- Landlord files
- Buyer/Tenant flows
- Hire Agent flows
- dashboards
- accepted bid summaries
- compensation logic
- listing detail pages
- database schema
- unrelated Photos/Tours/Documents tab

Fields needing conditional uploads:
1. Seller Disclosure Available
2. Survey Available
3. Inspection Report Available
4. HOA/Condo Docs Available
5. Flood Disclosure Available
6. Lead-Based Paint Disclosure
7. Environmental Report Available

Files expected:
- offer-seller-tabs/commission-based/tax-legal-hoa-disclosures.blade.php
- app/Http/Livewire/OfferListing/Seller/SellerOfferListing.php
- app/Http/Livewire/OfferListing/Seller/SellerOfferListingEdit.php

Implementation requirements:

1. Blade behavior
For each field:
- If value === “Yes”, show upload input directly under that field.
- If value is No, Not Applicable, Unknown, or blank, hide upload input.
- Upload input label should clearly match the question.

Example labels:
- Upload Seller Disclosure
- Upload Survey
- Upload Inspection Report
- Upload HOA/Condo Documents
- Upload Flood Disclosure
- Upload Lead-Based Paint Disclosure
- Upload Environmental Report

2. File properties
Add separate upload properties for each document type.

Suggested names:
- seller_disclosure_file
- survey_file
- inspection_report_file
- hoa_condo_docs_file
- flood_disclosure_file
- lead_based_paint_file
- environmental_report_file

3. Saved file path meta keys
Use separate meta keys so each uploaded document stays attached to the correct question.

Suggested names:
- seller_disclosure_file_path
- survey_file_path
- inspection_report_file_path
- hoa_condo_docs_file_path
- flood_disclosure_file_path
- lead_based_paint_file_path
- environmental_report_file_path

4. Storage
- Store files on the public disk.
- Use a clear folder structure such as:
  seller-disclosures/{listing_id or auction_id}/{document_type}/
- Do not overwrite existing file unless the user uploads a new one.
- If an existing file path exists, display a “View current file” link under that specific upload field.

5. Save/load
- On save/draft, if a new file exists, store it and save the path to meta.
- On edit/load, load existing file paths from meta.
- Preserve existing Yes/No field values.
- Do not require the file upload even when “Yes” is selected for now.
- Do not block draft saving.

6. Validation
Use safe validation only:
- Allowed file types: pdf, jpg, jpeg, png, doc, docx
- Max size: follow existing app upload limit if available; otherwise use 10MB.
- Do not add hard validation that breaks draft saves unless app already handles upload validation safely.

7. UI style
- Upload input should match existing upload/file-input styling.
- Place upload directly beneath the matching disclosure question.
- Do not put all uploads at the bottom.
- Add short helper text:
  “Upload the document related to this disclosure.”

8. Test one field first
Before duplicating all 7:
- Implement Seller Disclosure Available upload first.
- Confirm it appears when Yes is selected.
- Confirm upload saves.
- Confirm file link reloads on edit.

Only after that passes, replicate the same pattern to the other 6 fields.

Post-implementation verification:
- Each Yes selection reveals only its own upload field.
- No/Unknown/Not Applicable hides upload field.
- Each file saves to the correct meta key.
- Existing uploaded file appears under the correct disclosure.
- Create flow works.
- Edit flow works.
- Draft saving still works.
- No database migration was added.
- No unrelated files changed.

STOP after Phase B and report:
1. Files changed
2. Meta keys added
3. Storage path used
4. Pass/fail for each document upload field
