# 2) FIX: Edit screen “Acceptable Property Conditions” flashes all options

**Problem:** On edit, the component briefly renders “all options” (likely unfiltered) then re-renders selected values, causing a flash.

**Common Causes:**

* Options list is being reloaded async after initial render
* `wire:init` / lifecycle hydration causes temporary empty selected state
* `selectedConditions` starts null, then updates

**Fix Requirements:**

* Prevent the “all options flash” by ensuring the selected values are hydrated **before** the options render or by hiding the block until hydration completes.

**Implementation Options (pick one):**

### Option 1 (Recommended): Gate UI until hydrated

* Add a boolean like `public $hydrated = false;`
* In `mount()`: load listing → set selected arrays → then `$this->hydrated = true;`
* In Blade: wrap the conditions block:

  * Only render options when `$hydrated === true`
  * Otherwise render a skeleton/placeholder (or nothing)

### Option 2: Use `wire:key` + prefilled defaults

* Ensure the checkbox/radio group uses a stable `wire:key` tied to listing id
* Ensure selected array defaults to `[]` not `null`

**Safeguard:** Do not change the set of property condition options—only stop the flashing.

# 6) FIX: Next button on edit requires multiple clicks

**Problem:** On edits, “Next” requires clicking twice (or more). Typically caused by:

* Validation firing on first click but not showing error focus
* `wire:click` and `type="submit"` both triggering
* Button disabled/re-enabled due to loading state
* DOM re-render causing lost click

**Fix Requirements:**

* Ensure Next triggers exactly **one** handler.
* Ensure it is not blocked by Livewire loading state incorrectly.
* Ensure validation errors show immediately.

**Implementation Checklist:**

1. Ensure the Next button is **either**:

   * `type="button" wire:click="nextStep"`
     **OR**
   * `type="submit"` in a `<form wire:submit.prevent="nextStep">`
     (Do not mix both.)
2. Add `wire:loading.attr="disabled"` only if you also add a visible loading state.
3. Confirm `nextStep()` does not early-return due to missing required fields without surfacing errors.
4. If using Alpine/JS stepper, ensure it doesn’t intercept the first click.

**Safeguard:** Do not remove existing validation—fix the trigger mechanics.
