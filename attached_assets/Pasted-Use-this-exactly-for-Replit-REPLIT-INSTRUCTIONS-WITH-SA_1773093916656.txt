# REQUIRED ROOT-CAUSE CHECKS FOR EACH BROKEN LANDLORD FIELD

For **each** broken field above, inspect all of the following:

## 1. Wrapper protection

Check:

* outer container `wire:ignore`
* inner wrapper `wire:ignore`
* whether `wire:ignore.self` is incorrectly being used where full `wire:ignore` is needed

If Livewire is re-rendering the `<select>` subtree, Select2 will lose state.

---

## 2. Dynamic keying

If the options change based on property type / leasing mode / other conditional logic, add a proper unique `wire:key` so Livewire recreates the field only when it truly should.

Do not allow stale DOM reuse across different option sets.

---

## 3. Change handler must send full array

Confirm the JS sync always uses the **full current Select2 value** via `.val()`.

It must **not** send:

* only the last clicked value
* only the changed option
* selected text instead of selected array

Normalize null to `[]`.

---

## 4. No duplicate handlers

Unbind before rebinding, and use namespaced handlers.

If multiple handlers fire, one can overwrite the full array with a partial value.

---

## 5. Guarded re-init only

Check whether Select2 is being destroyed/reinitialized:

* after each change
* during `message.processed`
* during tab changes
* during unrelated property updates

Do **not** reinitialize unnecessarily.
If reinit is required, restore the full selected array correctly.

---

## 6. Duplicate field rendering

Check whether any of these fields are rendering twice, or whether two conditional sections are active at the same time.

Duplicate elements / duplicate selectors can cause selections to collapse or sync to the wrong field.

---

## 7. Livewire state vs browser state

Compare:

* browser DOM selected values
* Livewire property value

If Livewire is stale/partial, it may overwrite the correct browser state during rerender.

---

## 8. Array consistency

For every multi-select field above, verify:

* load into Livewire as array
* JS sync as array
* save as correct format already used in that flow
* reload as array
* display correctly on listing

No mixing of:

* string
* JSON string
* null
* single value

when the field is supposed to be multi-select.
