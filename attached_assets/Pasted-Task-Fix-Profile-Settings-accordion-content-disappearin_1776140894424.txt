Task: Fix Profile Settings accordion content disappearing after open

The Profile Settings accordion is still broken.

Actual current behavior

When a section is clicked:

the content briefly flashes
then disappears
the accordion remains visually open
but the body becomes blank/white

So this is not solved.

This is likely a CSS or DOM replacement issue, not just a trigger-button issue.

Goal

Make each Profile Settings section open and display its content normally without flashing or turning blank.

Required behavior:

click section = content opens and stays visible
click again = content closes
no flashing
no blank white panel
content remains readable and usable
What to inspect first
1. Inspect rendered DOM after click

Check whether the content is still present in the DOM after the panel opens.

Determine:

is the content being removed from DOM?
or is it still there but hidden by CSS?

Use browser dev tools and inspect the expanded panel after click.

2. Inspect CSS affecting the accordion content

Check all global and local CSS affecting:

.accordion-body
.accordion-collapse
.collapse
.collapse.show
.accordion-item
.card-body
.mainDashboard
any profile-settings specific wrapper
form field rows inside accordion bodies

Look specifically for:

display: none
visibility: hidden
opacity: 0
height: 0
max-height: 0
overflow: hidden
position: absolute
z-index
white text on white background
accidental color inheritance
transitions that hide content after expand
3. Check for JS conflict

Inspect whether there is still any conflict between:

Bootstrap collapse
Alpine
Livewire
custom.js
page-specific scripts

Do not leave multiple systems trying to control the same accordion.

Important implementation rule

Use one accordion system only.

Preferred solution

If this is now a normal server-rendered page:

use plain Bootstrap accordion only
no Alpine open/close logic
no Livewire state for accordion UI

OR

If using Alpine:

remove Bootstrap collapse behavior completely
use Alpine only

Do not mix them.

Strong recommendation

At this point, the safest fix is:

Option A — Use a simple custom accordion with Alpine only

Use:

one x-data wrapper
openSection state
x-show
no Bootstrap collapse JS at all

This avoids Bootstrap/global CSS conflicts.

OR

Option B — Use pure Bootstrap collapse only

If going this route:

remove custom JS/state
ensure no CSS overrides break .collapse.show

Choose one. Do not mix both.

Also inspect for this specific issue

Check whether the expanded panel body contains the content in HTML, but the content area is collapsing because of:

overflow: hidden
height: 0
empty parent wrapper
nested container with zero height
inherited background/text color issue

The white blank panel strongly suggests the content may still exist but is visually hidden.
