Replit Instructions — Counter Bid Prefills With the Other Party’s Latest Terms

Hire a Tenant Agent • Residential + Commercial ONLY

Scope (STRICT)

Apply ONLY to:

Hire a Tenant Agent listings

Residential + Commercial

The Bid Review → Counter Bid → Submit Counter workflow

Do NOT apply to Buyer/Seller/Landlord.

Goal

In the negotiation thread, the next counter form always starts with the other party’s most recent submitted terms, so the counter is literally “edit their offer and send back your version.”

This continues until either party clicks Accept Bid.

Required Behavior
1) Counter Bid Prefill Rule (Other Party)

When Tenant clicks Counter Bid on an Agent’s bid thread:

Prefill the editable form with the Agent’s most recent submitted terms for:

Broker Compensation & Agency Agreement Terms

Offered Services

If the Agent has never countered yet, use the Agent’s initial bid terms (their original bid).

When Agent clicks Counter Bid on that same thread:

Prefill the editable form with the Tenant’s most recent submitted terms (the last Tenant counter).

If the Tenant has never countered yet, use the Tenant listing’s original terms.

Key rule: The person countering edits the other party’s latest offer.

2) What gets prefilled (Must match exactly)

Auto-fill EVERYTHING inside:

Broker Compensation & Agency Agreement Terms (Residential OR Commercial version)

Offered Services checklist

“Other / Custom Services” text inputs

All conditional sub-fields (must re-trigger correctly after hydrate)

3) Persist Each Submission as a New “Offer Version”

Every time someone submits a counter:

Save a new “offer snapshot” tied to this bid thread with:

actor_role (tenant/agent)

broker_comp_terms_json

services_json

timestamp

This snapshot becomes the “latest offer” for that actor.

4) Loop Until Either Party Clicks Accept Bid

Countering continues back and forth until:

Tenant clicks Accept Bid OR

Agent clicks Accept Bid

Once accepted:

lock the thread

disable further countering

store the final accepted terms as the agreement outcome

Data / Implementation Notes (Add-only, Safe)
Store snapshots (recommended)

Create or use an existing history table for the negotiation thread. Add-only structure example:

bid_terms_snapshots

id

listing_id

bid_thread_id (or agent_bid_id)

actor_role = tenant | agent

broker_comp_terms_json

services_json

created_at

Prefill logic (exact)

When user clicks Counter Bid:

Determine currentActorRole (tenant or agent)

Set prefillActorRole = opposite(currentActorRole)

if tenant is countering → prefill from agent

if agent is countering → prefill from tenant

Then load:

latest snapshot where actor_role = prefillActorRole for that bid_thread_id

if none exists:

if prefillActorRole = agent → fallback to agent initial bid terms

if prefillActorRole = tenant → fallback to tenant listing original terms
