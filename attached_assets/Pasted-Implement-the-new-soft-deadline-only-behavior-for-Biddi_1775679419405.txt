Implement the new “soft deadline only” behavior for Bidding Period vs Traditional across all four roles:
- Tenant
- Buyer
- Seller
- Landlord

Goal:
Bidding Period and Traditional must now behave the same in terms of bid actions and negotiation behavior.
The only intended difference is:
- if listing type is Bidding Period, show the timer as informational only
- if listing type is Traditional, do not show the timer

The timer must no longer lock or restrict any actions.

Use the audit findings directly. Do not re-architect.

## Required behavior after change

### 1. Actions must no longer be timer-gated
For both Traditional and Bidding Period listings:
- listing owner can view bids at any time
- listing owner can accept at any time
- listing owner can reject at any time
- listing owner can counter at any time
- bidding agent can view and interact as currently allowed by role/state logic
- Bidding Period timer must not disable these actions

### 2. Timer becomes informational only
If listing type is Bidding Period:
- keep showing timer/countdown if already present
- relabel or restyle it as informational only if needed
- no lockout language
- no action restrictions
- no “must wait until timer ends” behavior

If listing type is Traditional:
- continue not showing the timer

### 3. Remove timer-based hard status transition behavior
Remove or neutralize the current auto transition of Bidding Period listings to Pending on timer expiry.
Specifically:
- Controller::autoTransitionBpToPending() must no longer create a blocking state for bid actions
- remove the 4 controller call sites if that is the safest path
- do not replace it with a new blocking mechanism

### 4. Keep core hired logic intact
Do NOT break:
- Hired Agent status
- sold / is_sold behavior
- accepted bid behavior
- notifications
- core listing lifecycle outside this timer change

### 5. Competing bids
If Competing Bids is retained:
- remove only the timer-active gate
- keep any submit-to-view or role/permission checks that are already valid
- do not allow timer status alone to block access anymore

### 6. Landlord backend bid submission guard
Update the Landlord bid submission guard so it does not block new bids merely because the listing is Pending or Expired due to old timer logic.
Only block if the listing is actually hired / sold / no longer open for real business reasons.
