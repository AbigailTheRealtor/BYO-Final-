1. SERVICES MUST AUTOPOPULATE FROM THE SELLER LISTING
==================================================

Problem to fix:
When an agent bids on a Hire a Seller’s Agent listing, the “Services the Seller Requests from Their Agent” section must auto-populate from the exact services the Seller selected on the original listing, same as Tenant and Landlord. It must not be blank, mixed, stale, or replaced with a different catalog.

Required behavior:
- When seller creates listing and checks requested services, those selected services become the baseline requested services for that listing.
- When agent opens bid form, those same selected services must already be shown in the same format used for Tenant/Landlord.
- When seller later views the bid, the requested services comparison must reference the seller’s original selected services.
- When counter bids occur, the latest counter terms must carry forward the correct seller requested services display and comparison logic.

Important:
This section is NOT a fresh agent profile block and NOT a new independent services list.
It is the seller listing’s requested services baseline, same concept as Tenant and Landlord.

Implementation rules:
- Reuse the same data loading pattern used in Tenant flow for listing-selected services.
- Reuse the same filtering/sanitizing logic used in Landlord after the recent fixes so only valid seller services tied to the seller property type are shown.
- Ensure stale services from another role/property type are never injected.
- Filter loaded services against the current Seller service catalog before rendering.
- Preserve section headings, grouping, emoji headings, order, and checked state exactly as seller listing creation defines them.

Expected label:
“Services the Seller Requests from Their Agent”

If “Other – Specify additional services as needed” exists in seller flow and was selected, preserve the exact same logic and display behavior currently used in seller listing creation/edit/view.

- Seller should see each bid as one bid from one agent, using the same card style and action layout as Tenant.
- Agent dashboard should also show the bid in the same “My Bids” pattern Tenant uses.

Meaning:
Do not introduce a separate seller-only card format.
Do not switch to a profile-first layout that breaks bid consistency.
Use the Tenant card structure and only substitute Seller-specific field labels/content.

==================================================

- All actions must be bid_id-specific, not listing-wide

Use the exact Tenant permission/action pattern already proven.
