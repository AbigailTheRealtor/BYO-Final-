## PART 3 — Most common real cause: the SHOW query doesn’t select the columns

If your controller/query does something like:

```php
TenantAgentAuction::select('id','field1','field2')->find($id)
```

…it will silently drop your new columns.

### Fix (SAFE):

Add the missing columns to the select list OR remove the select restriction.

Example:

```php
->select([
  'id',
  // ...existing columns
  'non_negotiable_features_amenities',
  'non_negotiable_features_amenities_other',
  'offered_lease_term',
  'offered_lease_term_other',
])
```

If you’re using `->toArray()` or `->only([...])` before sending to the view, add the new keys there too.

## PART 4 — Once debug proves the correct variable + columns exist, apply the FINAL display patch

### 4A) Use a robust decoder (handles JSON, array, string)

```php
@php
  $decodeToArray = function ($value) {
    if (is_array($value)) return $value;
    if ($value === null || $value === '') return [];
    if (is_string($value)) {
      $decoded = json_decode($value, true);
      return is_array($decoded) ? $decoded : [$value];
    }
    return [];
  };
@endphp
```

### 4B) Non-Negotiable display (final)

```php
@php
  $nn = $decodeToArray($auction->non_negotiable_features_amenities ?? null);

<div class="listing-row">

  @endif
</div>
```

# PART 5 — Fix: “Acceptable Property Conditions” flashes + scrolls tab

This is usually Livewire re-render + Alpine init + scroll restore.

### 5A) Prevent flash

Wrap only that section:

```html
<div wire:key="tenant-acceptable-property-conditions" x-data x-cloak>
  <!-- your options -->
</div>

<style>[x-cloak]{display:none !important;}</style>
```

### 5B) Stop repeated rerenders while clicking

Change **all** models in that section to `.defer`:

```html
wire:model.defer="acceptable_property_conditions"
```

### 5C) Prevent the page from jumping on render

If you have any JS that scrolls on step change, disable it for this step.
Also add to the outer container:

```html
<div style="scroll-margin-top: 120px;">
```

And make sure your “Next” button does NOT trigger a form submit + Livewire click at the same time.

# PART 6 — Fix: Next button needs 2 clicks (common)

Make it **type="button"** and use only one action.

```html

        wire:click.prevent="goNextStep"
        wire:loading.attr="disabled"
        wire:target="goNextStep">

</button>
```

If you currently have both `href` and `wire:click`, remove the `href` (keep navigation inside Livewire).
