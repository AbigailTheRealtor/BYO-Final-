GOAL:
Allow listing owners/admins to review saved AI Questions / Chatbot Knowledge Base answers after saving a listing, and generate a secure private AI Data Link that can be copied into Brainlee or another AI system.

FEATURES TO ADD:
1. Read-only “AI Knowledge Base (Internal)” section on each listing detail page.
2. Only visible to listing owner/admin.
3. Show only answered AI Questions, grouped by category.
4. If no answers exist, show helpful empty state.
5. Add “Edit Answers” button linking to Edit Listing.
6. Add “Copy AI Data Link” button.
7. Add optional “Regenerate Link” button.
8. AI Data Link returns safe JSON for Brainlee.

APPLIES TO THESE DETAIL PAGES:
- Seller property listing detail page
- Buyer criteria listing detail page
- Landlord property listing detail page
- Tenant criteria listing detail page

If these use separate Blade files, update each one.

SECTION LABEL:
AI Knowledge Base (Internal)

HELPER TEXT:
These answers are internal and help the listing chatbot respond accurately. They are not shown publicly on the listing page.

DISPLAY RULES:
- Collapse section by default.
- Group answers by category.
- Only show non-empty answers.
- Do not show unanswered questions.
- Do not expose answers to public users.
- Only owner/admin can view.
- If no answers exist, show:
  “No AI Knowledge Base answers have been added yet. You can add them in Edit Listing.”

BUTTONS:
- Edit Answers
- Copy AI Data Link
- Regenerate AI Data Link

SECURE AI DATA LINK REQUIREMENTS:
- Add nullable unique column:
  ai_share_token
- Use secure random token.
- Do not use listing ID as token.
- If token does not exist when owner/admin clicks Copy AI Data Link, generate and save one.
- Regenerate button should rotate the token and invalidate the old link.
- AI Data Link should not require login so Brainlee can read it.
- Link must not be discoverable or indexed.
- Add noindex headers if possible.

ROUTE:
GET /ai-knowledge/{token}

CONTROLLER LOGIC:
- Find listing by ai_share_token.
- If token invalid, return 404.
- Return JSON only.
- Do not render a public webpage.

JSON RESPONSE SHOULD INCLUDE ONLY SAFE LISTING CONTEXT:
- listing_id or public listing reference if already safe
- role/listing type: seller, buyer, landlord, tenant
- property_type
- transaction_type if available
- listing title/address only if already safe/public on the listing
- AI FAQ / chatbot knowledge base answers grouped by category
- answered questions only
- generated_at timestamp

DO NOT INCLUDE:
- User email
- User phone
- Seller private contact info
- Buyer private contact info
- Tenant private contact info
- Landlord private contact info
- Agent private contact info
- Broker compensation
- Referral fees
- Internal notes
- Admin notes
- Sensitive account data
- Any Hire-an-Agent data

DATA SOURCE:
Use existing AI Questions JSON field:
- listing_ai_faq
or
- chatbot_knowledge_base

If both exist, support both safely and normalize output.

NORMALIZED JSON EXAMPLE:
{
  "listing_type": "seller",
  "property_type": "residential",
  "knowledge_base": [
    {
      "category": "Property Details",
      "question_key": "roof_age_condition",
      "question": "What is the age and condition of the roof?",
      "answer": "Roof replaced in 2018 and is in good condition."
    }
  ]
}

CHATBOT SAFETY:
- If a question has no answer, do not include it in the JSON.
- Brainlee/chatbot should not hallucinate missing answers.
- Include a small JSON note:
  "missing_answer_behavior": "If information is not provided, state that it has not been provided and offer to connect the user or ask a follow-up."
