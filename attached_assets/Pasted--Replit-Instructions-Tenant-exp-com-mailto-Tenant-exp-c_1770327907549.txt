### Hire a Tenant’s Agent → “Next” Button Skips Tabs (Must Advance 1 Tab Only)

---

# Goal

In **Hire a Tenant’s Agent** listing creation (and edit/draft where tabs exist), clicking **Next** must:

* advance **exactly ONE tab forward**
* never skip to every other tab
* work consistently for Residential + Commercial

---

# SAFEGUARDS (MANDATORY)

1. **Tenant-only scope:** Do not change Buyer/Seller/Landlord tab navigation.
2. **Do not change tab order or labels.** This is navigation logic only.
3. **Single source of truth for active tab:** there must be only ONE state variable controlling the active tab (either Livewire or JS, not both fighting).
4. **No duplicate click handlers:** ensure the Next button has exactly one action pathway (either `wire:click` OR JS listener, not both).
5. **Validation must not cause double-advance:** if validation passes, move +1; if fails, do not move.
6. **Works for Draft/Edit:** same logic used everywhere the tab wizard exists.

---

# Most Common Root Causes (what to fix)

This “skips every other tab” typically happens when **two handlers fire**:

* Example: `wire:click="nextTab"` AND a JS `addEventListener('click', ...)` both increment the tab.
* Or: both a `<button type="submit">` triggers form submit (JS) AND Livewire click increments.
* Or: a `dispatchBrowserEvent('next-tab')` is listened to twice.

✅ Fix = ensure only one increment occurs per click and lock navigation with a simple guard.

## Step 5 — Add a “one-click guard” in Livewire to prevent double-advance

Even after removing duplicates, add a guard so this never returns.

### In Tenant Livewire component

Add:

```php
public bool $isNavigating = false;

public function goNext(): void
{
    if ($this->isNavigating) {
        return; // prevents double fire
    }
    $this->isNavigating = true;

    // OPTIONAL: validate only the current tab here
    // $this->validateCurrentTab();

    $this->activeTab = min($this->activeTab + 1, $this->maxTab());

    // release guard after render tick
    $this->dispatchBrowserEvent('tenant-nav-unlock');
}

public function goBack(): void
{
    if ($this->isNavigating) {
        return;
    }
    $this->isNavigating = true;

    $this->activeTab = max($this->activeTab - 1, 1);

    $this->dispatchBrowserEvent('tenant-nav-unlock');
}

public function maxTab(): int
{
    return 8; // set to your actual Tenant tab count
}
```

### In the Tenant Blade (only on this wizard page), unlock:

```html
<script>
  window.addEventListener('tenant-nav-unlock', () => {
    // slight delay ensures Livewire has applied state
    setTimeout(() => {
      @this.set('isNavigating', false);
    }, 50);
  });
</script>
```

✅ This prevents “double increment” even if a second click event sneaks back in later.

## Step 7 — Ensure “Next” isn’t firing twice due to form submit

If the wizard sits inside a `<form wire:submit.prevent="save">`, and Next is `type="submit"`, clicking Next can:

* submit the form (triggering logic)
* AND trigger `wire:click`

✅ Fix:

* Next must be `type="button"`
* Save/Submit buttons are `type="submit"` only on final step
