Goal:
Audit how saved drafts auto-populate in Hire Agent listings and verify why corresponding drafts do or do not appear immediately when a user selects a listing.

Scope:

- Shared Livewire component:
  app/Livewire/TenantAgentAuction.php
- Shared Blade view:
  resources/views/livewire/tenant-agent-auction.blade.php
- All four user types:
  Seller, Buyer, Landlord, Tenant

Do NOT modify code.
Do NOT change database schema.
Do NOT change validation.
Do NOT touch Offer Listing flows.
Do NOT touch Limited Service / legacy flows.

Expected Behavior:
When a user selects a listing inside the Hire Agent flow:
1. The system should immediately detect the selected listing.
2. Any saved drafts connected to that listing and user type should appear right away.
3. The user should not need to refresh, change tabs, or reselect the user type.
4. Drafts shown must correspond only to the selected listing and correct user type.
5. Draft selection should load the correct draft data into the Hire Agent form.

Audit Questions:
1. Which method runs when a listing is selected?
2. Does that method immediately refresh the available drafts?
3. Is the selected listing ID stored before draft lookup runs?
4. Are drafts filtered by listing ID, user type, and authenticated user correctly?
5. Is there any mismatch between:
   - listing_id
   - selected_listing_id
   - offer_listing_id
   - draft parent ID
   - user_type
6. Does the Blade draft modal/dropdown depend on a condition that prevents it from showing after listing selection?
7. Are Livewire events needed after listing selection to re-render the draft picker?
8. Does loadDraft() correctly preserve user_type and selected listing context?
