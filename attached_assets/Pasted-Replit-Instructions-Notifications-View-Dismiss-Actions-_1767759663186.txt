Replit Instructions — Notifications: “View” + “Dismiss” Actions

Platform: BidYourAgent
Applies To: Agent Dashboard & Tenant Dashboard
Scope: All notification types (Bid, Counter-Bid, Accepted, etc.)

OBJECTIVE

Update the “Recent Notices” component so that:

Each notification includes an explicit View action that takes the user directly to the relevant section (e.g., Counter Bid terms, Bids section, Accepted Bid Summary).

Each notification includes a Dismiss action that removes the notice from the list without navigating away.

Navigation is intentional, clear, and role-safe for both Agents and Tenants.

SAFEGUARDS (MUST FOLLOW)

DO NOT:

delete, rename, or modify any existing database tables, columns, models, or migrations

change existing notification text, icons, layout structure, or styling beyond adding buttons

alter bid, counter-bid, acceptance, or agreement logic

remove or repurpose the Dismiss button

auto-mark notifications as dismissed unless the user explicitly clicks View or Dismiss

YOU MAY:

add new nullable fields if absolutely required (non-breaking only)

add new controller routes/endpoints

add redirect logic based on notification type

add buttons or links within the existing notification UI

REQUIRED UI CHANGES
1) Add Explicit “View” Button

Each notification row must display:

View (primary action)

Dismiss (secondary action)

Button label: View
Placement: Right side of the notification, next to Dismiss
Consistency: Use the same label across Agent and Tenant dashboards

2) Button Behavior
View Button

Clicking View must navigate the user to the correct context related to that notification.

View should not dismiss the notification until navigation succeeds.

Dismiss Button

Clicking Dismiss:

marks the notification as dismissed (or read)

removes it from the “Recent Notices” list

does not navigate away from the page

must stop event propagation (in case the row is clickable elsewhere)

ROUTING & NAVIGATION LOGIC
3) Add Central Notification Redirect Route

Create a single route:

GET /notifications/{notification_id}/go


This route must:

Verify the notification belongs to the authenticated user

Verify the user is authorized to access the related listing

Resolve the correct destination URL based on notification type

Redirect the user to that destination

Unauthorized access must return 403.

4) Notification Type → Destination Mapping
Notification Type	Destination
Counter Bid Submitted	Listing page → Counter Bid section
New Bid Submitted	Listing page → Bids section
Bid Accepted	Accepted Bid Summary page
Other Updates	Relevant listing section or tab

Example URL patterns:

/listings/{listing_id}?view=counter

/listings/{listing_id}?view=bids

/accepted-bids/{accepted_bid_id}

LISTING PAGE BEHAVIOR
5) Auto-Open Correct Section

On the listing page:

Read the view query parameter

Automatically open and scroll to the appropriate section:

view=counter → Counter Bid terms

view=bids → Bids list

view=messages → Messages (if applicable)

Do not change existing layout or structure — only add conditional open/scroll logic.

DISMISS LOGIC
6) Dismiss Endpoint

Create or reuse:

POST /notifications/{notification_id}/dismiss


Behavior:

Verify ownership of the notification

Mark the notification as dismissed (do not delete)

Return a success response

Frontend removes the notification from the list without refreshing the page

FRONTEND INTERACTION RULES
7) Interaction Rules

Only View triggers navigation

Dismiss never triggers navigation

Notification rows do not require click-anywhere behavior

Buttons must be accessible and usable on desktop and mobile

TESTING REQUIREMENTS (MUST PASS)

Clicking View opens the correct section for each notification type

Clicking Dismiss removes the notice without redirecting

Unauthorized users cannot access /notifications/{id}/go

Counter-bid notifications land directly on the counter terms section

Works correctly for:

Agent Dashboard

Tenant Dashboard

Structure is reusable later for Buyer, Seller, and Landlord dashboards

FINAL UX DECISION (LOCKED)

Use explicit View + Dismiss buttons

Do not rely on click-anywhere notification rows

This pattern is intentional, auditable, legally safer, and scalable across BidYourAgent
