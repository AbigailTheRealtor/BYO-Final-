Replit Instructions — Tenant Listings Only: Allow Expiration Date Edit + Redirect After Create/Edit Bid
Scope (STRICT)

Apply changes ONLY to:

Hire a Tenant Agent listings (Residential + Commercial)

Listing edit screen for Tenant listings

Bid create/edit flow pages that originate from Tenant listings

Do NOT apply to Buyer/Seller/Landlord.

1) Edit Listing Rules — Expiration Date IS Editable
Requirement

When editing a Tenant listing, the user can edit everything except:

Listing Type (Traditional vs Bidding Period)

Bidding Timer fields (anything that drives the countdown / unlock logic)

✅ Expiration Date must remain editable, because it is not linked to the bidding timer.

Implementation Guidance

If “Expiration Date” is currently grouped with timer fields, separate it logically:

Keep Expiration Date enabled in edit mode

Only disable true timer configuration fields (end time / bidding duration / timer start/end that controls “actions unlock”)

Backend Safeguard

On update:

Reject/ignore changes to true timer fields (listing type + timer config)

Accept changes to Expiration Date normally

Acceptance Criteria

✅ Expiration Date field is editable and saves properly

✅ Listing Type cannot be changed

✅ Bidding timer config cannot be changed

2) Navigation Fix — After Create Bid or Edit Bid, Redirect to the Listing Page
Issue

After a user creates a bid or edits a bid, they are not being taken back to the listing automatically.

Required Behavior

When an Agent (or user acting as Agent) submits:

Create Bid
OR

Edit Bid
the app must immediately redirect to:

the listing detail page for that Tenant listing
(“go directly to the listing once they create it”)

Redirect Rule (Exact)

On successful response (HTTP 200/201), redirect to:

/tenant-listings/{listing_id}
(or your actual tenant listing route)

Do not redirect to the dashboard unless that is your listing detail.

For Edit Bid

After updating a bid:

redirect to the same listing detail page

preserve scroll position if possible (optional)

optionally open the “Bids” section (optional)

Implementation Notes (Framework-Agnostic)

Wherever the submit handler is:

Create Bid submit success:

must call navigate(listingUrl) / redirect(listingUrl) after success

ensure this happens only after the bid is actually saved

Edit Bid submit success:

same redirect behavior

Prevent double-submit edge cases

disable submit button during save (isSubmitting)

ensure isSubmitting resets on both success and error
