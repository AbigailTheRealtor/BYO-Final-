# Phase 1A — Tenant Representation Preferences & Compatibility Tab

What & Why
Add a new "Representation Preferences & Compatibility" tab to the Hire Tenant’s Agent wizard (Full Service flow only). This questionnaire lets Tenants describe their rental goals, communication preferences, negotiation style, decision-making approach, support expectations, risk tolerance, and ideal Agent working style.

This supports BidYourAgent’s premium representation marketplace direction by helping Agents better understand the Tenant’s needs before submitting representation proposals.

This task is additive-only. No existing tabs, fields, logic, routes, migrations, or unrelated flows may be modified.

Done looks like
- A new "Representation Preferences & Compatibility" tab appears in the Full Service Hire Tenant Agent wizard.
- The tab is positioned immediately before the existing "Broker Compensation & Agency Agreement Terms" tab.
- The tab only appears when user_type = tenant.
- All fields render with matching form styles, spacing, icons, placeholders, and tooltips.
- Text input/textarea placeholders follow: Enter [Field Title] (e.g., Example).
- Single-select fields use Select2 with placeholder "Select".
- Multi-select fields use Select2 with closeOnSelect: false and placeholder "Select".
- Required fields validate on Save Edit / Submit, not Save Draft.
- All fields persist on Save Draft and reload correctly.
- All fields persist on Save Edit and reload correctly.
- Older drafts/listings without compatibility_preferences open and save normally.
- No unrelated meta values are overwritten.
- Next/Back, correction mode, mobile wizard, and Submit/Finish remain correct.
- Navigation uses dynamic tab targeting/selectors instead of fragile hardcoded indexes where possible.
- Tenant Information / review area shows a "Representation Preferences & Compatibility" section with only non-empty values.
- No Seller, Buyer, Landlord, Offer Listing, Direct Hire, or Limited Service files are changed.

Out of scope
- AI scoring
- match percentages
- ranking/recommendation logic
- Offer Listing flows
- Direct Hire flows
- Limited Service / Flat Fee flows
- schema migrations
- route changes

Constraints — do not alter
- bidding logic
- broker compensation logic
- services selection logic
- Save Draft behavior outside the new fields
- existing review formatting outside the new compatibility block
- Accept / Counter / Reject flow

Steps

1. Add compatibility_preferences property and storage
- Add public $compatibility_preferences = [];
- Use existing JSON/meta storage only.
- Save under key: compatibility_preferences.
- Use reusable structure with a tenant_specific sub-array.
- Default to an empty structure when older drafts/listings do not contain the key.
- Do not overwrite unrelated meta keys.

2. Add validation rules
Required on Submit / Save Edit only, not Save Draft:
- communication_style
- negotiation_style
- primary_rental_goal
- representation_priorities
- preferred_agent_working_style

All other compatibility fields nullable.
