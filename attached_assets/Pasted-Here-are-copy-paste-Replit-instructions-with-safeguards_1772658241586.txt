# 1) Stop Livewire from re-rendering Select2 blocks constantly

**Symptom:** dropdown selections “flash”, revert, or feel delayed because Livewire keeps re-rendering the DOM.

### Required changes (Buyer pages + shared tenant pages that handle buyer draft/edit)

✅ Ensure the `<select>` is inside a `wire:ignore` wrapper:

* `wire:ignore` must be on the parent wrapper that contains the `<select>`.

**Safeguard:** Do not remove the field; just ensure Livewire doesn’t try to diff Select2 DOM.

---

# 2) Debounce “change” handlers so Livewire isn’t hammered

**Symptom:** every click sends too many Livewire updates, causing the whole form to lag.

### Required

In the Select2 `.on('change')` handlers, wrap `@this.set()` calls with a debounce timer.

Example pattern:

* Keep a `let timers = {};`
* On change:

  * clearTimeout(timers[fieldName])
  * timers[fieldName] = setTimeout(() => { @this.set(...) }, 150)

**Use 150–250ms** debounce.

**Safeguard:** This must still set values before Save. (Step 4 ensures pre-save sync.)

---

# 3) Only initialize Select2 once per field per render (guarded init)

**Symptom:** slow + glitchy because Select2 is being re-initialized repeatedly.

### Required

Before calling `$(selector).select2(...)`, add this guard:

* If element already has class `select2-hidden-accessible`, **do not re-init**.

Example:

* `if (!$(selector).hasClass('select2-hidden-accessible')) { init }`

**Safeguard:** If you destroy/recreate Select2 anywhere, you must reattach handlers. Prefer NOT destroying.

---

# 4) Add a “syncAllSelect2BeforeSave()” that runs immediately on Save

**Reason:** Debounce improves performance but can delay updates. Pre-save sync guarantees the server receives the current values.

### Required

Right before calling Livewire save action (Create/Draft/Edit):

* Read each Select2 value (`.val() || []`)
* Push to Livewire using `@this.set()` immediately (no debounce)
* If you use JSON bridges, set them too

This ensures:
✅ form remains fast during interaction
✅ save is always accurate

**Safeguard:** This sync function must only set fields related to the form, not trigger extra renders.

---

# 5) Reduce Livewire updates from normal text inputs (big lag source)

**Symptom:** typing feels delayed because Livewire sends requests on every keystroke.

### Required

On Buyer forms, change text inputs that do not need instant validation from:

* `wire:model="field"`

to:

* `wire:model.defer="field"`

Or at minimum:

* `wire:model.lazy="field"`

**Safeguard:** Do not change fields that rely on instant conditional logic (if any). Start with large text areas and “Other” inputs first.

---

# 6) Disable expensive Livewire hooks while typing/selecting

**Symptom:** message.processed hooks + initializeFullService() re-run too often, causing lag.

### Required

Inside your `Livewire.hook('message.processed', ...)`:

* Only call `initializeFullService()` when:

  * the relevant tab is active OR
  * the component actually changed (not every processed message)

Practical guard:

* Track the last initialized `wire:id` or a timestamp and avoid repeated init within 300ms.
* Or check if the form container exists and is visible before initializing.

**Safeguard:** Do not remove the hook; only guard it.
