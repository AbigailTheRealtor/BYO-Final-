## REPLIT INSTRUCTIONS — STRICT MODE (DO NOT SKIP STEP 1)

### STEP 1 — IDENTIFY THE ACTIVE FILE FIRST

Before writing **any code**, identify the exact file that renders this page.

The page I am testing is:

```
/hire/agent/auction/buyer
```

You must first determine:

1️⃣ The **route definition** for this URL
2️⃣ The **Livewire component used by the route**
3️⃣ The **Blade view rendered by the component**
4️⃣ Any **tab partials included inside the blade**

Print the exact file paths before making changes.

Example format:

```
Route:
routes/web.php → /hire/agent/auction/buyer

Livewire Component:
app/Http/Livewire/_________.php

Blade View:
resources/views/livewire/_________.blade.php

Included Partials:
resources/views/livewire/.../property-preferences.blade.php
resources/views/livewire/.../services.blade.php
```

Do **NOT assume filenames**.

Do **NOT edit files until the active path is confirmed.**

---

### STEP 2 — CONFIRM CREATE VS EDIT FLOW

Verify whether this route uses:

• Create component
• Edit component
• Shared component
• Separate tab partials

Many bugs happen because fixes are applied to the **edit flow instead of the create flow**.

Confirm which one is active.

---

### STEP 3 — VERIFY THE FIELD SOURCE

Before editing a field, confirm where the field options come from:

Possible sources:

• Blade array
• Livewire component property
• config file
• database table
• shared constants file
• JavaScript initialization

Print the source before editing.

---

### STEP 4 — IMPLEMENT FIX ONLY IN THE ACTIVE PATH

After confirming the active files, implement the fix **only in the files used by this route**.

Do NOT edit unused duplicates.

Do NOT modify unrelated flows.

---

### HARD SAFEGUARDS

Do NOT:

• rename database columns
• change field names
• modify unrelated roles (Seller / Tenant / Landlord)
• refactor architecture
• remove Livewire logic
• remove Select2
• modify existing tooltips or labels unless specifically requested

Fix **only the reported issue**.

---

### STEP 5 — VERIFY IN UI

After implementing the fix, verify directly on:

```
/hire/agent/auction/buyer
```

Confirm the fix works on:

• create listing
• tab navigation
• save draft
• reload page
• reopen draft
• listing display page

# Why This Template Works

Your platform likely has **4 versions of each listing flow**:

```
TenantAgentAuction.php
BuyerAgentAuction.php
LandlordAgentAuction.php
SellerAgentAuction.php
```

And also:

```
auction
auction-edit
auction-tabs
partials
shared select2 scripts
```

So when Replit fixes something like:

```
buyer-agent-auction-edit.blade.php
```

but your page uses:

```
buyer-agent-auction.blade.php
```

the bug **never actually gets fixed**.

This template forces them to **prove the active file first**.
