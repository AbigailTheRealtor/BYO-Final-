## Problem

Under **Leasing Terms**, after:

* the follow-up items are being rendered inside a **list (`ul/li`)** or **chip/tag container**
* and/or the container has a **fixed height / wrong line-height / absolute positioning**
* causing multiple rows of text to sit on top of each other

## What To Change (Implementation Notes)

### 1) Fix the HTML structure (most likely cause)

In the **Landlord listing display component** where Leasing Terms are rendered:

* Do **NOT** render these follow-up items as `<ul><li>`
* Do **NOT** render them as “tag pills/chips”
* Render them using the same pattern as other fields:

**Use a simple vertical stack:**

* `<div class="space-y-1">`
* each field row: `<div><strong>Label:</strong> Value</div>`

### 2) Remove/override list styling if a list is required

If a list is currently being used and you don’t want to refactor:

* remove list styling so no bullets/icons appear:

  * `list-style: none;`
  * `padding-left: 0;`
  * `margin: 0;`

### 3) Fix the CSS that is causing overlap

Check the wrapper around the Leasing Space follow-up block for any of these and remove them:

* `position: absolute`
* `top/left` offsets
* fixed height like `height: 24px` or `max-height` that clips
* `overflow: hidden`
* incorrect `line-height` (set it to normal)

**Required:** The container must allow multiple lines and grow naturally.

### 4) Make sure the follow-ups are only shown when they actually exist

Only show each line if the stored value is not empty.
