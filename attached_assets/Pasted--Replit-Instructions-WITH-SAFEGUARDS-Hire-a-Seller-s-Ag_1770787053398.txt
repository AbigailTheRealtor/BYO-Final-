# REQUIRED IMPLEMENTATION (Draft + Edit)

## Step 1 — Create a “Canonical Field Map” for these fields

In the Seller tenant flow, define the canonical Livewire properties and DB keys for each field:

**Must include:**

* `property_condition`
* `loan_type`
* `monthly_escrow_impounds` (not assumable, informational only)
* `loan_term_remaining_years`
* `loan_servicer_lender`
* `assumption_fee`
* `occupancy_requirement` (and `occupancy_requirement_other` if applicable)
* `acceptable_exchange_item`

✅ **Safeguard:** create a single array constant (or config) used in both save + load so keys can’t drift.

---

## Step 2 — Draft Save: ensure these fields are included in the payload

In the Seller Create/Draft component method that writes drafts (commonly `saveDraft()`, `saveAllMetadata()`, or similar):

✅ Add ALL listed fields to the draft payload.

**Safeguard**

* If your draft data is stored as `metadata` JSON, ensure these keys are included in that JSON.
* If stored in DB columns, ensure they are fillable and included in the `update()`/`create()` call.

**Merge-save safeguard**

* Use a field-by-field merge so saving a different tab doesn’t clear these.

---

## Step 3 — Draft Reopen: hydrate these fields on loadDraft()/mount()

When the user clicks a draft, the edit/draft component must set Livewire state from saved draft values.

✅ In `mount($draftId)` or `loadDraft($draftId)`:

* set each Livewire property from saved data
* ensure arrays stay arrays
* ensure booleans/empty values don’t become `"false"` or `false`

**Safeguard**
Normalize missing values:

* default to `null` or `''` (never `false`)

---

## Step 4 — Edit Save: ensure these fields are included in update payload

In Seller edit save method:

✅ Must write the same canonical keys back to storage:

* either metadata JSON merge
* or direct DB columns

**Safeguard**

* Do not have separate residential/commercial logic that excludes these fields.
* Do not rely on “only save current tab fields” — must merge save.
