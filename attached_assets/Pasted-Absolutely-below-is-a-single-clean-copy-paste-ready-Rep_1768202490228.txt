## A) “Other” OPTION — DISPLAY CLEANUP

### (View, Amenities, Property Features)

### Problem

When a user selects **Other** and enters custom text, the listing display:

* shows the literal word **“Other”**
* and/or inserts parentheses like `Other (Example Text)`

### Required Behavior

For the following fields:

* **View**
* **Amenities**
* **Property Features**

✅ Do **NOT** display the word **“Other”**
✅ Display **only the user’s custom “Other” text**, inline with other selections
✅ **No parentheses** should appear
✅ Format must match other multi-select fields that support “Other”

### Example (Correct)

**View:** Water View, Example View
❌ NOT: Water View, Other (Example View)

### Implementation Notes

For each field:

1. Take the selected options array
2. **Filter out** the literal value `"Other"`
3. If the custom Other text exists and is not empty, **append it as a normal value**
4. Render as comma-separated values (or your standard chip style), **without parentheses**
