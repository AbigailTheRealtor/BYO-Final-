If they come back with “values are empty”

Here are the two most common causes (so you can demand the right fix immediately):

Cause #1: Select/checkbox widgets aren’t syncing to Livewire

If Amortization Type / Payment Frequency are using a JS-enhanced select (Select2/etc.), Livewire often does not update unless explicitly wired.

Fix requirement:

Confirm those inputs are either:

plain <select wire:model="..."> without Select2, OR

Select2 “change” handler calls: @this.set('seller_amortization_type', value) etc.

Cause #2: Fields are inside conditionals that unmount before save

If a section unmounts (due to changing financing types or tabs), Livewire may lose values.

Fix requirement:

Add stable wire:key wrappers

Use wire:model.defer on all those inputs

Ensure tab navigation doesn’t reset those properties
