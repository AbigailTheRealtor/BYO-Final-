## CORE BUSINESS RULES

1. LISTING BASELINE
   When user creates a listing, the listing’s:

* Broker Compensation & Agency Agreement Terms
* Offered Services
* Additional Details where applicable

become the baseline source for agent bidding.

2. AGENT BID PRE-POPULATION
   When the agent opens the bid form, the bid form must pre-populate from the actual listing role/property-type structure.

Examples:

* Tenant Residential listing → Tenant Residential terms/services
* Landlord Commercial listing → Landlord Commercial terms/services
* Buyer Residential listing → Buyer Residential terms/services
* Seller Vacant Land listing → Seller Vacant Land terms/services

3. AGENT BID DISPLAY
   The exact terms/services the agent submits must be what renders in:

* Private Compensation & Agency Agreement Terms
* Offered Services
* Match Score / mismatch areas

No dropped fields.
No dropped services.
No stale values.
No remapped wrong options.

4. USER COUNTER PRE-POPULATION
   When user counters an agent bid, the form must pre-populate from the AGENT’S latest submitted bid/counter:

* Broker Compensation & Agency Agreement Terms
* Additional Terms
* Additional Details
* Offered Services

5. AGENT COUNTER PRE-POPULATION
   When agent counters a user counter, the form must pre-populate from the USER’S latest submitted counter:

* Broker Compensation & Agency Agreement Terms
* Additional Terms
* Additional Details
* Offered Services

6. LATEST RESPONSE RULE
   At every counter step, the system must always use the MOST RECENT opposite-party submission.
   Do NOT use:

* original listing
* first bid
* stale record
* mixed data from multiple records

7. ACCEPT / REJECT
   If accepted:

* move to acknowledgment / accepted bid summary flow
* final accepted terms must match the last accepted submission exactly

If rejected:

* state must update correctly without corrupting other bids
