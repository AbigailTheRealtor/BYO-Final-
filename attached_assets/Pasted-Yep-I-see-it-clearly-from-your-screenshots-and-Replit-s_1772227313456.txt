Those “collapsed” fields (Bedrooms, Bathrooms, Heated SqFt, Total Acreage) aren’t randomly “zero height.” What’s actually happening is:

## What’s really breaking the UI

Those fields are being **accidentally treated like Select2 fields**, so the underlying `<select>` gets **hidden** (Select2 hides the original select), but the Select2 UI container is **not being created or is being skipped** after the Livewire re-render.

Result: you see the label + icon, but the input area looks blank/collapsed.

This is almost always caused by one of these:

1. A global initializer like `$('select').select2()` (or `.form-select`) is being applied too broadly
2. A global CSS rule is hiding selects (ex: `select { height:0 }` or `visibility:hidden;height:0`) instead of only hiding Select2-managed selects
3. Guards are incorrectly “skipping” init because they check the wrong condition, so the new DOM select stays hidden
