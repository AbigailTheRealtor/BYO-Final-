Replit Instructions — Edit Bid Must NOT Clear Agent Info (Hire a Tenant’s Agent | Residential + Commercial)
✅ Problem

When an agent clicks Edit Bid and saves, the Agent Contact / Brokerage / phone / license / etc. fields get wiped, forcing re-entry.

✅ Required behavior

On Edit Bid, all previously saved agent info must:

pre-fill when the edit form loads, and

persist if the agent does not change it.

This is for Hire a Tenant’s Agent bids (Residential + Commercial).

🔒 SAFEGUARDS (MANDATORY)

Do NOT change field labels, tooltips, order, or UI.

Do NOT change DB schema.

Do NOT remove or rename any JSON keys.

Fix must be Tenant Agent only (Residential + Commercial).

Preserve all existing privacy rules.

1) Root cause to fix (most common)

The edit form submit/update logic is likely doing one of these:

rebuilding $payload = [] from only the fields present on the edit tab(s)

overwriting the entire get JSON with a partial payload

not loading the existing JSON into the edit component’s state

✅ Fix requirement: Merge existing bid JSON + new edits instead of replacing.

2) On Edit Bid LOAD: Pre-fill state from existing bid JSON

Where the edit form/component is initialized (controller edit() or Livewire mount()), you must load the existing bid JSON into the form state.

If using Livewire
public function mount($bidId)
{
  $this->bid = TenantAgentAuctionBid::findOrFail($bidId);

  $existing = $this->bid->get; // already decoded to array if json_decode(true) fix was applied
  if (is_string($existing)) $existing = json_decode($existing, true);
  if (is_object($existing)) $existing = (array) $existing;
  $existing = $existing ?: [];

  // IMPORTANT: hydrate ALL agent-info fields from existing JSON so they prefill
  $this->agent_phone = data_get($existing, 'agent_phone');
  $this->agent_email = data_get($existing, 'agent_email');
  $this->brokerage_name = data_get($existing, 'brokerage_name');
  // etc...

  // Also hydrate any arrays (social links, credentials, promo materials, etc.)
  $this->socialLinks = data_get($existing, 'socialLinks', []);
  $this->promoMaterials = data_get($existing, 'promoMaterials', []);
}

If using Controller + Blade

In edit($bidId), pass the decoded JSON to the view and set the input value="{{ old('field', $existing['field'] ?? '') }}"

3) On Edit Bid SAVE: Merge instead of overwrite (critical)
Required update logic

When saving edits, do NOT create a new JSON from scratch unless it includes ALL existing keys.

✅ Correct approach:

decode existing $bid->get into $existing

build $updates from submitted fields

$merged = array_replace_recursive($existing, $updates)

save $merged back to get

Example (Controller or Livewire)
$existing = $bid->get;
if (is_string($existing)) $existing = json_decode($existing, true);
if (is_object($existing)) $existing = (array) $existing;
$existing = $existing ?: [];

// Build ONLY what changed (or what is on the edit form)
$updates = [
  // example: compensation fields updated
  'tenantBrokerLeaseFlatFee' => $request->input('tenantBrokerLeaseFlatFee'),
  'tenantBrokerLeasePercent' => $request->input('tenantBrokerLeasePercent'),

  // IMPORTANT: DO NOT set agent fields to null/empty unless user actually changed them
];

// Merge so missing fields remain intact
$merged = array_replace_recursive($existing, array_filter($updates, fn($v) => $v !== null));

// If you MUST allow blanking fields intentionally, use a per-field rule instead of filtering everything globally.

$bid->get = $merged; // if cast to array/json, or json_encode($merged)
$bid->save();


✅ Key point: Never set agent info keys to empty just because that tab wasn’t edited.

4) Fix file: “Agent Info section not on edit step” (common cause)

If agent info fields are on a different tab and not included in the edit page, the update logic might still be clearing them.

✅ Requirement:

If the edit flow only submits a subset of tabs, the update must NOT overwrite keys from tabs not included.
