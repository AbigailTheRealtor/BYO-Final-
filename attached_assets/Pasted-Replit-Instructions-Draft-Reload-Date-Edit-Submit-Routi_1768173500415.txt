Expected Behavior

When a user saves a draft and reloads it:

City, County, State, ZIP should be pre-filled exactly as previously entered.

User should NOT have to re-enter them.

Required Implementation
1.1 Identify the source of truth

Confirm where these fields are stored on create:

Example: property_city, property_county, property_state, property_zip

Or: city, county, state, zip

Confirm what the draft loader reads from (often different key names or nested objects like location.city).

1.2 Normalize Draft Save + Load

Create a small mapping layer (do NOT rename fields):

On Save Draft: ensure these 4 fields are written into the draft payload AND persisted in DB draft JSON consistently.

On Load Draft: ensure these 4 fields are extracted back into the exact form state keys expected by the UI inputs.

1.3 Common Bug to Fix

Draft payload often stores:

county as an object {label,value}

or stores only value but UI expects {label,value}

or saves to listing.location.zip but UI reads listing.zip

Fix by converting shape both directions:

DB/Draft JSON shape ↔ UI component shape

Acceptance test

2) FIX “EDIT DATE IN DRAFT BREAKS SUBMIT” (BUYER + TENANT)
Expected Behavior

If a listing is in Draft and the user edits the date field:

Submit should still work

Likely Root Cause (what to check)
2.1 Date format mismatch after edit

The date picker often returns:

JS Date object

ISO string

localized string

or a timestamp

But backend expects a specific format, and when it receives the “new shape” it fails validation silently.

Required Implementation
2.2 Add a single canonical date serializer (no UI change)

At the point of Submit (and optionally Save Draft), normalize date fields:

Convert to a backend-approved string format consistently (example: YYYY-MM-DD or full ISO YYYY-MM-DDTHH:mm:ssZ depending on your schema).

Make this consistent across Buyer and Tenant flows.

2.3 DO NOT silently fail

If the backend rejects date:

Return a validation error and display it (toast / inline) instead of “nothing happens.”

Acceptance test

Tenant: Save Draft → reload → change date → Submit works

Buyer: Save Draft → reload → change date → Submit works
