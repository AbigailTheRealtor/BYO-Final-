SAFE PHASE 3 — Agent Profile Compensation Consistency Layer

GOAL:
Reduce compensation-option drift in Agent Profile Offer Presets by creating a safe, shared source of truth for compensation labels/options used in preset edit/display surfaces.

IMPORTANT:
This is Agent Profile / Offer Presets only.

DO NOT:
- Do not add MLS/tax/HOA/flood/legal/property fields.
- Do not touch Seller/Landlord Offer Listing intake forms.
- Do not touch accepted bid summaries unless only reading to confirm field names.
- Do not run destructive migrations.
- Do not rename existing DB columns.
- Do not delete saved compensation data.
- Do not change existing saved option values unless required to fix a known mismatch.
- Do not rewrite the full preset system.
- Do not change referral partner payout/admin logic.
- Do not change Phase 1 services configs unless a regression is found.

BACKGROUND:
Phase 1 completed:
- Services categorized and centralized through ServicesFormatter.
- Seller/Landlord/Tenant service order configs added.

Phase 2 completed:
- Preset save scope fixed.
- Custom services display correctly.
- Services display read-only on review/profile.
- Payment Timing for Broker Fees fixed.
- Referral Fee separated into agent-only tab.

KNOWN AUDIT FINDING:
Compensation field options are still hardcoded across multiple Blade files. Landlord has partial config support, but Buyer/Seller/Tenant options are still scattered. This creates drift risk.

Create a new shared config file unless an existing one is clearly better:

- buyer
- seller
- landlord
- tenant

Include only existing option labels/values currently used by the Agent Preset editor.

Do NOT invent new compensation types.
Do NOT change user-facing labels unless fixing an already-confirmed typo/mismatch.

The config should support:
- Residential
- Income
- Commercial
- Business
- Vacant Land
- Commercial Property / Residential Property where those labels are already used

TASK 3 — Replace hardcoded option arrays in Agent Preset edit view only

- Keep existing conditional display logic.
- Do not change database shape.

TASK 4 — Add helper methods if needed

If helpful, add small helper methods to AgentPresetController or a lightweight support class to retrieve:

- role compensation config

TASK 5 — Normalize only known mismatch risks

Fix only known safe mismatches discovered during audit, such as dropdown label/value mismatches that prevent conditional inputs from showing.

Do not broadly rename saved values.

If an old saved value is encountered that differs only by legacy text, support it as an alias rather than overwriting saved data.
