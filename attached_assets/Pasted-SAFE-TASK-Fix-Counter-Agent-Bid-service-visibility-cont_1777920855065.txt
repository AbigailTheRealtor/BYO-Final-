SAFE TASK — Fix Counter Agent Bid service visibility + contact/property info fields

GOAL 1 — Services should stay visible when unchecked:
Currently, when a user unselects a service, it disappears. Fix this.

Expected behavior:
- All Agent-proposed services should remain visible on the counter form.
- If the user unchecks a service, it should remain displayed but unchecked.
- On save/reload, unchecked services should still display as unchecked.
- Checked services should save as selected.
- Do not hide/remove services from the UI just because they are unchecked.
- Do not mutate the original Agent preset services.
- Treat the visible service catalog as the Agent’s originally proposed services.
- Treat checked/unchecked state as the counter selection.

Implementation guidance:
- Separate “available/proposed services” from “selected services.”
- The checkbox list should be rendered from the original proposed service list.
- The checked state should come from the current counter/request selected services.
- If selected services are empty only on first load, default to all proposed services checked.
- After the user saves, reload using saved selected services for checked state, while still rendering all proposed services.

GOAL 2 — Replace Buyer/Tenant Property Address section:
On Buyer and Tenant counter terms pages, do not show “Property Address” as the primary required section.

Instead show:

Section title:
Client Contact Information

Fields:
- Full Name
- Phone Number
- Email Address

Optional fields if easy and already supported:
- Preferred Contact Method
- Best Time to Contact

Rules:
- Buyer/Tenant may not have a specific property yet.
- Do not require Property Address for Buyer/Tenant in this counter flow.
- If an existing property address value exists, do not delete it from storage; just do not make it the main required field here.

GOAL 3 — Seller/Landlord should show contact info + property address:
For Seller and Landlord counter terms pages, show:

Section title:
Client Contact Information

Fields:
- Full Name
- Phone Number
- Email Address

Then show:

Section title:
Property Address

Fields:
- Property Address or existing address field
- City, State, ZIP if already supported

Rules:
- Keep Seller/Landlord property address because those roles are tied to a property.
- Do not remove existing address behavior for Seller/Landlord.

FIELD / SAVE SAFEGUARDS:
- Reuse existing columns/meta keys if they already exist.
- Do not create new schema unless no suitable storage exists.
- If using meta, use additive keys only.
- Preserve existing stored property address data.
- Do not overwrite unrelated listing/user/contact data.
- Validate email as email if validation already exists in this flow.
