# 1) Bug Fix — “Bathrooms: Other” text box shows even when user did NOT select “Other”

### Issue

In **Hire a Seller’s Agent → Property Details**, the **custom text input** for Bathrooms (“Other – specify”) is appearing automatically on load / by default, even though the user did not select **Other**.

### Expected Behavior

* The “Other” Bathrooms text box **must only appear** when the user selects **Bathrooms = Other**.
* If Bathrooms is changed away from Other, the “Other” text box must **hide**.
* If it hides, the system must **not keep stale Other text** unless Bathrooms = Other.

### Implementation Requirements

**A) Listing Creation (New Listing)**

1. Ensure Bathrooms has a clear default value that is **not** “Other” (ex: blank/null).
2. Render the **Bathrooms Other Text** input **only** when:

   * `bathrooms_selection == "Other"` (match the exact option value stored/used)
3. On initial form load:

   * If Bathrooms is null/blank/standard value → **Other textbox hidden**
4. When user selects Other:

   * Show the textbox immediately.
5. When user switches away from Other:

   * Hide textbox
   * Clear the “Other” textbox value in state (recommended) **OR** prevent saving it unless Bathrooms is Other (required at minimum).

**B) Listing Edit (Draft or Published Listing)**

1. When loading an existing listing:

   * If Bathrooms == Other → show textbox and populate it with saved value
   * If Bathrooms != Other → hide textbox (and ensure it does not “flash” visible on load)

**C) Saving + Display**

1. Saving rules:

   * Save `bathrooms_other_text` **only if** Bathrooms == Other
   * If Bathrooms != Other, store `bathrooms_other_text` as null/empty (or ignore in write logic)
2. Display rules:

   * If Bathrooms == Other and other_text exists → display the other_text
   * Otherwise display the selected Bathrooms option value as normal

### Acceptance Tests

* New listing → Bathrooms untouched → “Other” textbox never shows.
* Select Bathrooms = Other → textbox appears.
* Type text → save draft → reload → textbox shows and text persists.
* Change Bathrooms from Other to “2” (or any standard value) → textbox hides; save → reload → textbox remains hidden and “other” text is not displayed anywhere.
