For the agent dashboard Hire Agent listing flow only, replace the role-specific client information tab with an agent-specific credentials tab.

Objective

Agent-created Hire Agent listings should not use:

Seller Information
Buyer Information
Landlord Information
Tenant Information

Those labels are for the regular user/client dashboard.

For agent-created Hire Agent listings only, use an agent-specific tab and section instead.

Required changes
1. Rename the final info tab

For the agent dashboard Hire Agent create/edit flow only, change the role-specific information tab label to:

Agent Credentials & Contact Info

2. Rename the section heading inside the tab

Use the same heading inside the tab:

Agent Credentials & Contact Info

3. Replace/add fields in this tab

For the agent dashboard Hire Agent flow only, this tab should contain:

First Name
Last Name
Phone Number
Email
Brokerage
Real Estate License #
NAR Member ID (NRDS ID)
4. Intro/help text

Add a short description like:

“Provide your basic contact and license information. This section helps the other party verify your credentials and understand who they’re working with.”

5. Scope restriction

This change must apply only to the agent dashboard Hire Agent flow.

Do not change the regular user/client dashboard tabs, which should remain role-based:

Seller Information
Buyer Information
Landlord Information
Tenant Information
6. Listing/detail view consistency

Where agent-created Hire Agent listings or related forms display this section, use the same agent-specific naming and fields consistently.

Audit first

Before changing code, audit and report:

which files control the final info tab for agent-created Hire Agent forms
whether this tab is currently shared with user-side forms
what existing agent profile/license fields already exist and can be reused
safest way to keep user-side role tabs untouched
Safeguards
No DB renames
No DB deletes
No breaking user-side listing creation flows
No broad shared-form rewrite unless necessary
Reuse existing agent profile/license fields if available
Stop and report if the current tab/form is too tightly shared with user-side logic and needs a minimal scoped separation first
Deliverable

After implementation, report:

files audited
files changed
whether the change is agent-dashboard-only
final tab label used
final fields shown
whether existing agent profile/license data was reused
confirmation that user-side role information tabs were untouched
