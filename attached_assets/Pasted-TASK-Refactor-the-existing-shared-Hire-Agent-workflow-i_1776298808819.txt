TASK: Refactor the existing shared Hire-Agent workflow into a MULTI-MODE WORKFLOW ENGINE that supports both:

1. Hire Agent mode
2. Offer mode

AND update the Agent Dashboard sidebar/navigation so the OLD legacy listing flows are removed from the UI and replaced by the NEW workflow structure.

GOAL
We want to preserve the existing shared Hire-Agent system exactly as it works now, while making the underlying workflow reusable for a second business flow: Offer / Lease / Contract negotiation.

This is NOT a request to duplicate the Hire system.
This IS a request to abstract the reusable workflow infrastructure so the dashboard can support multiple workflow types with shared lifecycle logic and mode-specific fields.

We also want to clean up the dashboard/sidebar so the OLD listing flows are no longer shown and the NEW workflow structure is what users see.

--------------------------------------------------
CORE ARCHITECTURE RULE (CRITICAL)
--------------------------------------------------

DO NOT duplicate the entire Hire flow to create an Offer flow.

We want:
- one shared workflow engine
- shared infrastructure
- mode-specific field sets
- mode-specific labels
- mode-specific negotiation logic
- mode-specific summaries

DO NOT:
- clone Livewire components into full separate systems if avoidable
- clone timer logic
- clone draft/publish/edit infrastructure
- clone accept/counter/reject framework
- clone dashboard management pages
- clone summary engine if a shared summary framework can be used

If the proposed implementation would create two systems that drift apart, STOP and report.
