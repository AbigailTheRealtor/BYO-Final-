#941 - Restore Hire Seller Agent Search + Card + View Parity

IMPORTANT: Before editing, do a read-only audit first and confirm:
1. The exact save/submit methods in SellerAgentAuction.php.
2. Whether workflow_type = 'hire_agent' is missing.
3. Whether any Seller Offer Listing meta keys are being saved by the Hire Seller Agent flow.
4. The exact files that will be changed.

Apply only the smallest surgical fix needed.

DO NOT modify:
- routes
- migrations
- database schema
- validation rules
- Limited Service files/flows
- Buyer/Landlord/Tenant hire flows
- Seller Offer Listing architecture

What & Why
A regression occurred after Seller Offer Listing work was introduced. Newly created "Hire Seller's Agent" listings are not appearing under /search/seller-agent-needed, and may be rendering with the wrong card or detail view format. The root cause is that the Hire Seller Agent Livewire component does not stamp workflow_type = 'hire_agent' on save, and it may be inadvertently saving meta keys that belong exclusively to Seller Offer Listings (e.g. parcel_id, flood_zone_code, annual_property_taxes, seller_disclosure_available). The search query and detail page routing both rely on these signals to distinguish hire listings from offer listings — so without correct stamping, hire listings are silently excluded from hire search results and/or redirected to the wrong detail view.

This is a restoration task only. No architecture changes, no schema changes, no Limited Service changes.

Done looks like
- A newly created Hire Seller Agent listing appears under /search/seller-agent-needed
- The listing card on that page matches the Buyer / Landlord / Tenant hire agent card style (not Seller Offer Listing style)
- Opening a Hire Seller Agent listing loads hire_seller_agent/view.blade.php, not the offer listing detail view
- Seller Offer Listings are completely unaffected
- Buyer / Landlord / Tenant hire flows are unaffected

Out of scope
- Database schema or migration changes
- Route changes (unless a route is provably wrong)
- Redesigning or restructuring the Hire Seller Agent flow
- Any Limited Service legacy flows
- Offer Listing architecture

Steps

1. Audit — confirm the root cause
Read app/Http/Livewire/HireSellerAgent/SellerAgentAuction.php (and any related save/submit methods) to verify whether workflow_type = 'hire_agent' is being stamped on create/save. Check whether any Seller Offer Listing meta keys (from SellerOfferListingController::OFFER_LISTING_META_KEYS) are being saved by the hire form. Compare with how the Buyer, Landlord, and Tenant hire Livewire components stamp workflow_type.

2. Stamp workflow_type = 'hire_agent' on save
In the Hire Seller Agent Livewire component's save and submit methods, add a saveMeta('workflow_type', 'hire_agent') call matching the other hire flows. Ensure this runs for both draft saves and final submissions.

3. Remove any accidental offer-listing meta keys
If the Hire Seller Agent component is saving any keys from OFFER_LISTING_META_KEYS, remove those saves. These keys must only be written by the Seller Offer Listing flow.

4. Verify search query exclusion logic
In SellerAgentAuctionController::searchListing(), confirm the whereDoesntHave exclusions for workflow_type = 'offer_listing' and OFFER_LISTING_META_KEYS are correctly structured so that Hire Seller listings with workflow_type = 'hire_agent' and no offer meta keys pass through cleanly. Fix any overly broad exclusion that might reject hire listings.

5. Verify detail page routing
In SellerAgentAuctionController::viewDetail(), confirm the offer-listing redirect logic (workflow_type check + OFFER_LISTING_META_KEYS fallback check) does not fire for Hire Seller listings.

6. QA pass
Create a new Hire Seller Agent listing and verify:
(a) it saves with workflow_type = 'hire_agent'
(b) it appears in /search/seller-agent-needed
(c) the search card matches Buyer/Landlord/Tenant hire card style
(d) the detail page loads hire_seller_agent/view.blade.php
(e) Seller Offer Listings are unaffected
(f) Buyer/Landlord/Tenant hire listings still work unchanged
