#813 - Fix Seller HOA/Association Other Field Layout

What & Why

On /offer-listing/seller, the HOA / Association section has two visual bugs when "Is there an HOA?" is set to Yes and "Association Type" is set to Other:

1. An orphaned, unlabeled Select field appears between Association Name and the Fee Amount / Fee Frequency row.
2. Some icons in the HOA section render incorrectly — standalone icons appear outside their input-cover wrappers, or icons are duplicated around certain fields.

These are display-only bugs in the Seller Full Service Offer Listing blade files only.

Scope restrictions

ONLY touch:
- Seller Full Service Offer Listing blade files

DO NOT touch:
- Buyer, Tenant, Landlord, or Hire Agent flows
- Limited Service flows
- Livewire component logic unless absolutely required for persistence verification
- Database schema
- Validation
- Routes
- Controllers

Done looks like

- Setting Association Type = Other shows only one custom text input:
  "Describe the association type"
- No extra unlabeled Select appears beneath Association Name
- Association Name is followed directly by the Fee Amount / Fee Frequency row
- All HOA section icons render inside proper input-cover wrappers
- No standalone <i> icon tags appear in the HOA section
- association_type_other and association_fee_frequency_other use the same wrapper pattern as other text inputs
- Save Draft + reload still preserves:
  - association_type
  - association_type_other
  - association_name
  - association_fee_frequency
  - association amenities fields
- No regressions outside Seller Full Service Offer Listing

Steps

1. Audit:
resources/views/livewire/offer-listing/offer-seller-tabs/commission-based/tax-legal-hoa-disclosures.blade.php

Focus on the HOA / Association section around:
- Association Type
- Association Name
- Fee Amount / Fee Frequency

2. Remove orphan field:
Find the stray unlabeled input-cover/select block rendering a standalone “Select” field between Association Name and the Fee Amount / Fee Frequency row.

Remove ONLY the orphaned unlabeled block.
Do not remove:
- Association Type
- Association Type Other field
- Association Name
- Fee Amount
- Fee Frequency
- Any wire:model bindings

3. Fix icon rendering:
Audit all fields inside the HOA section.

Ensure every icon field uses:
<div class="input-cover has-icon" data-icon="...">

Remove standalone <i> tags outside wrappers.

Do NOT add data-icon to Select2/multiselect fields.

Ensure conditional “Other” text inputs use the same wrapper pattern as normal text inputs.

4. Verify persistence:
Confirm Save Draft + reload still preserves HOA values correctly.

Return:
- exact file(s) changed
- exact orphan block removed
- confirmation no non-seller files were modified
