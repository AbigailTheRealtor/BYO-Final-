==================================================

Current issue:
Uploading larger documents triggers PostTooLargeException before Laravel can process the request.

Required fixes:

1. Increase PHP upload limits to a safe launch-ready size
   Target minimum:
   - upload_max_filesize = 20M
   - post_max_size = 25M

   If the environment already supports higher safe values, that is acceptable.

2. Confirm Laravel validation also allows the same intended size
   Example target:
   - PDF / JPG / PNG allowed
   - max 20MB per file

3. Ensure oversized uploads do NOT fail silently
   Add graceful validation or user-facing error handling:
   - “File exceeds 20MB limit”
   - no blank page / no raw exception page in normal UX

4. If the stack needs a custom handler for PostTooLargeException, add one so users see a normal validation-style error instead of a framework exception page.

IMPORTANT:
Do not set tiny limits like 2MB or 5MB unless there is a hard technical reason.
The launch-ready experience should support normal PDFs and images.
