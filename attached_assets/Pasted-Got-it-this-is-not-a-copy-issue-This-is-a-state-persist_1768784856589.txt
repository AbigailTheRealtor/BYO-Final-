# ✅ FIX PART 2 — SAVE / DRAFT LOGIC (DATA IS BEING ERASED HERE)

## 🔴 PROBLEM

When you click **Save Draft** or switch tabs, Livewire is **re-initializing arrays**.

---

## ✅ REQUIRED CHANGE

### 🔧 In `saveDraft()` and `save()`

❌ REMOVE anything like:

```php
$this->purchasing_terms = [];
```

❌ REMOVE conditional resets like:

```php
if (!$this->buyer_preapproved) {
    $this->buyer_preapproval_amount = null;
}
```

---

### ✅ INSTEAD — ONLY MAP WHAT EXISTS

```php
$listing->update([
    'acceptable_special_sale_provisions' => $this->purchasing_terms['acceptable_special_sale_provisions'] ?? [],
    'buyer_open_to_assignment' => $this->purchasing_terms['buyer_open_to_assignment'],
    'assignment_fee_to_broker' => $this->purchasing_terms['assignment_fee_to_broker'],
    'target_closing_timeframe' => $this->purchasing_terms['target_closing_timeframe'],
    'maximum_budget' => $this->purchasing_terms['maximum_budget'],

    'offered_financing' => $this->purchasing_terms['offered_financing'] ?? [],
    'buyer_preapproved' => $this->purchasing_terms['buyer_preapproved'],
    'buyer_preapproval_amount' => $this->purchasing_terms['buyer_preapproval_amount'],

    // continue explicitly — no loops, no resets

```

🔒 **Safeguard:**
Never null a value unless the user **explicitly changed it**.
