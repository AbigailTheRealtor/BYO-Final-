==================================================
PART 1 — FIRST NAME / LAST NAME DISPLAY
==================================================

Issue:

- First Name
- Last Name

But review/submitted sections still show a combined “Name” field or omit the name fields entirely.

Required behavior:
Every Direct Hire “Your Details,” “Your Submitted Information,” review, submitted, and acknowledge display must show:

- First Name
- Last Name

Do NOT show a combined “Name” label when first_name and last_name are available.

Apply to:
- Buyer Direct Hire
- Seller Direct Hire
- Landlord Direct Hire
- Tenant Direct Hire

Requirements:
1. Audit Direct Hire detail display logic/partials.
2. Replace combined “Name” display with separate:
   - First Name
   - Last Name
3. Ensure all four roles show First Name and Last Name in:
   - counter Review / Your Details
   - submitted / Your Submitted Information
   - acknowledge page if applicable
4. Preserve existing styling/layout/grid spacing.
5. Do not rename database fields or change saved data structure.
6. If old submissions only have a combined name fallback, keep a safe fallback only when first_name/last_name are unavailable — but do not show “Name” when separate values exist.

==================================================

- -- Select --
- -- Select option --
- Select option
- Choose
- Choose one
- Select flexibility
- Select Desired Lease Term
- role-specific placeholder wording

Required behavior:
All Direct Hire select/dropdown placeholders should match the listing creation standard:

Requirements:
1. Audit all select fields in Direct Hire Blade files for Buyer, Seller, Landlord, and Tenant.

5. Do not change labels, tooltips, names, IDs, bindings, validation, saved values, or conditional logic.
