FIX-01 — Preserve user_type during draft load
--------------------------------------------------

File:
app/Http/Livewire/TenantAgentAuction.php

Problem:
loadDraft() overwrites $this->user_type from EAV metadata:
$this->user_type = $auction->get->user_type;

If EAV metadata is missing/null, the entire flow breaks.

Fix:
- Preserve the route-derived user_type as the source of truth.
- Only use EAV user_type if it exists and is valid.
- Never allow $this->user_type to become null during draft load.
- Add a safe fallback to the original route parameter.

Done looks like:
- Draft resume works again for all 4 roles.
- Correct tabs render after resume.
- Correct validation branch executes.
- No null user_type state exists.
