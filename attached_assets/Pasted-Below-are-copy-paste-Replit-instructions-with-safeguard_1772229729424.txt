# 1) ALL PROPERTY TYPES: “No Preferences (open to any condition)” not saving

## Field

**Acceptable Property Conditions → No Preferences (open to any condition)**

## Symptoms

* Does not save on creation
* Does not save on edit
* Likely also not persisting in drafts

## Fix Requirements

### A) Ensure the value is included in saving

* Identify the Livewire property/meta key used for “Acceptable Property Conditions” (example: `acceptable_property_conditions` or legacy key like `condition_prop_buyer`)
* Confirm the “No Preferences” option has a stable stored value (recommended: `no_preference`)

**Save rule:**

* If user selects “No Preferences”, it must be included in the stored array/value exactly as selected.

### B) Ensure edit/draft hydration restores it

* In draft/edit load (mount/loadDraft/loadListingData), ensure the property is set from meta into Livewire state.
* For Select2 + wire:ignore fields, ensure JS restores the values on init:

  * `$('#field').val(@this.get('field')).trigger('change.select2')`

### C) Guardrail (prevents common conflict)

If “No Preferences” is intended to be exclusive:

* When `no_preference` is selected, automatically clear other selected conditions
* When any other condition is selected, automatically remove `no_preference`

(If you do NOT want exclusivity, skip this part.)

**Must work in Create + Draft + Edit.**

Make sure these 3 Income fields are saved and restored everywhere:

2. **Acceptable Number of Units**
3. **Acceptable Unit Type** (multi)

✅ Confirm each appears in:

### D) Actual listing display

# 5) VACANT LAND: Acceptable Property Style “Other” text doesn’t save

## Symptom

User selects **Other** and types text, but it doesn’t persist.

## Fix Requirements

### A) Ensure there is an “Other” text meta key saved

* Confirm meta keys:

  * `acceptable_property_style` (selected value or array)
  * `acceptable_property_style_other` (typed text)

### B) Save normalization (prevents mismatch)

Right before save:

* If other text is non-empty → ensure selected value includes `Other` (or `other`, whichever your system uses)
* If other text is empty → clear the other meta key and remove `Other` from selections (optional)

### C) Draft/edit hydration

* Ensure `acceptable_property_style_other` loads back into the input on edit/draft.

### D) Listing display

* If selection includes Other and other text exists → display the other text (not the word “Other”)
* If other text exists but Other isn’t in selection (legacy edge case) → append other text anyway
