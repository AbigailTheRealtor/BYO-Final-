## D) FIX: Draft Save/Reload must persist missing fields

### Goal

When user **Save Draft** then opens the draft again, these values must still be populated:

* City / County / State / Zip
* Property Condition
* Monthly Escrow / Impounds
* Loan Term Remaining (Years)
* Date Loan Originated
* Loan Servicer / Lender
* Assumption Fee
* Occupancy Requirement

### Root causes (usually one or more)

1. Fields exist in UI but **not included** in draft save payload/validation
2. Livewire uses different property names than DB columns
3. Draft “load” method does not hydrate these fields back into Livewire state
4. Some fields are conditionally stored only on submit, not draft
5. Fillable/cast missing on model

### Step-by-step fix checklist

#### 1) Database / Model

* Confirm each field has a DB column.
* Confirm Seller listing model has them in `$fillable` (or handled explicitly in an update array).
* Add casts where appropriate:

  * dates: `date_loan_originated`
  * numeric: `monthly_escrow_impounds`, `loan_term_remaining_years`, `assumption_fee`

✅ **Safeguard:** If a column does not exist yet, create a migration that **adds** it (do not rename anything).

#### 2) Livewire state properties

Ensure the Livewire component has a property for each field, example:

* `$city`, `$county`, `$state`, `$zip`
* `$property_condition`
* `$monthly_escrow_impounds`
* `$loan_term_remaining_years`
* `$date_loan_originated`
* `$loan_servicer_lender`
* `$assumption_fee`
* `$occupancy_requirement`

#### 3) Draft Save method

In the method that runs on **Save Draft**, ensure these fields are included in the update/create array.

✅ **Must** be included both when:

* creating a new draft row
* updating an existing draft row

#### 4) Draft Load method (when user clicks a draft)

Wherever the component loads an existing draft (e.g., `mount($id)` or `loadDraft($id)`):

* Set Livewire properties from the model:

  * `$this->city = $listing->city;` etc.

**Common miss:** devs hydrate only “headline” fields and forget finance/assumption section.

#### 5) Blade: ensure `wire:model` binds to the correct Livewire property

Example mismatches that cause “not saved”:

* Input uses `wire:model="loan_term_years"` but save uses `$loan_term_remaining_years`
* Date input uses `wire:model="loan_originated"` but DB column is `date_loan_originated`

Fix by making names consistent OR mapping them explicitly in save/load arrays.
