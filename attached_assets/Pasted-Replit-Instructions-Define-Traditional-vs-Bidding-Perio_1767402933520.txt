1) DEFINITIONS — Listing Type Rules
A) Traditional Listing
Expected behavior:

No timer is shown anywhere on the listing.

Agent bids are private to other agents:

Agents cannot see other agents’ services or compensation summaries.

Agents can only view their own bid details.

Only the listing creator can see full bid details for every agent:

Full bid details display under Private Compensation and Agency Agreement Terms (or your existing private terms section).

Listing creator can take action anytime:

Accept / Counter / Reject buttons are available immediately after bids are received (no time restriction).

B) Bidding Period Listing (Timer Active)
Expected behavior:

The listing must show a countdown timer using the user’s input:

“Bidding Period” duration = the number of days entered at creation (and should show on listing).

Listing display should show remaining time in Days / Hours / Minutes / Seconds (like your screenshot).

The listing creator CANNOT accept/counter/reject until the timer ends:

Accept / Counter / Reject buttons must be disabled or hidden until the timer reaches 0.

When timer ends, Accept/Counter/Reject becomes active.

Agents CAN see other agents’ bid summaries (anonymous, no identity):

Agents can see:

Offered Services: X Services

Broker Compensation Summary

Tenant’s Broker Commission Structure

Tenant’s Broker Commission Fee

Agents can click:

“View Full Services & Broker Compensation Terms”

This shows the full list of services and the full Broker Compensation & Agency Agreement Terms that other agents entered

Still anonymous (Agent 1, Agent 2, etc. — no name/brokerage/contact)

The listing creator can always see full bid details (same as Traditional), but cannot act until timer ends.

2) IMPLEMENTATION REQUIREMENTS (WHAT TO BUILD / ADJUST)
2.1 Timer display logic (Listing View)
For Tenant listings ONLY:

If listing_type (or your field name) = Traditional:

Do not render the timer UI block at all.

If listing_type = Bidding Period:

Render timer UI block.

The duration must come from the user’s input field: bidding_period_days (or your existing key).

Countdown must be based on:

listing_created_at + bidding_period_days

Display remaining time in four blocks:

Days / Hrs / Mins / Secs (match current UI)

✅ Do not hardcode 4 days; use the listing’s saved value.

2.2 Listing creator action gating (Accept/Counter/Reject)
In the “Actual Bids Display” where the listing creator sees bids:

If Traditional:

Accept / Counter / Reject buttons are always enabled.

If Bidding Period:

Accept / Counter / Reject buttons must be disabled/hidden until the timer ends.

“Timer ended” condition:

now() >= listing_created_at + bidding_period_days

UI behavior (pick one—keep existing style)

Either:

Hide the buttons completely until timer ends, OR

Show them disabled with tooltip text:

“Actions unlock when the bidding period ends.”

✅ Whichever is easier, but MUST prevent action until end.
