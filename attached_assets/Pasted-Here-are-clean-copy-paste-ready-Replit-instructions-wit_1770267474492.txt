## PURPOSE

Fix Personal Photo upload behavior so that:

1. A **Personal Photo persists when a draft is saved and reloaded**
2. The user can **delete/remove a previously uploaded Personal Photo** at any time
3. Behavior is **consistent across all listing types**

---

# 1) SAFEGUARDS (NON-NEGOTIABLE)

1. **No destructive database changes**

   * Do NOT rename tables or columns
   * Do NOT delete existing stored images
2. **No text or label changes**

   * Do NOT change “Personal Photo” label, tooltip, or instructions
3. **Backward compatible**

   * Existing drafts with photos must continue to work
4. **Scoped to Personal Photo only**

   * Do NOT affect other uploads (documents, IDs, attachments) unless they share the same component intentionally
5. **No forced re-upload**

   * User must not be required to re-upload a photo after saving a draft

---

# 2) FIX #1 — Personal Photo MUST Persist When Draft Is Saved

### Issue

When a user uploads a **Personal Photo** and clicks **Save Draft**, the photo is lost when the draft is reopened.

### Required Behavior

* Uploaded Personal Photo must:

  * Be saved immediately (or on draft save) to storage
  * Be associated with the draft record
  * Re-render automatically when the draft is reopened

### Implementation Requirements

* Ensure Personal Photo file path/ID is stored in the **draft listing record**
* On draft load:

  * If a Personal Photo exists → show preview/thumbnail
  * Do NOT require re-upload
* Must work for:

  * Draft → reload
  * Draft → edit → save → reload
  * Draft → submit

### Safeguard

* If a draft has **no photo**, do not show broken previews or errors
* Null-safe rendering required

---

# 3) FIX #2 — User MUST Be Able to Delete the Personal Photo

### Issue

Once a Personal Photo is uploaded, user cannot remove it.

### Required Behavior

Add a **Delete / Remove Photo** option that:

* Is clearly visible once a photo exists
* Allows user to remove the photo before submitting OR while editing a draft

### Delete Action Requirements

When user clicks **Delete / Remove Photo**:

1. Remove photo preview from UI immediately
2. Clear the Personal Photo reference from the listing/draft record
3. Do NOT crash if user deletes then saves draft
4. Do NOT delete unrelated files

### Storage Handling (Safe)

* Preferred:

  * Soft delete / disassociate the file
  * Optionally clean up orphaned files later via background job
* Do NOT hard-delete files synchronously if risky

---

# 4) REQUIRED UI STATES

The Personal Photo field must support **all 4 states** cleanly:

1. **No photo uploaded**

   * Show upload input
2. **Photo uploaded (new)**

   * Show preview + Delete option
3. **Draft reopened with photo**

   * Show preview + Delete option
4. **Photo deleted**

   * Return to upload state
