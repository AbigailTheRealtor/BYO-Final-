Audit and standardize input box sizing across Hire Agent Listings and Create Offer Listings.

Scope:
- Hire Agent Listings: Buyer, Seller, Landlord, Tenant flows.
- Create Offer Listings: Buyer, Seller, Landlord, Tenant flows.
- Include create, edit, draft, and counter/review form areas where the same field components are reused.
- Do not change Limited Service legacy flows unless the same shared component is clearly used by the active Full Service flow.

Goal:
Make all standard input boxes visually consistent and aligned across roles and workflows, while preserving intentionally larger fields such as Description, Additional Details, Notes, Terms, Remarks, About/Marketing text, and other long-form textareas.

Audit requirements:
1. Find every regular text input, number input, date input, currency input, percentage input, phone/email input, select, and Select2 field in:
   - Hire Agent listing forms
   - Create Offer Listing forms
   - Edit/Draft versions of those forms

2. Confirm standard fields use the same height, padding, border radius, font size, icon alignment, and placeholder alignment.

3. Confirm fields with icons have consistent left padding and the icon does not overlap the placeholder, typed value, Select2 placeholder, or selected values.

4. Confirm Select2 single-select and multi-select boxes visually match regular input height unless they are intentionally multi-line because selected pills wrap.

5. Confirm true textarea/long-form fields are intentionally larger and not forced into regular input height. Examples:
   - Description
   - Additional Details
   - Additional Terms
   - Notes
   - Property Description
   - Marketing Remarks
   - Special Instructions
   - Other long-form explanation boxes

6. Confirm standard “Other” text inputs use the same regular input size as other single-line fields, not oversized textarea styling.

7. Compare the same field types across all roles:
   - Buyer vs Seller vs Landlord vs Tenant
   - Hire Agent vs Offer Listing
   - Create vs Edit/Draft

8. Fix any inconsistent box sizing at the shared CSS/component level where safe. Avoid one-off inline fixes unless the inconsistency is isolated.

9. Do not change labels, placeholders, tooltip text, database fields, validation, routes, controllers, or saved data behavior unless directly required to correct the visual box-size issue.

10. Do not reduce larger textarea fields that are intentionally designed for long-form content.

Done looks like:
- Regular input boxes are the same visual height across Hire Agent and Create Offer Listing flows.
- Select fields and Select2 fields align with regular inputs.
- Icon spacing is consistent across all fields.
- “Other” text inputs are regular single-line boxes.
- Description / Additional Details / Notes / Terms / Remarks remain larger textarea-style fields.
- Create, Edit, and Draft versions match.
- No Livewire behavior, saved data, validation, routes, or field names are broken.

After changes, provide:
1. List of files changed.
2. Summary of inconsistencies found.
3. Confirmation that regular inputs, Select2 fields, icon fields, and textarea fields were checked separately.
4. Confirmation that larger long-form boxes were preserved intentionally.
5. Before/after notes for any shared CSS or Blade component changes.
