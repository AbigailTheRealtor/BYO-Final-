Replit Instructions — Accepted Bid Summary + Inline E-Sign (Same Page)
Applies To

Hire a Tenant’s Agent (Residential + Commercial)

Action: User selects “Accept” on an Agent Bid

Page: Accepted Bid Summary (single page)

E-Sign: Inline at bottom of summary (scroll-to, not new page)

SAFEGUARDS — MUST FOLLOW

DO NOT:

Create, remove, or redirect to any separate e-sign page.

Change routing or URLs for Accepted Bid Summary.

Delete or rename database tables, migrations, or existing columns.

Rename any existing field keys used in the bid summary or signatures.

Modify Buyer, Seller, or Landlord flows.

Change approved legal/disclosure language text (only placement/combination allowed).

Alter global layout or shared components outside this page.

DO:

Scope all changes strictly to Hire a Tenant’s Agent → Accepted Bid Summary.

Reuse existing signature/audit fields where possible.

Keep all changes backward-compatible.

REQUIRED BEHAVIOR (FINAL)
1) “Accept” Action Behavior

When a Tenant clicks Accept on an Agent Bid:

Record Accepted Date/Time (server-side, stored in UTC)

Redirect user to the Accepted Bid Summary page

Do NOT prompt for signature at this moment

The summary page must:

Display full accepted bid details

Show signature status (Pending / Signed)

Allow inline e-sign at the bottom

2) “Sign Now” Button Behavior

“Sign Now” does NOT navigate to another page

It scrolls the user to Section 8: Signature Acknowledgment

No routing changes required

3) Accepted Date vs Signed Date (KEEP BOTH)
Accepted Date (Top of Summary)

This represents when the Tenant accepted the bid.

KEEP THIS FIELD, but update formatting:

Replace

Accepted Date: 2026-01-04 07:02:18


With

Accepted Date: Jan 4, 2026 at 2:46 AM ET


Rules

Use Month / Day / Year

12-hour time

Display timezone of the user who accepted (ET, PT, etc.)

Source = bid acceptance timestamp

Signed Date (Bottom – Signature Section)

This represents when each party acknowledged the summary.

Keep signature date/time in Section 8 only

Do not remove Accepted Date because:

Acceptance ≠ acknowledgment

Both are legally and audit-useful

4) Inline E-Sign (Bottom of Same Page)
Fields Shown to User

Only display:

Type Your Full Legal Name (required)

Single required checkbox with combined text:

Important: By signing below, you acknowledge receipt and review of this Accepted Bid Summary. This is an acknowledgement only, not a contract execution. I confirm that I have reviewed the Accepted Bid Summary and acknowledge its contents.

Sign button:

Tenant view: “Tenant: E-Sign Acknowledgment”

Agent view: “Agent: E-Sign Acknowledgment”

Fields NOT Shown in UI

Do NOT show:

Timestamp input

Timezone field

IP address field

These are recorded automatically in the background.

5) Signature Data to Record (Background Only)

When a user signs, automatically capture:

Full legal name (typed)

Role (Tenant or Agent)

Signed date/time (UTC)

User timezone (IANA if available)

IP address

None of these need to be editable or visible to the user.

6) Signature Acknowledgment Display Cards (Section 8)
Before Signing

Show:

Signature: —
Date/Time: Pending
IP Address: —

After Signing

Show:

Signature: [Typed Full Legal Name]
Date/Time: Jan 4, 2026 at 8:29 PM ET


Do NOT display the IP address publicly.
(Keep it stored for audit/admin use only.)

7) Download PDF Button Logic
Button Visibility

Always visible at top of summary

Behavior Rules

If both Tenant and Agent have signed:

Enable download

Serve the finalized PDF

If either party has not signed:

Block download

Show message:

“PDF will be available once both parties have signed.”

8) PDF Generation Rules

Generate the final PDF only once, immediately after the second signature

Store the generated PDF

Reuse stored file for all future downloads

Do not regenerate on every click

9) Admin Copy Requirement

After both parties have signed:

Store a copy of the final PDF

Make it accessible in Admin’s account/dashboard

Associate it with:

Listing ID

Tenant

Agent

Acceptance date

Signature completion date

(No new admin UI required if admin already has document access.)

10) “Other” Field Display Fix (Required)

If Tenant Agency Agreement Timeframe = Other:

Display the user-entered custom text

Do NOT display the word “Other”

Apply this rule:

On Accepted Bid Summary

On generated PDF
