# 🚨 TASK: Standardize Agent Bid Forms + Add Default Profile System + Per-Bid Autofill Toggle + Default Profile Dashboard

Use the **Hire a Tenant Agent bid form** as the **source-of-truth format** and apply the same:

* general UI consistency

to all agent bid forms for:

* Hire a Tenant Agent

* Hire a Seller Agent
* Hire a Buyer Agent

## Required behavior

* allow the agent to choose **Use Default** or **Skip Autofill**
* if **Use Default** is chosen, populate the profile-related sections from the matching saved default
* if **Skip Autofill** is chosen, leave those profile-related sections blank unless existing system logic already pre-fills something else

## Important safeguards

* this toggle applies to the default profile autofill, not to listing-derived compensation fields
* listing-derived compensation fields should still follow their own load rules
* the toggle should be available only when relevant, meaning when a matching default exists for that role + property type
* once the bid already has saved values, do not keep re-asking or reapplying defaults

## UX note

This can be implemented as either:

* an initial prompt/banner/section on first load, or
* a clear toggle/button row near the top of the bid form

But it must be easy to understand and must not clutter the form.

# STEP 15 — AUTO-FILL PRIORITY ORDER

When loading a bid form, populate data in this order:

## Priority 1 — Existing saved bid record

If the bid already exists and has saved values, use those first.

## Priority 2 — Listing-derived values

For compensation/agreement terms from the original listing, use listing values only if the bid fields are still empty.

## Priority 3 — Saved default profile

For profile-related sections, use the agent’s saved default profile for that exact role + property type only if:

* the bid fields are still empty, and
* the agent has chosen Use Default or default application is the selected first-load action for that new bid

## Priority 4 — Blank

If nothing exists, leave the field blank.

# STEP 16 — NEVER OVERWRITE EXISTING SAVED BID DATA

This is critical.

If the bid already contains saved data:

* do not replace it with default profile values
* do not replace it with listing-derived values
* do not re-autofill after the user edits fields
* do not let Livewire re-renders overwrite user-entered data

## Rule

Saved bid data always wins.

# STEP 17 — LIVEWIRE / RE-RENDER SAFEGUARD

Autofill must not run on every render.

## Only run autofill when:

* component mounts for first load, and
* the bid does not already have saved data for those fields, and
* the logic determines the new-bid autofill decision for that one initial load

## Do NOT:

* run autofill inside `render()`
* run autofill on every tab change
* run autofill inside generic re-render hooks that can fire repeatedly
* reapply defaults after user input

## Required guard logic

Use a clear check such as:

* if existing bid data is present → do nothing
* else if fields are empty and first-load conditions are met → apply listing/default autofill once

The core requirement is:
**autofill must be first-load only and never overwrite saved or edited bid data**
