# WHAT TO FIX

## PART A — Edit must enforce the same required fields as Create

### Problem

Right now, on Edit, a user can:

* delete a required field
* click Save Draft or Submit/Save Edit
* and the edit still saves

That is wrong.

### Required result

On Edit, if any required field is blank:

* block save
* show required-field error state
* show the missing-fields notification/banner
* do not allow save until all required fields are filled in

This must work across all tabs, not just the current tab.

---

## PART B — Required fields across all tabs must be enforced before edit save

### Required behavior

When editing a listing:

* if tab 1 has a missing required field
* and tab 3 has another missing required field
* and tab 6 has another missing required field

then:

* Save Draft must fail
* Submit/Save Edit must fail
* user must fill in all of them before save is allowed

### Important

Do not only validate visible tab fields.
Validate **all required fields for the listing**, across the entire edit form.

---

## PART C — Listing Type and Current Representation Status with Broker must not be editable after creation

### Required result

On Edit for Seller, Buyer, Landlord, and Tenant:

* **Listing Type** cannot be changed
* **Current Representation Status with Broker:** cannot be changed

### Required UX

* these fields should be locked/disabled/read-only on edit
* if the user somehow tries to change them:

  * show a **red validation-style notice**
  * message should clearly say they cannot be changed after listing creation
* do not let altered values save

### Important

Do not only disable visually.
Also enforce this server-side or in the edit save handler so modified posted values are ignored/rejected.
