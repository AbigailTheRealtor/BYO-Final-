Run a focused audit and fix pass for listing status consistency across the Hire Agent flows.

Current issue:
Different parts of the listing view are showing conflicting status information.

Examples observed:
- Traditional listings showing “Bidding Period Ended”
- Duplicate banners like “Bidding Ended” and “Bidding Period Ended” on the same listing
- Listings marked “Active” while also showing an ended bidding banner
- Status badge, warning banners, and action/edit state do not appear to use the same logic source

This task should identify and fix status inconsistency across Tenant, Landlord, Buyer, and Seller listing detail pages.

CRITICAL SAFEGUARD:
- Do not redesign the flows
- Do not modify listing creation structure
- Do not modify bid submission, counter, accept/reject, match score, or Accepted Bid Summary logic unless directly required by the status consistency fix
- Do not change database schema
- Fix status derivation and banner rendering only

----------------------------------
PART 1 — AUDIT STATUS SOURCES
----------------------------------

For each flow (Tenant / Landlord / Buyer / Seller), audit the exact logic used for:

1. Overall listing status badge
   Examples:
   - Active
   - Pending
   - Expired
   - Sold / Hired
   - Draft

2. Bidding/timer banner
   Examples:
   - Bidding Period Ended
   - Bidding Ended
   - Listing pending — not accepting new bids

3. Action/edit state
   Examples:
   - Accept / Counter / Reject available or hidden
   - “Bidding has ended — edit unavailable”

4. Listing type detection
   Determine how each page detects:
   - Bidding Period
   - Traditional

5. Expiry / timer end detection
   Determine:
   - how listing expiration is calculated
   - how bidding period end is calculated
   - whether the page distinguishes:
     a. listing expired
     b. bidding period ended but listing still active
     c. pending
     d. sold / hired

Return exact files and line areas responsible for each.

----------------------------------
PART 2 — IDENTIFY CONFLICTS
----------------------------------

Find and report all cases where:
- Traditional listings can show bidding-period banners
- duplicate “ended” banners can render together
- status badge and banner can conflict
- active status can coexist with an ended banner in a way that is misleading
- edit/action messaging is inconsistent with the actual status

----------------------------------
PART 3 — FIX STATUS CONSISTENCY
----------------------------------

Implement a consistent rendering rule across all role flows.

Target behavior:

1. Traditional listings
- must NOT show “Bidding Period Ended”
- must use listing expiration / pending / sold status only

2. Bidding Period listings
- may show a bidding-ended banner only if the bidding window truly ended
- must not show duplicate ended banners
- status badge and banner must not conflict confusingly

3. Status hierarchy should be clear and deterministic.
Preferred priority:
- Draft
- Sold / Hired / Finalized
- Pending
- Expired
- Active
And bidding-window messages should be a secondary state, not a conflicting primary status

4. If the intended product behavior is:
   “listing can remain Active after bidding period ends, but no new edits/actions are allowed”
   then make that explicit and consistent in both badge/banners and editability text.
   Do not allow the page to look contradictory.

----------------------------------
PART 4 — DEDUPE BANNERS
----------------------------------

Ensure only one appropriate status/banner message appears for a given listing state.
Examples:
- Do not show both “Bidding Ended” and “Bidding Period Ended”
- Do not show “Bidding Period Ended” on Traditional listings
- Do not show an ended bidding banner if the page has already resolved to a more important state like Pending or Expired unless that secondary state is intentionally displayed and clearly worded

----------------------------------
PART 5 — VERIFY WITH THESE CASES
----------------------------------

Test these cases after the fix:

1. Traditional + active
2. Traditional + expired
3. Bidding Period + active before timer ends
4. Bidding Period + active after timer ends
5. Pending listing
6. Sold / hired listing
7. Draft listing if viewable
8. Buyer listing
9. Seller listing
10. Landlord listing
11. Tenant listing

----------------------------------
OUTPUT REQUIRED
----------------------------------

Return:
1. exact files changed
2. exact status logic issues found
3. exact rules used after the fix
4. confirmation that:
   - Traditional listings no longer show bidding-period banners
   - duplicate ended banners are gone
   - status badge, banners, and action/edit state are consistent
   - no unrelated flow logic was changed
