Fix 1 — Direct Hire must persist listing referral attribution:
File:
app/Http/Controllers/HireAgentDirectController.php

Issue:
Direct Hire / Hire Me listings are created without calling:

ReferralLinkService::persistListingReferral($auction)

Impact:
referring_agent_id is not written to the listing.
agent_referral_links.listing_count does not increment.
My Referrals does not show these listings.
Accepted hire referral attribution may also be lost.

Task:
- In commitListingAndBid(), after the listing/auction is saved and before or during the same transaction flow where other listing metadata is persisted, call the same referral persistence service used by the standard Hire Agent listing creation paths.
- Match the existing pattern used in Seller/Buyer/Landlord/Tenant Livewire hire-agent listing components.
- Do not duplicate referral records.
- If the service is already safe/idempotent, use it directly.
- If needed, guard so it only runs once per created listing.

Done:
- Direct Hire-created listings receive referring_agent_id when a referral code/session exists.
- agent_referral_links.listing_count increments correctly.
- Direct Hire listings appear in My Referrals under the listing stage.
- Accepted hire referral attribution can flow from listing to accepted summary.

Fix 2 — Normalize referral fee display keys:
Issue:
Standard Hire Agent listing form stores referral fee as listing meta:
referral_percentage

Direct Hire / Agent preset bid flow stores referral fee as bid meta:
referral_fee_percent

This causes Direct Hire listings to show blank referral fee in:
- Hire Listings Hub Ref % column
- Public listing header
- some bid/detail views

Task:
- Do not rename meta keys.
- Do not migrate existing data.
- Add safe fallback reads wherever referral fee is displayed:
  1. Prefer listing meta referral_percentage when available.
  2. Fall back to bid meta referral_fee_percent when listing meta is empty and a direct-hire seed bid exists.
- Keep formatting consistent with a trailing % sign.
- Do not create automated fee math.

Done:
- Standard created listings still show referral fee normally.
- Direct Hire-created listings no longer show “—” when the bid contains referral_fee_percent.
- Referral fee display is consistent in hub/header/detail views.
