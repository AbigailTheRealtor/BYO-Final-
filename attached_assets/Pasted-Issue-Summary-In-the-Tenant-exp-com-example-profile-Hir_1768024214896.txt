Issue Summary:
In the Tenant@exp.com
 example profile, Hire a Tenant and Hire a Landlord listings correctly save and display under their own tabs. However, Hire a Buyer and Hire a Seller listings are incorrectly being saved/displayed under the Hire a Tenant tab.

Result:
When a user creates a Buyer or Seller listing, the “actual listing” display shows the Tenant listing fields/questions instead of the Buyer/Seller fields.

Cause:
The prior developer did not finish the backend models/fields for Buyer and Seller listings and incorrectly mapped Buyer/Seller listing creation + retrieval to the Tenant listing schema/routes.

Required Fix:
Buyer listings must create/read/update/display using the Buyer listing schema and fields, and Seller listings must create/read/update/display using the Seller listing schema and fields, matching the exact questions/text/format already defined for each flow.
