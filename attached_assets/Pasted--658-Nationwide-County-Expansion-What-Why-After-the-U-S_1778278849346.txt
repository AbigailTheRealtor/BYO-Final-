#658 — Nationwide County Expansion

What & Why:
After the U.S. city import, us_cities now contains 32,145 cities, but only 12,541 have county_id populated because us_counties contains only 875 counties. Full U.S. coverage requires expanding us_counties to include all U.S. counties/parishes/boroughs/census areas and then re-running county mapping.

Goal:
Expand nationwide county coverage so City → County → State auto-population works across the U.S., not just partially.

STRICT SAFEGUARDS:
- DATA-ONLY TASK.
- DO NOT modify Livewire components.
- DO NOT modify Blade files.
- DO NOT change Select2/autocomplete logic.
- DO NOT touch users, listings, bids, drafts, or accepted bid summaries.
- DO NOT truncate us_cities.
- DO NOT delete existing counties.
- DO NOT run migrate:fresh.
- DO NOT reseed unrelated tables.
- DO NOT perform legacy cleanup.
- Use dry-run first if available.
- Stop and report before any risky operation.

Relevant finding:
Task #653 expanded us_cities from 971 to 32,145, but 19,123 cities still lack county_id because us_counties only has 875 counties. County expansion was flagged as follow-up #658.

Relevant files:
app/Console/Commands/ImportCensusCities.php
app/Models/UsCity.php
app/Models/UsState.php
app/Models/UsCounty.php

Find how us_counties was originally seeded.
Identify whether there is already:
- a county import command
- a county seeder
- a Census county dataset reference
- a county mapping utility
- a FIPS/county source file

Do not change anything yet.

Record:

3. Find best official source
Use an official U.S. Census county dataset if already supported in the codebase.
Preferred source:
- Census county / county equivalent data with state FIPS, county FIPS, county name, and state abbreviation.

4. Dry-run expansion
If an existing dry-run mode exists, run it first.
If not, create a dry-run/report-only plan before making changes.

Dry-run/report should show:
- counties currently in us_counties
- counties expected after import
- new counties to add
- duplicate counties
- naming conflicts
- county equivalents handled:
  - Louisiana parishes
  - Alaska boroughs/census areas
  - independent cities where applicable
  - District of Columbia

STOP after dry-run/report unless safe to proceed.

5. Live data expansion
Only after dry-run is clean:
- Add missing counties to us_counties.
- Preserve existing county IDs.
- Do not delete or overwrite existing counties unless clearly safe.
- Use state_id relationships correctly.
- Store names consistently with existing format, e.g. “Pinellas County”.

6. Re-run county mapping
After counties are expanded, re-run the existing city-to-county mapping logic only.
Do not re-import or duplicate cities unless the existing command requires it safely.
