We previously fixed many legacy placeholders that used inconsistent wording like:
- Describe...
- Specify...
- Enter details...
- Type here...
- Other...
- -- Select --

The platform standard is now:
- Consistent placeholder formatting
- Consistent capitalization
- Consistent example formatting
- Consistent “Select” placeholders
- Consistent Create/Edit parity across all roles

This task is an audit-first pass only. Do not mass-change code blindly before identifying all inconsistencies.

Platform Placeholder Standards

1. Select placeholders
All select/select2 placeholders should simply be:

Select

Remove inconsistent versions like:
- -- Select --
- Select timeline
- Select status
- Choose one
- Choose option
- Select application
etc.

2. Standard input placeholder format
Use:

Enter [field title] (e.g., Example)

Examples:
- Enter desired sale price (e.g., 450000)
- Enter lease term (e.g., 12 Months)
- Enter property description (e.g., Recently Renovated Kitchen, Waterfront Views, Flexible Lease Terms)

3. “Other” companion input rule
Do NOT use the word “other” inside placeholder text.

Use only the actual field title.

Correct:
- Enter appliance (e.g., Wine Refrigerator)
- Enter amenity (e.g., Pickleball Court)
- Enter property feature (e.g., Outdoor Kitchen)
- Enter financing type (e.g., Seller-Held Second Mortgage)

Incorrect:
- Enter other appliance...
- Enter other amenity...
- Enter other feature...

4. Capitalization rules
Inside all example text:
- Capitalize the first letter of each example phrase
- Use title-style capitalization
- Preserve acronyms like HOA, MLS, ADU, ZIP

Incorrect:
(e.g., recently renovated kitchen, waterfront views)

Correct:
(e.g., Recently Renovated Kitchen, Waterfront Views)

5. Consistency requirements
- Same/similar fields across roles should use matching placeholder structure
- Create and Edit/Draft flows must match
- Shared fields between Hire Agent and Offer Listings should follow the same formatting style unless intentionally role-specific
- Do not introduce legal wording changes
- Do not rename labels
- Do not alter validation or functionality
