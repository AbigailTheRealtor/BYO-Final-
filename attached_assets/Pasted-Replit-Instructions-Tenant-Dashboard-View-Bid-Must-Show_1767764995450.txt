Replit Instructions — Tenant Dashboard “View Bid” Must Show FULL Agent Bid Package (All Tabs/Sections)
GOAL

When a Tenant goes to Dashboard → Agent Bids and clicks View Bid, the system must open a dedicated Full Agent Bid Preview that displays the agent’s complete bid submission (the same content shown in the listing’s “Private Bid Summary”), including:

Agent Overview & Qualifications

Agent Presentation & Promotional Materials

Agent Credentials

Agent Contact Information

Private Compensation & Agency Agreement Terms (ALL fields/questions)

Offered Services (full detail)

Match Score (full module)

View Bid must NOT redirect to the listing page.

SAFEGUARDS (MUST FOLLOW)
DO NOT:

rename/delete any existing database fields, migrations, models, or keys

change existing bid submission logic, timers, counter logic, or accept/counter/reject behavior

alter the listing page’s “Private Bid Summary” layout/content (reuse it)

show other agents’ bids in the View Bid preview

expose internal scoring formulas/weights

change Traditional listing privacy rules

MUST:

reuse the same rendering logic/components used by the listing’s “Private Bid Summary”

load the agent’s latest submitted bid snapshot (not drafts)

keep the View Bid preview read-only except for Accept/Counter/Reject actions

design this view to be role-neutral for future Buyer/Seller/Landlord dashboards

REQUIRED DASHBOARD BEHAVIOR (Tenant View)
Dashboard cards remain summary-only

The Agent Bids dashboard should remain a comparison surface:

Match Score badge

Broker Terms % / Services % (optional)

Differences / Extra Services / Not Offered (optional)

Buttons: View Bid / Accept / Counter / Reject

✅ Do NOT show full agent profile sections or full agreement terms on the dashboard card.

REQUIRED “VIEW BID” BEHAVIOR
When Tenant clicks “View Bid”

DO NOT send them to the listing page

Open a dedicated Full Agent Bid Preview page (recommended) or modal

ROUTE (Recommended)

Create/use a route like:

/tenant/agent-bids/{bid_id}

This route must:

load only the selected bid + minimal listing context (role + listing ID)

not load the full listing page UI

not display other agents’ bids

FULL AGENT BID PREVIEW — REQUIRED LAYOUT
1) Context Header (Uniform)

Display at top:

Hire a Tenant’s Agent

Listing ID: {listing_id}

(Required so later it can show “Hire a Buyer’s Agent”, etc.)

2) Agent Identity + Status (This is part of the bid package)

Show:

Agent Name

Agent Email

Status (Active / Countered / Accepted)

Match Score badge

3) Full Agent Bid Package Sections (MANDATORY)

Render these sections in this order (or the same order as your listing’s Private Bid Summary), and include all fields/questions:

A) Agent Overview & Qualifications (FULL)

all text answers and selections

any “about me” / experience / specialties / service area info

B) Agent Presentation & Promotional Materials (FULL)

video links / uploads

marketing links

documents/assets the agent attached (if applicable)

C) Agent Credentials (FULL)

license details (as you store them)

certifications/designations

years active, etc. (whatever your form includes)

D) Agent Contact Information (FULL)

phone, email, preferred contact method, etc. (as your platform stores)

E) Private Compensation & Agency Agreement Terms (FULL)

Show every term/question the agent submitted, including:

Tenant’s Broker Commission Structure

Tenant’s Broker Commission Fee

protection period

termination/retainer (options + amounts)

agency relationship type

and all other questions/fields included in that section

F) Offered Services (FULL)

Show the same detailed service view you already display in the listing’s Private Bid Summary:

selected services

extra services

not offered services (if present)

G) Match Score (FULL MODULE)

Must display:

Overall Match %

Broker Compensation % + X/Y fields matched

Services % + X/Y services matched

Footer label: Compared to Your Bid

4) Action Buttons (Required)

At bottom of Full Bid Preview:

View Details (optional if already on full view; can remove)

Accept

Counter

Reject

Back to Agent Bids

Actions must apply to the selected bid_id.

DATA RULES (CRITICAL)
Latest submitted snapshot only

The View Bid preview must display:

the agent’s latest submitted bid version

not in-progress edits or drafts

Counter privacy

If countering exists, counter details must remain private to the Tenant + that agent, consistent with your current counter logic.

QA CHECKLIST (MUST PASS)

View Bid opens full bid preview (not listing page)

Full preview includes all Agent sections (Overview, Presentation, Credentials, Contact)

Full preview includes ALL Private Compensation & Agency Agreement Terms fields/questions

Full preview includes full Offered Services detail

Full preview includes full Match Score module + “Compared to Your Bid”

No other agent bids are visible inside the preview

Dashboard remains summary-only

Listing page Private Bid Summary remains unchanged (component reused)
