TASK 1 — Lease Option “Yes” Text Color Consistency (All Roles)
Issue

Under Interested in a Lease Option Agreement, when user selects Yes, the text color differs between roles (Landlord vs Seller/Buyer/Tenant).

Required Change (UI styling only)

Use the exact same “blue” Yes selection color styling currently used in Hire a Landlord.

Apply that same styling to Hire a Tenant, Hire a Buyer, Hire a Seller.

Requirements

This must be consistent in all property types where the Lease Option question appears.

Must affect only the “Yes” selected state for this specific field (and any shared “Yes/No” toggle used there), not global buttons.

Implementation Notes (acceptable approaches)

Best: refactor to a shared toggle component (Blade partial or Livewire subcomponent) and reuse it across roles.

Or: apply a shared CSS class used by Landlord’s flow to the other flows’ Lease Option Yes/No control.

Acceptance Test

On each flow, selecting Yes should show the same blue color as Landlord (identical hex/class).

Selecting No should remain unchanged.

TASK 2 — “$ $” Duplicate Symbol Bug (Lease Option Compensation Fields)
Issue

For Seller, Buyer, Tenant, Landlord listings:
If user selects Yes on Interested in a Lease Option, then in these two fields:

Compensation for Creating the Lease Option Agreement

Compensation if Purchase Option is Exercised

…when the user switches compensation type to $, the UI shows a duplicate symbol like “$ $” and the second $ must be removed.

Required Change

Ensure only one currency symbol is rendered when $ is selected.

Must work consistently in both fields above, in all roles.

Likely Cause

Two sources rendering $:

one from a “prefix” UI element (input-group prepend), and

another from the input formatting / mask / placeholder / conditional label.

Implementation Requirements

If the selected type is $, render exactly one prefix symbol.

If the selected type is % (or other types), render the correct symbol once.

Do not change any stored values, only the UI rendering.

Acceptance Test

Toggle Yes → open those Lease Option compensation inputs → switch type to $:

shows $ [input] (single $ only)

no doubled prefix under any rerender (switch tabs, back/forward, save draft, reload)
