# TASK: Complete Default Profile System for Agent Presentation & Marketing Materials + Agent Credentials & Contact Info With Property-Type Overrides

## GOAL

Extend the existing default profile system so it also saves and auto-fills all fields under:

* Agent Presentation & Marketing Materials
* Agent Credentials & Contact Info

for all agent bid types:

* Hire a Seller Agent
* Hire a Buyer Agent
* Hire a Tenant Agent
* Hire a Landlord Agent

The system should support:

* one main default profile per role
* optional property-type-specific default profile overrides

This allows agents to reuse one main default profile across all property types, while also giving them the ability to save a different default profile for a specific property type when needed.

---

# HOW IT SHOULD WORK

## Main role default profile

Each role should have one main default profile:

* Seller Default Profile
* Buyer Default Profile
* Tenant Default Profile
* Landlord Default Profile

This main role default profile should populate all property types under that role unless a property-type-specific override exists.

---

## Property-type-specific override

Each property type under a role can also have its own override profile.

Example for Seller:

* Seller Residential Default
* Seller Income Default
* Seller Commercial Default
* Seller Business Default
* Seller Vacant Land Default

If an agent saves a property-type-specific default profile, that version should be used for that property type moving forward.

It should not overwrite the main role default profile or the default profiles for the other property types.

---

# EXAMPLE BEHAVIOR

## Example 1

An agent first saves a default profile while bidding on **Seller Residential**

That becomes the main **Seller Default Profile**

Now when the agent opens:

* Seller Residential
* Seller Income
* Seller Commercial
* Seller Business
* Seller Vacant Land

the system should use the main Seller default profile unless that property type has its own saved override.

## Example 2

Later, the agent opens **Seller Commercial**, edits the profile wording for commercial use, and saves that as the Commercial default profile

From that point forward:

* Seller Commercial uses the Seller Commercial default profile
* Seller Residential still uses the main Seller default profile
* Seller Income still uses the main Seller default profile
* Seller Business still uses the main Seller default profile
* Seller Vacant Land still uses the main Seller default profile

This same override logic should apply to Buyer, Tenant, and Landlord.

When loading a bid form, use this order:
