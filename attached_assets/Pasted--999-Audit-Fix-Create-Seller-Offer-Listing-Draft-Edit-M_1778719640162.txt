ISSUES TO AUDIT + FIX

1. Draft vs Edit button state is wrong
Problem:
On Create Seller Listing, when loading a saved draft, the button says “Save Edit” even though it is still a draft. It should say “Save Draft.”
Only after the listing is actually created/published should it say “Save Edit.”

Expected:
- /offer-listing/seller create mode = Save Draft
- Loaded saved draft that has not been published = Save Draft
- Existing created listing edit mode = Save Edit
- Clicking Save Draft must not submit/publish the listing.
- Clicking Save Edit should only update an already-created listing.

Audit:
Check how the blade determines edit mode. Do not rely only on URL /edit/{id} if the record is actually a draft. Check listing status/workflow flags if available.

2. Next button not working on Create Seller Listing
Problem:
When a draft is loaded, the Next button stops working.

Expected:
- Next/Back works normally after loading a draft.
- Tab validation still works.
- No console errors.
- Draft hydration should not break tab navigation.
