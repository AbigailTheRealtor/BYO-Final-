# Task: Fix Hire Agent draft modal so saved drafts appear when user clicks role-specific Hire Agent listing links

Problem:
On Hire a Landlord's Agent, saved drafts are not appearing when opening an existing listing link like:
/hire/agent/auction/landlord/38

Also, for all four Hire Agent flows:
- Hire a Tenant's Agent
- Hire a Buyer's Agent
- Hire a Seller's Agent
- Hire a Landlord's Agent

When the user selects the role under “Select Agent Type,” the saved draft modal should appear every time a draft exists — not only the first time they originally selected that role from the dashboard.

Important context:
All four flows should use the shared TenantAgentAuction Livewire component and shared tenant-agent-auction Blade view. Do not create separate logic unless the existing architecture already requires it.

3. Where saved drafts are queried and where $hasDrafts is set.
4. Whether drafts are being filtered incorrectly when a listing ID exists in the URL.
5. Whether the draft modal is only triggered on initial dashboard selection instead of also triggering on role/listing load.

Required behavior:
1. If a user opens any Hire Agent role flow and a saved draft exists for that user + role + flow, the draft modal should appear.
2. This must work from:
   - dashboard role selection
   - direct URL entry
   - clicking a “Hire a Landlord's Agent” listing link
   - clicking a “Hire a Tenant's Agent” listing link
   - clicking a “Hire a Buyer's Agent” listing link
   - clicking a “Hire a Seller's Agent” listing link
3. Example URL that must work:
   /hire/agent/auction/landlord/38
4. The presence of listing ID 38 must not prevent the draft lookup unless the intended design is to show only drafts connected to that same listing. If current drafts are role-based, preserve role-based behavior.
5. The modal should not show if there is no saved draft.
6. The modal should not show duplicate times after Livewire refreshes.
7. The modal should be re-triggerable when switching/selecting a different Agent Type if that role has a draft.
8. Landlord, Tenant, Buyer, and Seller should all behave consistently.

Implementation guidance:
- Centralize draft detection inside TenantAgentAuction.php so all four roles use the same logic.
- Normalize role values consistently, likely:
  tenant, buyer, seller, landlord
- Make sure the route parameter role and selected Agent Type update the same internal property.
- Make sure draft lookup runs when:
  - component mounts
  - role is changed from Select Agent Type
  - listing ID is present in the URL
- If using a browser event to open the modal, dispatch it after $hasDrafts is confirmed true.
- Add a front-end guard so the same modal does not open multiple times from one render cycle.
- Do not rely only on dashboard-origin session flags to show the draft modal.
