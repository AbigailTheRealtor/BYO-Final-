* “Other” inputs **do not appear when reopening edit**

Please investigate the **underlying synchronization problem** rather than patching each symptom separately.

The likely issue involves:

* Livewire hydration timing
* Select2 initialization order
* selected `<option>` attributes not matching component state
* conditional visibility logic only running on `change` instead of also running on initial load

The goal is to **fix the underlying state/UI synchronization** so these issues do not recur.
