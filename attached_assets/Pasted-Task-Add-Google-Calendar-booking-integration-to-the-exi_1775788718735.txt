Task: Add Google Calendar booking integration to the existing Voice Receptionist system.

Goal:
Allow the AI receptionist to capture booking intent during calls and create a calendar event when Google Calendar is connected. If calendar is not connected or booking fails, the system should gracefully fall back to a callback/request capture flow.

Context:
Voice system is already working with:
- inbound AI call handling
- ElevenLabs + fallback
- inbox integration
- lead capture
- call classification
- SMS follow-up
- Ring-First → AI fallback
We are EXTENDING the existing system only.

CRITICAL RULES:
- Do NOT refactor VoiceCallService unless absolutely required
- Do NOT create a separate booking system
- Do NOT break current voice flow, inbox logic, or conversation storage
- Use additive changes only
- Reuse existing calendar integration patterns if they already exist in the project

---

## IMPLEMENTATION

Goal behavior:
If a caller wants to book an appointment, the AI should:
1. detect booking intent
2. collect date, time, and caller name if missing
3. attempt to create a calendar event if Google Calendar is connected
4. verbally confirm success if booking succeeds
5. if booking cannot be completed, gracefully say someone will follow up to confirm
