### Problem 2: Broker Compensation & Agency Agreement Terms count is wrong

The Terms Match is also wrong because it appears to be counting **each input box as its own term**, instead of counting the correct parent agreement terms / logical fields.

Example of what should **not** happen:

* flat fee amount = 1
* percentage amount = 1
* another side of same combo field = 1

That inflates the score.

For combo structures, the scoring should count the **logical agreement term**, not every child input box.

# Desired scoring behavior

## A. Offered Services scoring should count only checked service items

### Correct rule

The denominator should equal:

**the number of actual service checkboxes selected by the Tenant on the original listing**

The numerator should equal:

**how many of those original selected service checkboxes are still included in the Agent bid / counter**

### Example

If Tenant selected 38 services:

* denominator = 38
* if Agent kept all 38 = 38/38
* if Agent removed 2 = 36/38
* if Agent added 5 extra services = still compare against the original 38 only

Added services can be shown separately, but they should not increase matched count.

---

## B. Terms scoring should count logical agreement terms, not raw inputs

The denominator should equal:

**the number of logical agreement terms the Tenant originally filled out**

The numerator should equal:

**how many of those logical terms still match in the Agent bid / counter**

### Important

Count **one logical term**, not multiple child boxes.

## Part 2: Fix Broker Compensation & Agency Agreement Terms scoring

### Build a logical term map

Instead of comparing raw every-input keys, define a **Tenant logical scoring map**.

Each logical term should count as **one term**.

Example structure:

* Tenant’s Broker Commission Structure
* Tenant’s Broker Lease Fee
* Payment Timing for Broker Fees
* Interested in Purchasing a Property
* Tenant’s Broker Purchase Fee
* Interested in a Lease-Option Agreement
* Compensation for Creating the Lease-Option Agreement
* Compensation if Purchase Option is Exercised
* Protection Period Timeframe
* Early Termination Fee
* Retainer Fee
* Retainer Fee Amount
* Retainer Fee Application
* Tenant Agency Agreement Timeframe
* Acceptable Brokerage Relationship
* Additional Terms

That means if a field is a combo field like:

* flat fee + percentage
  or
* percent + flat fee

it still counts as **one logical term**, not 2 or 3.

---

## Part 3: Treat combo fields as one scored unit

For any combo fee structure:

Examples:

* Flat Fee + Percentage of Net Aggregate Rent
* Percentage of Total Purchase Price + Flat Fee

The score should evaluate the **entire logical term** as one term.

### Example

`Tenant's Broker Lease Fee` counts as one term:

* compare fee structure type
* compare any required sub-values for that term
* then determine whether that one term matches / changed / differs

Do not separately count:

* structure dropdown = 1
* flat fee input = 1
* percentage input = 1

That would be wrong.
