Audit every tooltip across both major listing systems and make them launch-ready:

1) Create Offer Listing flows:
- Seller
- Buyer
- Landlord
- Tenant

2) Hire Agent Listing flows:
- Hire a Seller’s Agent
- Hire a Buyer’s Agent
- Hire a Landlord’s Agent
- Hire a Tenant’s Agent

Goal:
Review every tooltip/icon help message in these create/edit/draft flows and confirm the text is accurate, role-correct, legally safe, neutral, polished, and consistent across the platform.

Audit requirements:
- Tooltip language must match the correct workflow:
  - Offer Listing tooltips must not mention “Hire Agent,” “Agent bids,” or agent-service language unless the field truly relates to Agent involvement.
  - Hire Agent tooltips must not sound like property-offer/listing tooltips unless the field is shared and appropriate.
- Tooltip language must match the correct role:
  - Seller, Buyer, Landlord, Tenant, Agent, Broker, Owner must be used correctly.
  - Remove any copied tooltip text from the wrong role.
  - Example: no Tenant wording inside Landlord flow, no Seller wording inside Buyer flow, etc.
- Tooltips must be neutral and usable whether the client or an Agent is completing the form on their behalf.
- Capitalization must follow platform standard:
  - Capitalize Seller, Buyer, Landlord, Tenant, Agent, Broker, Owner when referring to platform roles.
  - Use ZIP, MLS, HOA, COA, CDD, etc. consistently where applicable.
- Grammar, punctuation, and sentence structure must be polished and launch-ready.
- Tooltip wording should explain the field clearly without repeating the placeholder word-for-word.
- Remove outdated, confusing, duplicate, or overly long tooltip text.
- Make sure tooltip text aligns with the actual field label, dropdown options, conditional logic, and public visibility rules.
- For address/location tooltips, confirm privacy language is correct:
  - City, County, and State may display publicly where intended.
  - Full address/unit details should only display publicly if that is the intended behavior for that flow.
- For compensation, fees, agreement terms, protection periods, retainers, early termination fees, lease fees, purchase fees, seller/buyer broker fees, and referral-related fields:
  - Make tooltip language clear, non-legal-advice, and consistent.
  - Do not create promises, guarantees, or legal conclusions.
  - Keep wording platform-neutral and compliance-safe.
- For service checklists:
  - Tooltips should describe the service clearly.
  - Make sure Full Service language does not appear in Limited/flat-fee contexts unless intentional.
  - Do not modify frozen/legacy Limited Service logic unless the tooltip is currently visible in the active launch flow.
- For “Other” fields:
  - Tooltip should explain that the user can enter a custom value.
  - Do not use “Describe...” placeholder language.
  - Do not display the literal word “Other” on public/review pages when custom text exists.

Files/areas to audit:
- All Blade partials and Livewire views used by Offer Listing create/edit/draft flows.
- All Blade partials and Livewire views used by Hire Agent create/edit/draft flows.
- Any shared tooltip arrays, helper methods, config files, or JS tooltip initialization code.
- Do not limit the audit to one role. Compare all four roles side-by-side.

Safeguards:
- Do not change database schema, migrations, routes, controllers, or validation unless a tooltip is clearly generated from one of those places.
- Do not change field names, IDs, Livewire properties, saved data keys, or form logic.
- Do not remove tooltip icons unless the field should not have a tooltip.
- Do not alter Select2 behavior, icons, placeholders, or styling unless directly required to fix tooltip display.
- Keep changes text-only whenever possible.
- Preserve existing working tooltip rendering and icon behavior.
- Do not touch unrelated pages outside Create Offer Listing and Hire Agent Listing flows.

Deliverables:
1) Provide a list of every file changed.
2) Provide a role-by-role summary of tooltip issues found and fixed.
3) Confirm there are no remaining wrong-role tooltip references.
4) Confirm Offer Listing tooltips and Hire Agent tooltips no longer bleed into each other.
5) Confirm capitalization and grammar are consistent.
6) Confirm no tooltip text still uses outdated “Consumer” language where the platform standard requires Seller, Buyer, Landlord, Tenant, or Client.
7) Confirm no visible launch-flow tooltip contains copied text from the wrong role.
8) Run grep/search checks for common wrong-role words inside each role folder before finalizing.

Important:
This is a launch-polish audit. Be thorough, but keep the changes safe and surgical.
