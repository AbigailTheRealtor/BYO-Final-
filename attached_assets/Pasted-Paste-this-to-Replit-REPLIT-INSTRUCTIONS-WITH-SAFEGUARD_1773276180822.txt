The Submit button behavior is:

* it briefly flashes dark green

# REQUIRED INVESTIGATION

Please inspect for all of these possible causes:

## 1) Hidden required fields

Check whether there are hidden required inputs on:

* current tab
* previous tabs
* conditionally hidden sections
* file upload/video sections
* commercial/residential conditional sections

A hidden required field can cause the button to briefly enable and then fail browser validity.

## 2) JS form-validity gating

Check whether the button enabled/disabled state is controlled by custom JS such as:

* `checkFormValidity()`
* `syncBeforeSave()`
* mutation/change listeners
* delayed re-check after click

If so, identify exactly why it is turning the button back off.

## 3) Livewire submit method mismatch

Check whether:

* the button is calling the wrong method
* the final tab is using the wrong submit handler
* draft and final submit use different paths and only draft works

## 4) Validation failing silently

Check whether submit is actually firing but failing server-side validation without visible feedback.

If so:

* surface the validation errors visibly
* do not leave the button in a fake dead state

## 5) JS-controlled field sync

Check whether Select2 or other JS-controlled inputs are not being synchronized into Livewire state before submit.

If required data is missing server-side because JS state was never synced, that can cause silent validation failure.

## 6) Button state reset after click

Check whether:

* `wire:loading`
* disabled attributes
* class toggles
* state watchers
* Alpine/JS listeners

are resetting the button immediately after click.
