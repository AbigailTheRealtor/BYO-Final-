Replit Instructions — Fix “Submit Button Disabled After Editing Draft Listing Date” (All 4 Listing Types) + Keep Redirect Fix
Problem

When a user reopens a Draft listing and changes Listing Date, the Submit button becomes disabled or stays disabled even though all required fields appear completed.

This occurs on:

Tenant drafts

Buyer drafts

Seller drafts

Landlord drafts

Required Behavior

For ALL 4 listing types:

Reopen Draft

Change Listing Date

Submit button must remain enabled (or re-enable correctly)

Clicking Submit triggers store() once

Success → redirect to detail page (existing redirect work must remain)

SAFEGUARDS (NON-NEGOTIABLE)
DO NOT:

delete/rename DB tables/columns/models/migrations

rename routes/controllers/Livewire components/views

change any UI text (labels/tooltips/placeholders/tabs)

remove validation rules

change required field definitions from business requirements

ONLY ALLOWED:

fix Livewire binding for listing_date on draft edit forms

fix any “canSubmit/disabled” gating logic so it updates correctly

ensure conditional required fields are only required when visible/applicable

add minimal temporary logs

PART A — Fix Listing Date binding on Draft Edit screens (most common cause)

On ALL 4 “Edit Draft” components (not just create forms), ensure Listing Date input is Livewire-bound:

✅ REQUIRED:

wire:model.lazy="listing_date" OR wire:model.defer="listing_date"

If Listing Date uses a JS datepicker, it MUST sync to Livewire:

on date change: @this.set('listing_date', selectedValue)

Important: This must be on the DRAFT edit view too (often different blade file than create view).

PART B — Fix Submit Disabled Logic (stale computed flags)

Find what controls the disabled state of the Submit button on draft edit pages.

Common patterns:

$canSubmit

$isComplete

$formValid

disabled attribute using a computed rule

REQUIRED
B1) Ensure the disabled state is NOT permanently tied to “draft mode”

Example bug:

if record is draft, submit is disabled unless some flag is manually toggled

Fix:

submit must be allowed for drafts when required fields are satisfied

B2) Ensure the disabled state recomputes after listing_date changes

If $canSubmit depends on $this->listing_date:

it must recompute when listing_date updates

✅ REQUIRED Livewire hook (choose one):

implement updatedListingDate() to re-run validation readiness logic
OR

ensure computed $canSubmit is derived live from current component properties (not from initial mount snapshot)

B3) Remove “front-end gating” that blocks submission silently

If the button is disabled based on client logic AND it’s incorrect, users can’t submit at all.

✅ Preferred approach:

allow Submit button to be clickable

rely on backend validation (show errors) if something is missing

If you must keep disabled gating, it must be 100% accurate and must update on any change.

PART C — Fix hidden/conditional required fields causing false “incomplete” state

This is the #1 reason “everything looks filled out” but Submit stays disabled:

backend requires a field that is hidden due to conditional logic

or Livewire never received a value for a field that appears filled out (JS widget not bound)

REQUIRED

On each draft edit component, log validation-required fields vs actual values when user changes listing_date:

log missing required fields

log listing_date value format

Update validation rules so conditional fields are required ONLY when the condition is active.

Example (conceptual):

required_if:leasing_type,Commercial etc.

Do NOT remove rules; make them conditional properly.

PART D — Ensure Draft Edit uses the same submit method & redirect event

When user clicks Submit from a draft edit page:

It must call the same store() / submit() method path as create submit

It must reach the force-redirect dispatch once saved

✅ Confirm the draft edit submit uses:

<form wire:submit.prevent="store">

submit button wire:target="store"

PART E — Acceptance Criteria (must test all 4 types)

For each listing type (Tenant/Buyer/Seller/Landlord):

Create listing → Save Draft

Reopen draft

Change Listing Date
✅ Submit button becomes enabled (or stays enabled)

Click Submit once
✅ One toast
✅ Immediate redirect to detail page
✅ Listing appears under correct tab

REQUIRED PROOF BACK FROM REPLIT

They must paste:

Listing Date input line (draft edit view) for each type showing:

wire:model.lazy or wire:model.defer

Submit button line showing:

wire:target="store"

wire:loading.attr="disabled"

The logic controlling disabled state (the $canSubmit or equivalent) and how it updates when listing_date changes.
