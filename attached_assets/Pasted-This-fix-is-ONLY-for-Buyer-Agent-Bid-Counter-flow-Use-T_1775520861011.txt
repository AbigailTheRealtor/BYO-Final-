This fix is ONLY for **Buyer Agent Bid → Counter flow**.

## PROBLEM

When a Buyer clicks **Counter** on an Agent Bid, the form is incorrectly pre-populating from:

* the Buyer’s original listing

INSTEAD OF:

* the Agent’s latest submitted bid

This creates incorrect negotiation flow and mismatched data.

---

## EXPECTED BEHAVIOR (MATCH TENANT EXACTLY)

When BUYER clicks Counter on an Agent Bid:

✔ Pre-populate from:
→ the AGENT’S latest submitted bid (or latest counter)

NOT:
✘ original Buyer listing
✘ partial/stale values

---

## SOURCE OF TRUTH RULE (CRITICAL)

Follow Tenant logic exactly:

IF latest record exists:
use latest_counter_terms (if exists)
ELSE:
use original_agent_bid_terms

NEVER fallback to listing once an agent bid exists.

---

## WHAT MUST PRE-POPULATE (ALL FROM SAME SOURCE)

From the SAME latest agent bid record:

1. Broker Compensation & Agency Agreement Terms
2. Additional Terms
3. Additional Details
4. Offered Services (FULL service checklist)

Do NOT mix sources between sections.
