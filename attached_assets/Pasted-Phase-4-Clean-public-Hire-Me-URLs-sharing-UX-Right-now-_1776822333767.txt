We are moving to Phase 4: Clean public Hire Me URLs + sharing UX.

Objective

Add a clean, shareable public URL structure for the direct Hire Me flow while preserving the existing direct-hire route and core flow.

The current direct route works technically, but it is too internal-looking for sharing:

/hire/agent/direct/{agentId}/{role}/{propertyType}

We need a cleaner public-facing sharing layer.

Goal

Create a public Hire Me URL structure like one of these patterns:

/hire/{agentSlug}/{role}/{propertyType}
or
/hire/{agentSlug}/{role}
with property type optional when appropriate

Use the safest structure based on the current app architecture.

Do not remove the existing direct route yet.
The existing route should remain intact as the underlying working path until the clean public route is verified.
