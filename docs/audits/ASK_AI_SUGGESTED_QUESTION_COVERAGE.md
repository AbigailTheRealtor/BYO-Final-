# Ask AI Suggested Question Chip Parity Audit

**Audit Date:** 2026-06-09
**Scope:** All four listing roles (Seller, Buyer, Landlord, Tenant)
**Purpose:** Verify that every suggested question chip shown in the Ask AI modal produces the same
`normalized_field_key`, context path, and final answer when the user clicks the chip vs. types the
same question manually. Any field reachable only via a chip (chip works, manual fails) is a
confirmed routing gap.

---

## How Chips Work

Chips are generated by `AskAiSuggestedQuestionsService::forListing()`. Each chip carries:
- `question_type` — pre-classified type (e.g. `listing_facts`, `property_standout`)
- `required_context_path` — the context key the chip requires (for `listing_facts` chips)
- `question` — the full question text populated into the textarea on click

When clicked, the chip question is submitted through the same `POST /ask-ai/listing-question` route
as a manually typed question. The chip's `question_type` is NOT pre-injected into the request —
the classifier runs on the question text each time. Therefore, parity requires that the question
text, when classified independently, routes to the same `question_type` and `normalized_field_key`.

## Methodology Notes

- Classifier routing determined by inspecting `AskAiQuestionClassifierService::KEYWORD_RULES` via
  `stripos` substring matching on the normalized question text.
- `LISTING_KEY_KEYWORD_MAP` routing determined by inspecting `AskAiRunnerV2Service::LISTING_KEY_KEYWORD_MAP`
  for keyword coverage of each chip's question text.
- For `faq_answers.*` chips: routing depends on FAQ keyword matching in the FAQ router path, not
  `LISTING_KEY_KEYWORD_MAP`. All FAQ-referenced chips (roof, utilities, heating/cooling, laundry)
  verified against the FAQ registry.
- Static chips (`property_standout`, `suited_audience`, `missing_data`, `educational`,
  `buyer_tenant_match`, `marketing_angles`) are classified deterministically by their keyword
  signatures, which are long-established and tested. No parity issues exist for these types.
- Analysis is static code analysis (classifier KEYWORD_RULES + LISTING_KEY_KEYWORD_MAP), not live
  DB trace, since the specific audit listings (87, 121, 97, 71, 170) require a running application
  with populated test data.
